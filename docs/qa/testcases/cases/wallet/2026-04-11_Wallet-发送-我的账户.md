# 发送 - 我的账户（选择地址 - 账户 Tab）

> 规则文档：`docs/qa/rules/account-rules.md` - 第 1、2、3.4 章
> 测试端：iOS / Android / Desktop / Extension（Web 端无常规 Send，仅 §10 涉及 Web 端 Swap 自定义接收地址）
> 变更说明：
> - v1 首版——覆盖发送流程「选择地址」弹窗中「账户」Tab 的账户列表展示、派生路径切换（BTC/LTC）、BTC 新鲜地址、搜索与选择代入逻辑
> - v2 补充——QR Wallet / 隐藏钱包（含未创建标准钱包的场景）/ 钱包删除·重置后联想清理 / 账户头像一致性 / Web 端 Swap 自定义接收地址屏蔽 / 名称截断（钱包/账户）
> 关联用例：`2026-04-03_Wallet-最近转账.md`（最近 Tab）、地址簿用例（地址簿 Tab）

---

## 前置条件与测试数据

1. 已导入 HD（已备份）/ HW / 私钥账户 / Keyless Wallet / QR Wallet，各含至少一个账户
2. 已创建 HD（未备份）/ 观察账户 / 外部账户（用于验证隐藏规则）
3. 已创建 **隐藏钱包**（基于已备份主钱包）和 **未创建标准钱包的隐藏钱包**（仅有隐藏钱包，无主钱包账户）
4. 存在 **已删除的隐藏钱包**、**已重置的 HW**（用于验证移除后分组/账户均不再展示）
5. BTC / LTC 账户已创建全部派生路径地址
6. BTC 已开启「新鲜地址」，且至少有一个已使用过的旧地址
7. 各钱包/账户在创建时已由系统生成头像（颜色 + 图案/emoji，不可修改），用于验证跨入口头像一致性

## 1. 账户列表展示与过滤规则

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 可选账户类型展示：<br>HD（已备份）/ HW / 私钥账户 / Keyless Wallet / QR Wallet / 隐藏钱包 / 未创建标准钱包的隐藏钱包 | 进入发送页 → 点击「选择地址」→ 切换到「账户」Tab | 1. 每种钱包**独立分组**展示，账户名称和地址可见，支持点击选择<br>2. 条目数与该钱包内账户数一致<br>3. 未创建标准钱包的隐藏钱包**仍必须显示并可选择**（不能因无主钱包而隐藏整个分组） |
| ❗️❗️P0❗️❗️ | 不展示：<br>观察账户 / 外部账户 / HD（未备份）/ 已删除的隐藏钱包 / 已重置的 HW | 同上 | 以上账户/钱包**均不显示**在「账户」Tab（观察账户与外部账户无转账权限；HD（未备份）需先完成备份；已删除的隐藏钱包与已重置的 HW 已从本地移除，其分组与账户应一并清除） |
| P1 | 不同链下地址格式 | 分别在 EVM / BTC / Cosmos / Solana 网络下打开「账户」Tab | 每个账户展示的地址格式与当前链一致（EVM 为 0x 开头、BTC 为 1/3/bc1q/bc1p 开头（与当前选中派生路径一致）、Cosmos 为 bech32 等） |
| P1 | 当前发送账户自身 | 从账户 A 发起转账 → 打开「账户」Tab | 账户 A 自身也在列表中（允许自转） |

## 3. BTC / LTC 派生路径切换

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | BTC / LTC 路径切换按钮展示 | BTC / LTC 网络下打开「账户」Tab（已创建多个路径地址） | 钱包分组标题旁显示路径切换下拉按钮（如 "Nested SegWit ▽"） |

| ❗️❗️P0❗️❗️ | LTC 路径切换<br>（Nested SegWit / Native SegWit / Legacy） | 点击路径下拉 → 选择目标路径 | 账户列表与下拉文案均按所选路径刷新，地址格式对应正确 |
| P1 | 仅创建单路径时无切换按钮 | BTC / LTC 网络下仅创建了一个路径 | 不显示路径切换下拉按钮 |
| P1 | 非 BTC/LTC 链无快捷路径切换<br>（EVM / Solana / Cosmos 等） | 对应网络下打开「账户」Tab（系统设置中已切换过派生路径） | 1. **不显示**路径切换下拉按钮<br>2. 展示的是系统设置中**当前选定路径**对应的地址 |
| P1 | 路径切换后选择代入 | BTC 切换到 Taproot → 选择某账户 | 代入的地址为 Taproot 格式地址（bc1p 开头），其他路径同理 |

## 4. BTC 新鲜地址

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 开启新鲜地址时列表展示 | BTC 开启新鲜地址 → 打开「账户」Tab | 账户展示的是**当前新鲜地址**（未使用过的地址） |
| ❗️❗️P0❗️❗️ | 选中新鲜地址代入 | 直接点击账户列表中展示新鲜地址的账户 | 代入发送表单的是**新鲜地址** |
| ❗️❗️P0❗️❗️ | 搜索旧地址能匹配 | 在搜索框粘贴该账户**已使用过的旧地址** | 1. 联想匹配到该账户<br>2. 搜索结果中展示的是**旧地址**（与搜索内容一致） |
| ❗️❗️P0❗️❗️ | 选中搜索到的旧地址代入 | 搜索旧地址 → 点击匹配结果 | 代入的是搜索结果展示的**旧地址**，而非新鲜地址 |
| P1 | 关闭新鲜地址时列表展示 | BTC 关闭新鲜地址 → 打开「账户」Tab | 账户展示的是固定的首个接收地址 |

## 5. 搜索与联想

**搜索范围**：钱包名称、账户名称、地址（含新鲜地址和历史旧地址）。搜索不区分大小写。

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 按关键字搜索<br>（账户名 / 钱包名 / 地址） | 在搜索框输入/粘贴关键字（如账户名 `piggy` / 钱包名 `ran` / 地址片段） | 列表实时过滤：账户名匹配 → 仅显示该账户；钱包名匹配 → 显示该钱包下所有账户；地址匹配 → 显示地址命中的账户 |
| P1 | 大小写不敏感搜索 | 输入 `PIGGY` / `Piggy` / `piggy` | 三者搜索结果一致 |
| P1 | 搜索无匹配结果 | 输入不存在的关键字 | 显示空状态，不显示无关结果 |
| P1 | 清空搜索恢复列表 | 1. 输入关键字搜索<br>2. 清空搜索框 | 恢复完整账户列表，派生路径过滤恢复生效 |
| P1 | BTC/LTC 搜索跨路径匹配 | BTC 当前选中 Native SegWit → 搜索一个 Legacy 路径的地址 | 搜索结果**跨所有派生路径**，能找到 Legacy 地址 |
| P1 | 搜索结果 Tab 计数 | 输入关键字，「账户」Tab 有匹配结果 | Tab 标签显示匹配数量（如 "账户 (3)"） |
| P2 | Tab 自动切换 | 在「最近」Tab 搜索关键字 → 最近 Tab 无匹配但账户 Tab 有匹配 | 自动切换到「账户」Tab |

## 6. 地址输入框联想（跨 Tab）

> 地址输入框的联想提示是跨「最近」「账户」「地址簿」三个数据源的全局行为。

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 输入我的账户地址触发联想 | 在地址输入框粘贴一个属于「我的账户」的地址 | 下方联想提示该地址属于我的账户，显示账户名称 |
| ❗️❗️P0❗️❗️ | BTC 旧地址联想标签正确 | BTC 开启新鲜地址 → 粘贴某账户已使用过的**旧地址** | 联想标签**正确指向该账户名**（而非其他账户或空白） |
| ❗️❗️P0❗️❗️ | 钱包删除后联想清理<br>（HD / HW / 隐藏钱包 / QR Wallet） | 1. 记录该钱包下某账户地址<br>2. 钱包选择器中删除该钱包<br>3. 回到发送页粘贴该地址 | 1. **不再**显示"我的账户"标签<br>2. **不再**显示原账户名<br>3. 按外部地址处理 |
| ❗️❗️P0❗️❗️ | 钱包重置后联想清理 | 1. 重置前记录某账户地址<br>2. 执行钱包重置（清除所有钱包）<br>3. 导入新钱包后粘贴旧地址 | **不再**显示"我的账户"标签（联想数据已随重置清除） |
| P1 | 输入的地址同时存在于多个来源 | 粘贴的地址同时在「最近转账」和「我的账户」中 | 联想提示同时标注来源（如既是最近转账又是我的账户） |
| P1 | 输入非本人账户地址 | 粘贴一个外部地址（不在三个来源中） | 不展示联想提示，直接进入金额输入流程 |

## 8. BTC/LTC 默认 Tab 与交互细节

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| P1 | 默认选中 Tab<br>（BTC/LTC → 账户 Tab；其他链 → 最近 Tab） | 对应网络下打开「选择地址」弹窗 | BTC/LTC 默认选中「账户」Tab；EVM / Solana / Cosmos 等默认选中「最近」Tab |
| P1 | Lightning Network 不支持账户与地址簿选择 | Lightning 网络下打开「选择地址」弹窗 | 仅显示「最近」Tab；**「账户」和「地址簿」Tab 不显示**，不支持从我的账户或地址簿选择收款地址 |
| P2 | 名称过长截断<br>（钱包名 / 账户名） | 钱包名或账户名超长时查看列表 | 对应名称**省略号截断**（如 `MyVeryLongWallet...` / `MyVeryLongAccount...`），不挤压头像、地址与路径切换按钮，保持可视区域内完整布局 |
| P2 | 账户名 + 地址同行展示 | 账户名过长 + 地址需完整展示 | 账户名截断优先，地址保持完整可见（或按设计规则换行展示） |

## 9. 钱包账户头像一致性

> 头像由系统在创建钱包/账户时生成，**创建后不支持修改**。本章仅验证跨入口展示的视觉一致性。

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| P2 | 头像与钱包选择器一致<br>（钱包分组头像 / 账户条目头像） | 打开「账户」Tab，分别对比钱包分组标题与账户条目的头像 | 分组标题头像与**钱包选择器**中对应钱包头像完全一致；账户条目头像与**钱包选择器账户切换面板**中对应账户头像完全一致（颜色 / 图案 / emoji 全部一致） |
| P2 | 同一账户跨入口头像一致 | 对比「账户」Tab、「最近」Tab、账户详情、钱包选择器四处展示的同一账户 | 四处展示的头像**完全一致**（同色、同图案/emoji） |
| P2 | 同助记词账户头像一致 | 同助记词导入的 HD / HW / QR 钱包中，对比相同 index 账户的头像 | 同助记词下相同 index 的账户头像保持一致（遵循 `account-rules.md` §1） |

## 10. Web 端 Swap 自定义接收地址（特殊屏蔽规则）

> **仅 Web 端** Swap 功能的「自定义接收地址」场景：**屏蔽「最近」「账户」「地址簿」三个 Tab**，仅允许手动输入地址。Web 端无常规 Send 功能，其他端（Desktop / Extension / iOS / Android）无此屏蔽限制，均不在本章覆盖范围。

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | Web 端 Swap 自定义接收地址屏蔽全部 Tab | Web 端 Swap 页面 → 开启"自定义接收地址"开关 → 点击地址选择 | 1. **不显示**「最近」Tab<br>2. **不显示**「账户」Tab<br>3. **不显示**「地址簿」Tab<br>4. 仅保留手动输入入口 |
| ❗️❗️P0❗️❗️ | Web 端 Swap 可手动输入地址 | 同上场景 | 地址输入框正常可用，可手动粘贴外部地址，地址格式校验正常 |
