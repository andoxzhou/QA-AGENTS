# Wallet - 最近转账

> 需求文档：`docs/qa/requirements/Wallet-最近转账.md`
> 规则文档：`docs/qa/rules/wallet-rules.md` - 第 5.0 章
> 测试端：iOS / Android / Desktop / Extension
> 变更说明：最近转账由“本地记录”升级为“链上/API 优先 + 多级回退 + EVM 聚合”

---

## 1. 数据来源与基础展示

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 1. 当前账户存在 `transfer-recipient` API 返回数据<br>2. 进入发送页收款人选择面板 | 打开发送页，切换到「最近转账」Tab | 1. 显示最近转账列表<br>2. 列表内容与 API 返回地址一致<br>3. 列表按最近转账时间倒序 |
| ❗️❗️P0❗️❗️ | 1. 当前账户 API 返回为空<br>2. 当前网络为非 EVM 链<br>3. 当前链历史存在有效转出记录 | 打开「最近转账」Tab | 1. 显示从当前链历史提取的最近转账地址<br>2. 列表按最近转账时间倒序 |
| ❗️❗️P0❗️❗️ | 1. 当前账户 API 不支持或调用失败<br>2. 当前网络为 EVM 链<br>3. 本地链历史存在有效转出记录 | 打开「最近转账」Tab | 1. 显示从 EVM 链历史提取的最近转账地址<br>2. 列表按最近转账时间倒序 |
| P1 | 1. API、链历史均无结果<br>2. stored recipients 有数据 | 打开「最近转账」Tab | 1. 显示 stored recipients 中的地址<br>2. 列表按更新时间倒序 |
| P1 | 1. API、链历史、stored recipients 均无结果 | 打开「最近转账」Tab | 1. 显示空状态<br>2. 不显示最近转账列表项 |
| P1 | 最近转账列表存在超过 20 条候选记录 | 打开「最近转账」Tab | 1. 列表数量 = 20<br>2. 仅包含最近 20 条记录 |

## 2. EVM 跨链聚合与链信息展示

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 1. 同一 EVM 地址在不同 EVM 链均存在最近转账记录<br>2. 最近一次转账发生在其中一条链 | 打开 EVM 网络下的「最近转账」Tab | 1. 同一地址仅显示 1 条记录<br>2. 该记录排序按最新地址时间生效 |
| ❗️❗️P0❗️❗️ | 同一 EVM 地址分别以 checksum 地址和 lowercase 地址写入历史/存储 | 打开 EVM 网络下的「最近转账」Tab | 1. 列表中该地址仅显示 1 条记录<br>2. 不出现 checksum 大小写导致的重复项 |
| P1 | EVM 聚合结果中每条记录带有实际发生转账的 `networkId` | 打开 EVM 网络下的「最近转账」Tab | 1. 列表项显示对应链名称徽标/文案（如实现支持）<br>2. 链名称与最近一次转账所在网络一致 |
| P1 | 非 EVM 链存在最近转账记录 | 打开非 EVM 网络下的「最近转账」Tab | 1. 仅显示当前链的最近转账结果<br>2. 不混入其他链地址 |

## 3. 搜索与选择带入

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 最近转账列表已展示多条记录 | 点击任意最近转账地址 | 1. 自动带入地址输入框<br>2. 最近转账列表隐藏<br>3. 进入金额输入流程 |
| ❗️❗️P0❗️❗️ | 最近转账记录包含 memo / note / destinationTag | 点击该条最近转账地址 | 1. 地址带入正确<br>2. memo/tag 带入正确 |
| P1 | 最近转账列表包含钱包内地址、地址簿地址、普通外部地址 | 查看列表文案 | 1. 钱包内地址显示钱包账户名称<br>2. 地址簿地址显示地址簿名称<br>3. 普通外部地址显示原始地址 |
| P1 | 最近转账列表已展示 | 搜索钱包名 / 地址簿名 / 地址关键字 | 1. 列表实时过滤<br>2. 仅显示匹配项 |
| P1 | 最近转账记录中存在 memo/tag 信息 | 搜索 memo/tag 关键字 | 1. 列表显示匹配该 memo/tag 的记录（如实现支持）<br>2. 不匹配的记录不显示 |
| P2 | 最近转账列表已展示 | 输入大小写混合的地址关键字搜索 | 1. 搜索结果大小写不敏感<br>2. 匹配结果正确 |
| P2 | 最近转账列表已展示 | 搜索不存在的关键字 | 1. 列表显示空状态<br>2. 不显示无关结果 |

## 4. 记录更新、删除与账户切换

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 当前地址不在最近转账列表中 | 完成一笔新的有效转账后返回发送页，打开「最近转账」Tab | 1. 新地址出现在最近转账列表中<br>2. 新地址排序位于更靠前位置 |
| ❗️❗️P0❗️❗️ | 当前账户存在最近转账记录 | 删除其中一条最近转账记录 | 1. 当前列表中该记录被移除<br>2. 不再显示已删除项 |
| ❗️❗️P0❗️❗️ | 账户 A 与账户 B 拥有不同最近转账数据 | 从账户 A 切换到账户 B，打开「最近转账」Tab | 1. 列表刷新为账户 B 的记录<br>2. 不显示账户 A 独有记录 |
| P1 | 当前账户 API 有最近转账数据，本地缓存为空 | 重装 / 重置后恢复同一账户，打开「最近转账」Tab | 1. 仍可显示接口返回的最近转账记录<br>2. 不因本地缓存缺失而直接显示空态 |
| P1 | 当前账户所有数据源均无最近转账数据 | 重装 / 重置后恢复同一账户，打开「最近转账」Tab | 1. 显示空状态<br>2. 不显示历史残留记录 |

## 5. 过滤规则

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 历史中存在 `Failed` 状态转账 | 打开「最近转账」Tab | `Failed` 交易对应的收款地址不显示在最近转账列表 |
| ❗️❗️P0❗️❗️ | 历史中存在 `Dropped` 状态转账 | 打开「最近转账」Tab | `Dropped` 交易对应的收款地址不显示在最近转账列表 |
| ❗️❗️P0❗️❗️ | 历史中存在 scam tx | 打开「最近转账」Tab | scam tx 对应的收款地址不显示在最近转账列表 |
| ❗️❗️P0❗️❗️ | 历史中存在 `functionCall` 合约交互记录 | 打开「最近转账」Tab | 合约交互对应地址不显示在最近转账列表 |
| ❗️❗️P0❗️❗️ | 历史中存在 0 金额转账 | 打开「最近转账」Tab | 0 金额转账对应地址不显示在最近转账列表 |
| ❗️❗️P0❗️❗️ | 历史中存在向当前账户自身地址的转账 | 打开「最近转账」Tab | 当前账户自身地址不显示在最近转账列表 |
| P1 | 查询地址资料结果识别为合约地址 | 打开「最近转账」Tab | 合约地址不显示在最近转账列表 |
| P1 | 最近转账记录的 memo 以 `Call:` 开头 | 打开「最近转账」Tab | 该记录不显示在最近转账列表 |

## 6. Memo / Tag 与链差异

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | Cosmos 链最近转账包含 memo | 打开「最近转账」Tab 并选择该记录 | 1. 列表项展示 memo 信息（如实现支持）<br>2. 选择后 memo 带入正确 |
| ❗️❗️P0❗️❗️ | XRP / Stellar 类链最近转账包含 destinationTag / note | 打开「最近转账」Tab 并选择该记录 | 1. 列表项展示 tag/note 信息（如实现支持）<br>2. 选择后 tag/note 带入正确 |
| P1 | 最近转账记录不包含 memo/tag | 打开「最近转账」Tab 并选择该记录 | 1. 列表项不显示空 memo 占位<br>2. 选择后 memo/tag 输入区域保持为空 |

## 7. 异常与回退策略

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | `transfer-recipient` API 返回 `supported=true` 且有数据 | 打开「最近转账」Tab | 1. 展示 API 结果<br>2. 不继续回退到 stored recipients 覆盖当前结果 |
| ❗️❗️P0❗️❗️ | `transfer-recipient` API 返回 `supported=false`，当前网络为 EVM，链历史有数据 | 打开「最近转账」Tab | 1. 展示 EVM 链历史提取结果<br>2. 不直接显示空态 |
| P1 | `transfer-recipient` API 调用异常，当前网络为 EVM，链历史有数据 | 打开「最近转账」Tab | 1. 展示 EVM 链历史提取结果<br>2. 页面不报错、不白屏 |
| P1 | `transfer-recipient` API 返回空，stored recipients 有数据 | 打开「最近转账」Tab | 1. 展示 stored recipients 结果<br>2. 页面不显示空态 |
| P1 | `transfer-recipient` API 返回空，当前网络为非 EVM，链历史有数据 | 打开「最近转账」Tab | 1. 展示当前链历史提取结果<br>2. 页面不显示空态 |
| P2 | API、链历史、stored recipients 均为空，且搜索关键字非空 | 打开「最近转账」Tab 并输入搜索关键字 | 1. 显示空状态<br>2. 不显示错误数据 |
