# 发送 - 最近转账

> 需求文档：`docs/qa/requirements/Wallet-最近转账.md`
> 规则文档：`docs/qa/rules/wallet-rules.md` - 第 5.0 章
> 测试端：iOS / Android / Desktop / Extension
> 变更说明：最近转账数据通过 `transfer-recipient` API 接口获取，服务端已做过滤（不含 Swap 中心化渠道等记录），客户端按接口结果展示；API 无数据时回退到本地历史记录
> **备注 / Memo / Tag 行为（2026-04-08）**：最近转账和地址簿都可能携带 memo / tag 等数据，点击选择时自动带入发送表单。修改地址后 memo / tag 保留不清空。
> 展示上限：最近转账列表**最多显示 20 条**（经 EVM 去重、过滤与按时间倒序排序后，仅展示最近的 20 条）
> **本地数据账户隔离（2026-04-13）**：本地历史记录按账户进行隔离存储，切换账户后仅显示当前账户的本地记录，不再跨账户共用。

> **接口自动化覆盖**（见 `最近转账-Apifox-TestCases.json`）：条数上限 20、时间倒序、EVM 跨链去重、checksum 大小写去重、过滤规则（Scam / Swap / Failed / Dropped / 合约交互）、supported 字段响应结构。以上数据正确性由接口自动化保障，人工只需验证前端 UI 展示与接口数据一致。
> **Swap 中心化渠道过滤**：当前服务端已过滤 Changelly、ChangeHero、Exodus、Houdini 四个中心化渠道的订单数据，接口不会返回这些渠道产生的转账记录。如后续新增中心化渠道，需同步将其加入服务端过滤名单并更新接口自动化断言。

## 1. 数据来源与回退

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | API 有数据 | 打开「最近转账」Tab | 列表展示与接口返回一致（数据正确性已由接口自动化保障） |
| ❗️❗️P0❗️❗️ | API 返回为空或不支持，本地历史记录有数据 | 打开「最近转账」Tab | 回退展示本地历史记录，不直接显示空态 |
| P1 | API 调用异常 | 打开「最近转账」Tab | 页面不报错、不白屏，有本地数据则展示 |
| P1 | API 和本地历史记录均无数据 | 打开「最近转账」Tab | 显示空状态 |

## 2. EVM 跨链聚合与链信息展示

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| P1 | 同一 EVM 地址跨链记录 | 打开 EVM 网络下的「最近转账」Tab | 确认前端列表无重复项（去重逻辑已由接口自动化保障） |
| P1 | EVM 聚合结果中每条记录带有 `networkId` | 打开 EVM 网络下的「最近转账」Tab | 列表项显示对应链名称徽标/文案，与最近一次转账所在网络一致 |
| P1 | 非 EVM 链存在最近转账记录 | 打开非 EVM 网络下的「最近转账」Tab | 仅显示当前链结果，不混入其他链地址 |

## 3. 搜索与选择带入

**搜索范围**：仅收款地址和钱包账户名称，memo / tag 不参与搜索。

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 最近转账列表已展示多条记录 | 点击任意最近转账地址 | 1. 自动带入地址输入框<br>2. 最近转账列表隐藏<br>3. 进入金额输入流程 |
| ❗️❗️P0❗️❗️ | 最近转账记录携带 memo / tag | 点击选择该条 | 1. 收款地址带入正确<br>2. memo / tag 自动带入发送表单 |
| P1 | 最近转账列表包含钱包内地址、地址簿地址、普通外部地址 | 查看列表文案 | 1. 钱包内地址显示钱包账户名称<br>2. 地址簿地址显示地址簿名称<br>3. 普通外部地址显示原始地址 |
| P1 | 最近转账列表已展示 | 输入与**地址**或**钱包账户名称**匹配的关键字 | 1. 列表实时过滤<br>2. 仅显示匹配项 |
| P1 | 同上 | 输入 memo / tag 相关关键字 | 不命中、不过滤列表 |
| P2 | 最近转账列表已展示 | 输入大小写混合的地址关键字搜索 | 1. 搜索结果大小写不敏感<br>2. 匹配结果正确 |
| P2 | 最近转账列表已展示 | 搜索不存在的关键字 | 1. 列表显示空状态<br>2. 不显示无关结果 |

## 4. 记录更新、重置与账户切换

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 完成一笔新转账 | 返回发送页，打开「最近转账」Tab | 1. 新地址出现在列表中<br>2. 排序位于最前（列表约 15 秒刷新一次） |
| ❗️❗️P0❗️❗️ | 给同一地址多次转账 | 打开「最近转账」Tab | 同一地址只显示一条，时间更新为最近一次 |
| ❗️❗️P0❗️❗️ | 账户 A 切换到账户 B | 切换后打开「最近转账」Tab | 1. API 数据刷新为账户 B 的记录<br>2. 本地历史记录按账户隔离，仅显示账户 B 的本地记录 |
| P1 | 重装 / 重置后恢复账户，API 有数据 | 打开「最近转账」Tab | 仍可显示接口返回的记录，不因本地缓存缺失而显示空态 |
| P1 | 重装 / 重置后恢复账户，API 无数据 | 打开「最近转账」Tab | 显示空状态，不显示历史残留记录 |

## 5. Memo / Tag 带入

> 最近转账和地址簿都可能携带 memo / tag 等数据，点击选择时自动带入发送表单。

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | Cosmos 链最近转账记录带有 memo | 选择该记录 | memo 自动带入发送表单 |
| ❗️❗️P0❗️❗️ | XRP / Stellar 最近转账记录带有 tag / note | 选择该记录 | tag / note 自动带入发送表单 |
| P1 | 最近转账记录无 memo / tag | 选择该记录 | 对应输入区为空，不出现误导性占位 |
| P1 | 选择带有 memo 的记录后，手动修改收款地址 | 1. 选择带 memo 的记录<br>2. 修改地址 | memo 保留不被清空 |
