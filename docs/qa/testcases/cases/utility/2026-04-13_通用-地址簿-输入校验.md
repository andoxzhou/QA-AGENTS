# 通用 - 地址簿 - 输入校验

> 规则文档：`docs/qa/rules/utility-rules.md` - 第 5 章
> 测试端：iOS / Android / Extension / Desktop / Web
> 前置依赖：「地址簿 - 添加地址」用例中添加的数据未删除（地址簿已存在 `BTC-taproot`、`38Xegnipu2RhZouctnGnwmDRk2bLXfDHf4` 等 77 条记录）

---

## 前置条件

1. APP 已初始化，已创建/导入钱包
2. 「地址簿 - 添加地址」用例已执行完毕，数据未删除
3. 已进入添加地址页面

---

## 1. 名称校验

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 名称栏为空 | 输入 `123456789012345678901234`（24 字符） | 不显示报错提示 |
| ❗️❗️P0❗️❗️ | 名称栏为空 | 输入 `1234567890123456789012345`（25 字符） | 显示最大输入 24 字符提示 |
| ❗️❗️P0❗️❗️ | 名称栏已有内容 | 删除全部内容，使名称栏为空 | 显示名称不能为空提示 |
| ❗️❗️P0❗️❗️ | 名称栏为空 | 输入纯空格 `"    "` | 显示名称不能为空提示 |
| ❗️❗️P0❗️❗️ | 地址簿已存在名称 `BTC-taproot` | 输入 `BTC-taproot` | 显示名称已存在提示 |
| ❗️❗️P0❗️❗️ | 名称栏为空 | 输入 `E 简體サ한？বাং@हिल2 êйไїế※★` | 不显示报错提示 |

## 3. 保存按钮状态

> 仅在名称和地址都已输入且无报错时，保存按钮可点击。

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 名称和地址都已输入且无报错 | 查看保存按钮 | 保存按钮可点击 |
| ❗️❗️P0❗️❗️ | 名称未输入，地址已输入且无报错 | 查看保存按钮 | 保存按钮不可点击 |
| ❗️❗️P0❗️❗️ | 名称已输入且无报错，地址未输入 | 查看保存按钮 | 保存按钮不可点击 |
| ❗️❗️P0❗️❗️ | 名称或地址任一项显示报错 | 查看保存按钮 | 保存按钮不可点击 |

## 4. Memo / Tag 校验

> 不支持 memo 的网络（BTC/EVM/LTC 等）不显示该字段。所有支持 memo 的网络留空均可保存（可选字段）。

| 优先级 | 网络 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | Bitcoin / Ethereum | 字段显示 | 选择网络，观察表单 | 不显示 Memo/Tag 输入框 |
| ❗️❗️P0❗️❗️ | XRP Ledger | 字段显示 | 选择 XRP Ledger | 显示 Tag 输入框 |
| ❗️❗️P0❗️❗️ | XRP Ledger | 正常值 | Tag 输入 `12345` | 可保存 |
| ❗️❗️P0❗️❗️ | XRP Ledger | 边界值 | Tag 输入 `1234567890`（10 字符） | 可保存 |
| ❗️❗️P0❗️❗️ | XRP Ledger | 超限 | Tag 输入 `12345678901`（11 字符） | 提示「最大为 10 个字符」 |
| ❗️❗️P0❗️❗️ | XRP Ledger | 格式错误 | Tag 输入 `abcdef` / `-123` / `12.34` / `!@#$` | 提示「必须是正整数」，不可保存 |
| P1 | XRP Ledger | 值为 0 | Tag 输入 `0` | 可保存 |
| ❗️❗️P0❗️❗️ | Stellar | 字段显示 + 正常值 | 选择 Stellar，Memo 输入 `hello` | 显示 Memo 输入框，可保存 |
| P1 | Stellar | 边界值 / 超限 | 输入 28 字节 / 29 字节 | 28 字节可保存；29 字节截断或提示超限 |
| P1 | Stellar | 多字节字符 | 输入 `测试备注一二三四五六`（30 字节 UTF-8） | 截断或提示超限 |
| ❗️❗️P0❗️❗️ | TON | 字段显示 + 正常值 | 选择 TON，Memo 输入 `payment-001` | 显示 Memo 输入框，可保存 |
| P1 | TON | 边界值 / 超限 | 输入 123 字符 / 124 字符 | 123 可保存；124 截断或提示超限 |
| ❗️❗️P0❗️❗️ | Cosmos 系 / Algorand | 字段显示 + 正常值 | 选择 Cosmos（或 Algorand），输入 `test-memo` | 显示 Memo（或 Note）输入框，可保存 |
| P1 | Cosmos 系 / Algorand | 边界值 / 超限 | 输入 512 字符 / 513 字符 | 512 可保存；513 截断或提示超限 |

## 5. 粘贴与扫描

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| P1 | 1. 地址栏已有内容<br>2. 剪贴板中有字符 | 点击地址栏「粘贴」按钮 | 1. 地址栏已有内容被清空<br>2. 剪贴板中的字符填入地址栏 |
| P1 | 地址栏为空或有内容 | 点击地址栏「扫描」按钮 | 1. 唤起通用扫描功能<br>2. 扫码获取的字符填入地址栏 |
