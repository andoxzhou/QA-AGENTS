## 2. 发送地址输入页 - 输入与校验（通用）

> 代码来源：`packages/kit/src/views/BulkSend/pages/BulkSendAddressesInput/components/AddressesInput/useMultiLineAddressValidation.ts`
> 适用于一对多（接收方输入）/ 多对一（发送方输入）/ 多对多（发送方 + 接收方输入）。
> 每行一个地址，支持 `address` 或 `address,amount` 两种格式；空行 / 仅空格行自动过滤。

### 2.1 输入来源与解析

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 手输 / 粘贴多行 | 文本框直接输入或粘贴多行地址 | 按行号展示；每行 `trim` 后独立校验；空行与仅空格行自动过滤 |
| ❗️❗️P0❗️❗️ | 地址簿多选 | 点击地址簿图标 → 多选地址 → 确认 | 选中地址按行写入输入框 |
| ❗️❗️P0❗️❗️ | CSV 导入 | 点击上传 → 选择 CSV | 解析后按行写入；格式错误显示 CSV 解析错误（含行号 / 原因）可重新上传 |
| P1 | 行数上限（如存在 `maxLines`） | 粘贴超过 `maxLines` 行的地址 | 顶部错误条显示「超出最大条数，当前 N 条，上限 M 条」，下一步按钮不可点 |

### 2.2 格式模式识别（address-only / address+amount）

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 仅地址格式 | 输入 `addr1` / `addr2` → 下一步 | 识别为 address-only 模式；金额设置页显示「指定金额」/「范围金额」 |
| ❗️❗️P0❗️❗️ | 地址+金额格式 | 输入 `addr1,100` / `addr2,200` → 下一步 | 识别为 address+amount 模式；金额设置页多出「自定义金额」标签页 |
| ❗️❗️P0❗️❗️ | 格式混用（自动模式） | 首行 `addr1`，次行 `addr2,100` | 提示次行需与首行一致「需仅地址」或「需地址+金额」；错误行高亮，不可进入下一步 |
| ❗️❗️P0❗️❗️ | 强制仅地址（多对一接收方等） | 输入带 `,` 的行 | 提示「仅允许地址」；不可进入下一步 |
| ❗️❗️P0❗️❗️ | 逗号分段不等于 2 | 输入 `addr1,100,备注` 或 `addr1,` | 提示「格式无效」；不可进入下一步 |

### 2.3 地址合法性

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 无效地址 | 输入 `0x123`（长度不符）/ 校验和错误地址 / 与当前网络不匹配的地址 | 错误信息带网络名：「该地址不是合法的 `{network}` 地址」；错误行红色高亮 |
| ❗️❗️P0❗️❗️ | 校验期间切换网络 | 输入合法 ETH 地址 → 未等校验完成切换到 BTC | 旧校验结果被作废（按新网络重校）；错误信息中的网络名同步为新网络 |
| P1 | 地址首尾含空格 | 输入 ` addr ` | 自动 `trim` 后校验；合法地址通过 |
| P1 | 错误显示条数上限 | 输入 10 条全部错误的地址 | 最多展示 5 条详细错误；末尾追加「还有 N 条错误」摘要 |

### 2.4 金额合法性（仅 address+amount 模式）

> 校验走 `validateTokenAmount(allowZero=false)`；最小转账金额取自 `vaultSettings.nativeMinTransferAmount` 或 `minTransferAmount`。

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 金额为空 / 非数字 | `addr,` / `addr,abc` | 提示「金额无效」；错误行高亮，不可进入下一步 |
| ❗️❗️P0❗️❗️ | 金额为 0 / 负数 | `addr,0` / `addr,-1` | 提示「金额不能为零或负数」；错误行高亮，不可进入下一步 |
| ❗️❗️P0❗️❗️ | 金额小于链最小转账金额 | 输入低于 `minTransferAmount` 的值（各链不同） | 提示「最小转账金额为 `{amount} {symbol}`」；不可进入下一步 |
| ❗️❗️P0❗️❗️ | 金额小数位超过代币 decimals | 输入 `0.123456789` 到 decimals=6 的代币 | 提示「最多支持 `{decimals}` 位小数」；不可进入下一步 |
| ❗️❗️P0❗️❗️ | 未选择代币时输入金额 | 清空代币选择 → 校验 | 提示「未选择代币」；不可进入下一步 |

### 2.5 重复地址检测（模式差异）

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 一对多 / 多对多 - 重复接收 / 发送方 | 输入相同地址 ≥2 次 → 下一步 | 黄色警告标记重复行并指出「第 N 行重复」；弹窗提示重复数，用户确认后可继续 |
| ❗️❗️P0❗️❗️ | 多对一 - 重复发送方 | 输入相同发送方地址 ≥2 次 → 下一步 | 红色错误，错误行高亮；不可进入下一步 |
| P1 | 重复地址归一化 | 输入 `ABC`、`abc`（同一 EVM 地址大小写变体） | 按 `normalizedAddress` 判定重复 |

### 2.6 账户归属 / Allowlist（`checkAllowlist` 开关开启时）

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 发送方不在当前钱包 | 多对一 / 多对多输入一个不属于本钱包的地址 | 提示「未找到账户」；不可进入下一步 |
| ❗️❗️P0❗️❗️ | 接收方不在 Allowlist（启用白名单时） | 一对多接收方输入既非我账户也非地址簿的地址 | 提示「该地址不在白名单」；不可进入下一步 |
| P1 | 接收方在地址簿 | 接收方使用地址簿中保存的地址 | 允许通过；无 Allowlist 警告 |

## 3. 金额与间隔输入页 - 输入与校验（通用）

> 代码来源：`packages/kit/src/views/BulkSend/utils.ts` — `validateRangeInput` / `validateIntervalSettings` / `checkSenderInsufficientBalance` / `filterNumericInput` / `filterIntegerInput`
> 适用于三种模式的金额设置；**间隔部分仅多对一 / 多对多**（一对多跳过间隔页）。

### 3.1 输入过滤（所有金额输入框）

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 字母 / 特殊字符 | 输入 `abc!@#` 到金额框 | 自动过滤，仅保留数字与小数点 |
| ❗️❗️P0❗️❗️ | 多个小数点 | 输入 `1.2.3` | 仅保留第一个小数点，显示 `1.23` |
| ❗️❗️P0❗️❗️ | 小数点开头 | 输入 `.5` | 自动规范为 `0.5` |
| ❗️❗️P0❗️❗️ | 间隔框输入非整数 | 输入 `5.5` 或 `abc` 到间隔 | 自动过滤，仅保留整数 |

### 3.2 指定金额（Specified 标签页）

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 空值 / 零值 | 金额框为空 / 输入 0 | 下一步按钮不可点；显示对应错误 |
| ❗️❗️P0❗️❗️ | 一对多：金额 > 余额 | 输入超过可用余额的值 | 提示「余额不足」；不可进入下一步 |
| ❗️❗️P0❗️❗️ | 一对多：金额 < 最小转账金额 | 输入低于链最小转账金额的值 | 提示「最小转账金额为 `{amount} {symbol}`」 |
| ❗️❗️P0❗️❗️ | 一对多：点击 Max | 点击 Max 按钮 | 进入 Max 模式；按余额平均分配；总额不超过可用余额；Max 模式直接视为有效 |
| ❗️❗️P0❗️❗️ | 多对一 / 多对多：Max 模式 | 点击 Max | 每个发送方按各自余额发送全部；若任一发送方余额未加载成功则按钮不可用 |
| ❗️❗️P0❗️❗️ | 小数位超过 decimals | 输入小数位超过代币 decimals | 提示「最多支持 `{decimals}` 位小数」 |
| ❗️❗️P0❗️❗️ | 总额聚合超余额（多对一/多对多） | 同一发送方被多次使用，累计金额 > 其余额 | 提示「余额不足」（通过 `checkSenderInsufficientBalance` 聚合判定） |

### 3.3 范围金额（Range 标签页）

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | min / max 任一为空 | 只填 min 或只填 max | 下一步不可点（`rangeError` 未通过） |
| ❗️❗️P0❗️❗️ | 余额为 0 | 代币余额 = 0 时输入范围 | 提示「余额不足」 |
| ❗️❗️P0❗️❗️ | min > 余额 | 一对多：`rangeMin` 大于余额 | 提示「余额不足」 |
| ❗️❗️P0❗️❗️ | max > 余额 | 一对多：`rangeMax` 大于余额 | **允许通过**；生成时以 `balance / count` 作为有效 max |
| ❗️❗️P0❗️❗️ | min >= max | 输入 min = 10，max = 10 或 max = 5 | 提示「请输入正确区间」；不可进入下一步 |
| ❗️❗️P0❗️❗️ | min < 最小转账金额 | 输入 min 低于链最小转账金额 | 提示「最小转账金额为 `{amount} {symbol}`」 |
| ❗️❗️P0❗️❗️ | 多对一 / 多对多：min 超过最小发送方余额 | 多个发送方余额不同，取最小余额作约束 | `rangeMin > 最小发送方余额` 时提示「余额不足」 |
| P1 | 生成结果金额分布 | 有效区间执行下一步 → 进入详情页 | 每笔金额在 `[min, max]` 内；总额不超过余额；采用 smart decimals（大值少位数，小值保留有效数字） |

### 3.4 自定义金额（Custom 标签页，仅 address+amount 场景）

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 列表展示 | 进入 Custom | 顶部显示总金额；列表展示每笔 发送方 → 接收方 / 金额 |
| ❗️❗️P0❗️❗️ | 某行金额异常 | 编辑某行为 0 / 负数 / 非法 / 超 decimals | 对应行显示错误；总额停止更新；下一步不可点（`transferInfoErrors` 非空） |
| ❗️❗️P0❗️❗️ | 余额不足（聚合） | 修改导致同一发送方累计超过余额 | 该发送方所有相关行标红提示；下一步不可点 |
| P1 | 删除某行 | 点击行尾删除 | 该行移出列表；总额自动更新；剩余可继续 |

### 3.5 间隔设置（多对一 / 多对多）

> `validateIntervalSettings`：仅 `Specified` 模式校验；上限常量 `BULK_SEND_INTERVAL_MAX_SECONDS = 600`；默认值 min=5 / max=30。

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | 默认值 | 进入间隔页 | 默认 Specified 模式；min=5，max=30；下方提示「请输入 0~600 秒」 |
| ❗️❗️P0❗️❗️ | min 为空 | 清空 min | 提示「请填写最小间隔」；确认不可进入下一步 |
| ❗️❗️P0❗️❗️ | max 为空 | 清空 max | 提示「请填写最大间隔」；确认不可进入下一步 |
| ❗️❗️P0❗️❗️ | max <= 0 | 输入 max = 0 | 提示「最大值必须大于 0」 |
| ❗️❗️P0❗️❗️ | min >= max | 输入 min = 30，max = 30 或 max = 10 | 提示「最大值需大于最小值」 |
| ❗️❗️P0❗️❗️ | min 或 max > 600 | 输入 700 | 提示「请输入合适的区间（0~600 秒）」 |
| ❗️❗️P0❗️❗️ | None 模式 | 切换为「无间隔」 | 隐藏 min / max 输入框；跳过校验；交易连续发送 |

## 4. 交易确认页 - 前置检查与提交（通用）

> 代码来源：`packages/kit/src/views/BulkSend/pages/BulkSendReview/BulkSendReview.tsx`

| 优先级 | 场景 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| ❗️❗️P0❗️❗️ | Confirm 按钮文案 | 进入确认页 | 有授权项时显示「授权并确认」；无授权项时显示「确认」 |
| ❗️❗️P0❗️❗️ | 费率加载中 | 费率状态 = Loading | Confirm 按钮 disabled（避免用过期费率提交） |
| ❗️❗️P0❗️❗️ | 费率加载失败 | 费率状态 = Error | Confirm 按钮 disabled；顶部 Alert 显示错误，提供「重试」按钮 |
| ❗️❗️P0❗️❗️ | SOL 余额不足（含 ATA 租金 + Gas） | SOL 一对多 SPL，SOL 不足 | `insufficientSol=true`；Confirm disabled；卡片显示所需 SOL / ATA 数 / 租金 |
| ❗️❗️P0❗️❗️ | 提交前预检失败（`precheckUnsignedTxs`） | 模拟 nonce 过低 / RPC 拒绝 | 弹出失败原因；停留在 Review 页；Confirm 可重试 |
| ❗️❗️P0❗️❗️ | Fee 异常二次确认 | Gas 预估显著高于链均值（多对多 / 多对一只查第一笔） | 弹出 Fee 过高确认弹窗；用户取消则不提交；确认则跳过剩余 overflow 检查并继续 |
| ❗️❗️P0❗️❗️ | 授权金额编辑 | 点击授权卡片编辑 → 修改金额 / 切换 isUnlimited / 重置 | 变更同步到 approvesInfo；重置返回原始额度；重新预估费率 |
| ❗️❗️P0❗️❗️ | Tron 逐笔确认 | Tron 模式下点击 Confirm | 交易逐笔弹出签名（非一次性批量签名）；一笔失败后续停止 |
