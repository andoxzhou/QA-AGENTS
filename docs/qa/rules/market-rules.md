# Market 模块测试规则文档

> 本文档记录 Market 模块的核心测试规则。
> 生成 Market 模块测试用例时，必须参考本文档。

---

## 1. 模块概述

Market 模块包含以下核心功能：
- 首页主标签：自选、现货、合约
- 热门趋势卡片：美股、Layer 1、表情包等
- 现货列表：代币价格、涨跌幅、市值、流动性、交易额等
- 合约列表：价格、涨跌、交易额、合约持仓量、资金费率
- K 线图表（价格走势、技术指标）
- Token 详情页
- 搜索与热门代币（表格样式）
- 兑换入口（代币详情页发起兑换）
- 限价单功能

## 2. 行情列表规则

### 2.1 数据展示

| 字段 | 规则 |
|-----|------|
| Token 名称 | 显示名称和符号 |
| 价格 | 显示当前价格（**统一 USD 计价**，不跟随用户默认货币） |
| 涨跌幅 | 显示 24h 涨跌幅（百分比） |
| 成交量 | 显示 24h 成交量（USD） |
| 市值 | 显示市值（USD，可选） |

### 2.2 列表功能

| 功能 | 规则 |
|-----|------|
| 排序 | 支持按价格、涨跌幅、成交量排序 |
| 搜索 | 支持按名称、符号、合约地址搜索（详见下方"搜索功能规则"章节） |
| 收藏 | 支持收藏 Token，显示在收藏列表（详见下方"收藏功能规则"章节） |
| 分类 | 主标签：自选、现货、合约；现货支持网络筛选；合约支持二级筛选（加密货币、股票、贵金属等） |

### 2.3 数据更新

| 规则项 | 规则描述 |
|-------|---------|
| 实时更新 | WebSocket 推送价格更新 |
| 刷新频率 | 价格更新节流（防止闪烁） |
| 离线处理 | 断网时显示缓存数据 + 离线提示 |

## 3. 搜索功能规则

| 规则项 | 规则描述 |
|-------|---------|
| 搜索类型 | 支持 Symbol 搜索、合约地址搜索 |
| 搜索排序 | 搜索结果包含主币时，主币排在第一位；其他代币按照流动性从高到低倒序排序；排在第一位的结果显示👍标识（社区认可度） |
| 主币识别 | 主币包括 BTC、ETH、SOL、SUI 等原生链主币 |
| 流动性排序 | 非主币代币按照流动性从高到低倒序排序，流动性高的排在前面 |

### 3.1 搜索结果与热门代币 - 表格样式

| 规则项 | 规则描述 |
|-------|---------|
| 展示样式 | 搜索结果和热门代币列表均采用表格样式 |
| 表格字段 | 收藏、名称、价格、流动性、24h成交额、市值；All 分类下展示数量（前三） |
| 名称列 | 代币图标 + 名称 + 验证标识 + 合约地址；图标右下角显示网络标识；地址可复制，格式 0x1234...abcd |
| 价格列 | 当前价格 + 24h涨跌幅；涨跌幅绿涨红跌 |
| 表头固定 | 表头在 Market Tab 固定，内容区域纵向滚动 |

### 3.2 热门代币（Trending）

| 规则项 | 规则描述 |
|-------|---------|
| 数据源 | 使用 Market 数据源；接口获取前 30 热门代币，按 24h 交易量确定主要网络，每代币仅取主要网络，展示 15 条 |
| 点击交互 | 点击代币行直接跳转 Market 交易页面，跳过代币详情中间页 |

### 3.3 边缘 Case

| 场景 | 规则 |
|-----|------|
| 价格/成交额为空或 0 | 显示「--」或「$0」，不隐藏该行 |
| 图标加载失败 | 显示默认占位图标 |
| 数值过长 | 价格保留有效数字，大数值用 K/M/B/T 缩写 |
| 名称过长 | 截断显示，末尾加省略号 |

## 4. 收藏功能规则

| 规则项 | 规则描述 |
|-------|---------|
| 收藏入口 | 观察列表、分类列表、详情页、搜索列表、钱包首页 |
| 状态同步 | 所有入口的收藏状态保持一致，同步延迟 ≤ 1 秒 |
| 空状态处理 | 自选列表为空时，显示默认推荐的 8 个热门代币（4x2 网格布局），所有代币默认被勾选，支持勾选/取消勾选后批量添加 |
| 防抖处理 | 快速连续点击收藏/取消收藏按钮时，需实现防抖处理 |
| 前置条件 | 收藏功能不需要登录，可忽略云备份相关报错 |

## 5. K 线图表规则

### 5.1 图表功能

| 功能 | 规则 |
|-----|------|
| 时间区间（K 线上方） | 支持 1m / 15m / 1h / 4h / D / W 等；具体默认选中项以产品定义为准（需求曾出现「默认 1m」与「首次加载 15m」两种描述，需对齐口径） |
| 下方时间范围 | 支持 1 日 / 5 日 / 1 月 / 3 月 / 6 月 / 1 年 / All 等；桌面端可展示用户时区，移动端若无时区 UI 则不要求 |
| OHLC | O/H/L/C 定义与周期一致；可与 OKX、TradingView 等参考源在统一周期与时区下对照 |
| K 线类型 | 默认蜡烛图；支持切换其他类型（以产品列表为准） |
| 价格 / 市值 | 支持价格与市值视图切换 |
| 缩放与平移 | 支持缩放与上下、左右平移浏览历史数据 |
| 十字线 | 显示时间、O、H、L、C 等（以端上展示字段为准） |
| 指标 | 支持 VOL、EMA、MACD、RSI、布林带等；支持多指标组合、上限、移除；自定义指标全局生效（跨 Token 保留） |
| 移动端 K 线点击 | 支持点击蜡烛查看详情（时间、交易量、涨跌幅等，以产品为准） |

### 5.2 数据一致性

| 规则项 | 规则描述 |
|-------|---------|
| K 线/列表一致 | K 线收盘价与列表价格一致 |
| 多源策略 | 取中位/优先源/降级 |
| OHLC 数据 | Open/High/Low/Close 数据正确 |

### 5.3 性能要求

| 指标 | 要求 |
|-----|------|
| 加载时间 | K 线数据加载 < 2s |
| 帧率 | 交互帧率 ≥ 30 FPS |
| 内存 | 长时间查看不出现内存泄漏 |

## 7. Swap Pro Mode 规则

> 兑换功能入口在 Token 详情页，由各端实现具体形式。

### 7.1 功能概述

| 规则项 | 规则描述 |
|-------|---------|
| 入口 | Market Token 详情页 |
| 状态机 | 询价→路径→授权→Swap→Pending→Success/Fail |

### 7.2 核心验证

| 验证项 | 规则 |
|-------|------|
| 滑点设置 | 支持自定义滑点 |
| Gas 费用 | 显示预估 Gas 费 |
| 价格影响 | 显示 Price Impact |
| 最小收到量 | 显示 Minimum Received |
| 授权流程 | ERC20 代币需要授权 |

### 7.3 与普通 Swap 差异

| 差异点 | Pro Mode | 普通 Swap |
|-------|---------|----------|
| 入口 | Token 详情页 | Swap 页面 |
| 默认 Token | 当前查看的 Token | 需手动选择 |
| 行情数据 | 使用 Market 行情 | 使用 Swap 报价 |

### 7.4 股票代币支持规则

> 股票代币（on 后置的美股代币，如 AAPLon、MSFTon 等）在 Market 代币详情页的兑换流程中单独路由，以保证兑换汇率与询价速度。

| 规则项 | 规则描述 |
|-------|---------|
| 股票代币列表 | 后端维护，参考 CoinGecko 分类（ondo-tokenized-assets）；股票代币为 on 后置的美股代币 |
| 路由规则 | 股票代币询价走 1inch，不走 OKX DEX |
| 非股票代币 | 继续走 OKX DEX 路由 |
| 不开盘识别 | 识别股票代币不开盘状态，需有明确提示或限制 |
| 最低金额 | 股票代币交易最低 20U（BN、OKX 均采用） |

## 8. 限价单规则

> 限价单功能通过 CowSwap 渠道实现，支持网络：Ethereum、Arbitrum、Base

### 8.1 基本规则

| 规则项 | 规则描述 |
|-------|---------|
| FromToken | 不支持直接出售 ETH，需换成 WETH |
| ETH → WETH | 自动转换逻辑验证 |

### 8.2 订单有效期

| 选项 | 说明 |
|-----|------|
| 5 分钟 | 短期订单 |
| 30 分钟 | 短期订单 |
| 1 小时 | 中期订单 |
| 1 天 | 中期订单 |
| 3 天 | 长期订单 |
| 7 天 | 长期订单 |
| 1 个月 | 长期订单 |

### 8.3 部分成交

| 规则项 | 规则描述 |
|-------|---------|
| 启用 | 允许部分成交 |
| 禁用 | 仅完全成交 |
| 处理逻辑 | 需验证两种模式的不同处理 |

### 8.4 特殊授权规则

| 场景 | 规则 |
|-----|------|
| USDT → ETH | Ethereum 上需要二次授权 |
| USDT → 其他 | 不需要二次授权 |

## 9. Confirm Page V2 规则（Market Swap Pro Mode 确认页）

### 9.1 入口与生命周期

| 规则项 | 规则描述 |
|-------|---------|
| 入口 | Market 详情页点击主操作按钮后拉起 Review Order 确认页，不直接发起链上交易 |
| Wrap/Unwrap | 同样拉起确认页，步骤文案与 Wrap 场景匹配 |
| 防重复触发 | Action loading / allowance checking / approve matching pending 期间按钮不可重复触发 |
| 容器兼容 | tab 页面与 overlay / modal 页面两种容器下均可正常打开关闭 |
| 状态重置 | 关闭后重新打开展示当前最新数据，不残留旧状态 |

### 9.2 步骤编排

| 场景 | 步骤展示 |
|-----|---------|
| 无需授权普通 Swap | 仅 Confirm / Sign / Submit，不额外出现 Approve |
| 需授权普通 Swap | Approve + Swap / Sign 组合步骤 |
| Wrap / Unwrap | 仅 Wrap 步骤，不混入 Approve / Swap |
| 签名链路 | Sign 或 Sign and Submit |
| 步骤标题 | Confirm Swap / Sign / Sign and Submit / Approve and Swap / Approve and Sign 与链路一致 |

### 9.3 授权判定与费用刷新

| 规则项 | 规则描述 |
|-------|---------|
| allowance 判定 | 已满足不展示 Approve；不足展示 Approve |
| spender 变化 | 以最新有效 spender / allowanceTarget 重新判定，不沿用旧状态 |
| 费用刷新 | Approve 完成后 gas / network fee 刷新为最新值 |
| pending 状态 | approve pending amount 命中时保持 pending / 禁点状态 |
| fee 编辑 | Approve and Sign 场景下支持 fee 编辑能力 |

### 9.4 异常处理

| 场景 | 规则 |
|-----|------|
| 打开失败（标准 Error） | 展示对应错误文案 |
| 打开失败（非 Error） | 展示 unknown error |
| 授权/签名/发送失败 | 状态停留或回退，可重新发起 |
| 签名链路中断 | 清理当前确认上下文 |
| 关闭/取消 | 主表单与确认页状态隔离 |

## 10. 通用规则

### 数据一致性
- K 线/列表/详情价格一致
- 多源差异策略（取中位/优先源/降级）

### 状态机
- WS 断连 → 重连 → 补数据 → 防闪烁
- 切币/切交易对

### 法币计价
- Market 全模块价格统一 USD 计价，不跟随用户默认货币
- 仅 Token Details 页面在默认货币非 USD 时，主价格下方展示次级法币等值价格
- Pro Mode「我的仓位」与其他模块共用，保留现有货币偏好逻辑

### 极端行情
- 精度丢失处理
- 科学计数法显示
- 涨跌幅异常处理
- 停牌/新币无数据处理

### 性能
- 长列表虚拟化
- K 线交互帧率
- 缓存/预加载策略

### 可观测
- 数据源降级提示
- 请求失败可恢复路径

## 11. 规则维护指南

### 如何更新规则

1. **发现规则变更**：
   - 在测试过程中发现规则与文档不一致
   - 收到产品/开发通知规则变更
   - 限价单支持新网络

2. **更新文档**：
   - 直接修改对应章节
   - 在变更记录中记录更新时间和原因
