# Wallet 模块测试规则文档

> 本文档记录 Wallet 模块的核心测试规则。
> 生成 Wallet 模块测试用例时，必须参考本文档。

---

## 1. 模块概述

Wallet 模块包含以下核心功能：
- 钱包首页（资产展示、余额刷新）
- 转账（主币、代币、NFT）
- Token 管理（添加、隐藏、搜索）
- 历史记录（交易列表、详情）
- NFT（展示、发送）
- 法币出入金
- 授权管理

## 2. 钱包首页规则

### 2.1 资产展示

| 规则项 | 规则描述 |
|-------|---------|

|-------|---------|

| 筛选 | 支持按链筛选 |

## 3. 转账规则

### 3.1 核心功能覆盖

**软件钱包测试用例应包含**（按以下顺序输出）：
- (a) 代币转账（含普通代币、免 Gas 代币）
- (b) 主币转账
- (c) 私钥转账（如适用）
- (d) 私钥导入导出
- (e) DApp 交易（连接、签名、合约交互）

**硬件钱包测试用例应包含**（仅以下三项，按以下顺序输出）：
- (a) 代币转账（含普通代币、免 Gas 代币）
- (b) 主币转账
- (c) DApp 交易（连接、签名、合约交互）

> **排序说明**：代币转账优先于主币转账输出，因为代币转账场景更复杂、覆盖面更广。

### 3.2 钱包类型分组规则（强制）

**核心原则**：生成转账测试用例时，必须将软件钱包和硬件钱包分开，输出为两篇独立文档。

**分组方式**：
1. **软件钱包文档**：`YYYY-MM-DD_Wallet-<链名>-软件钱包测试.md`
   - 保存目录：`docs/qa/testcases/cases/wallet/`
2. **硬件钱包文档**：`YYYY-MM-DD_Hardware-<链名>-硬件钱包测试.md`
   - 保存目录：`docs/qa/testcases/cases/hardware/`

### 3.3 金额转账测试规则

所有涉及金额转账的用例必须覆盖：

| 金额类型 | 定义 | 注意事项 |
|---------|-----|---------|
| 最大值 | 钱包余额（扣除账户最低余额和 Gas） | 考虑锁定金额 |
| 最小值 | 链规则允许的最小转账金额 | 参考各链规则 |
| 中间值 | 余额的中间值 | 覆盖正常场景 |

### 3.4 转账状态机

```
构建 → 签名 → 广播 → Pending → Success/Fail → 加速/取消/替换
```

### 3.5 各链转账规则

**详细的各链转账规则请参考**：`docs/qa/rules/transfer-chain-rules.md`

包含内容：
- 各链最小转账金额
- 账户最低余额限制
- Gas 规则
- Memo/Tag 规则
- 硬件钱包支持情况

### 3.6 软件钱包转账用例模板约束（强制）

生成软件钱包转账用例时，除遵守 `docs/qa/qa-rules.md` 的表格结构外，额外遵守以下约束以提升可读性与可维护性：

#### 3.6.1 前置条件与测试数据
- 用例开头用**纯文字**描述「前置条件与测试数据」，不使用表格，不使用“前置条件 0/第 0 章”等编号。
- 推荐结构（必须包含序号与换行）：
  - `1. 已导入测试助记词钱包`
  - `2. 账户 1 与账户 2 之间互转；余额不足则改用另一个账户`
  - `3. 覆盖网络：...`（仅当用例覆盖多个网络时需要；单一网络用例可省略该行）
- 前置条件中已声明的共性信息（如“已导入助记词、账户有余额”）**不得**在每条用例的“场景”里重复。
- 前置条件描述要**简洁**，避免冗余状态描述（如“钱包可解锁”）。
- 账户准备口径：默认使用**账户 1 与账户 2 互转**；余额不足时改用另一个账户作为付款账户。

#### 3.6.2 场景参数格式
- “场景”列仅写参数，统一为三行结构（按需省略第 3 行）：
  - `1. 网络：XXX、主币：YYY` 或 `1. 网络：XXX、代币：YYY`
  - `2. 金额=最小值 ...` / `2. 金额=中间值 ...` / `2. 金额=最大值 Max`
  - `3. 备注=...`（仅当填写备注时出现）
- 金额覆盖要求：必须显式标注 “最小值 / 中间值 / 最大值 Max”。

#### 3.6.3 操作步骤与预期结果的复用写法
- 第一条用例的“预期结果”写完整通用检查点；后续用例可使用：
  - 操作步骤：`同上（按场景参数）`
  - 预期结果：`同上` + 仅补充差异点
- 禁止在用例文件中新增“通用预期/附录”章节来承载通用断言；通用断言必须能与每条用例一一对应（通过“同上”复用实现）。
- 禁止在用例文件中新增“变更记录”章节（变更记录只保留在规则或需求文档中）。

#### 3.6.4 备注（Memo）用词与边界
- 用词统一：全篇统一使用“备注”，不混用 “Memo/memo”。
- 超过链规则上限时：预期结果统一写为 **“禁止提交”**。

## 5. 历史记录规则

### 5.0 最近转账

| 规则项 | 规则描述 |
|-------|---------|
| 数据源优先级 | 优先读取 `transfer-recipient` API；EVM 无结果时回退本地链历史；仍无结果时回退 stored recipients；其他链可再从当前链历史提取 |
| 条数上限 | 最近转账列表最多显示 20 条 |
| 排序 | 按最近转账时间倒序 |
| EVM 聚合 | EVM 链共享最近转账池，跨 EVM 链聚合 |
| EVM 去重 | EVM 地址按 lowercase 去重，避免 checksum 差异导致重复 |
| 链名展示 | EVM 聚合结果可展示最近一次转账所在链名称 |
| 过滤规则 | scam tx、Failed/Dropped、functionCall、合约地址不显示；**中心化 `transfer-recipient` 等接口返回已由服务端过滤，Swap 相关中心化转账记录不进入列表**；链上记录中「0 金额」与「转给自己」允许展示，客户端不做额外过滤 |
| Memo/Tag | 最近转账列表**可展示**与该笔记录关联的 memo/备注/tag；选择后**可**将地址与 memo/note/destinationTag 一并带入；**不要求**地址在地址簿；若同时命中地址簿联系人，合并/覆盖以产品实现为准 |
| 搜索与联想 | 仅地址、钱包账户名称；memo / note / destinationTag **无**搜索、无联想 |
| 账户维度 | 最近转账与当前 accountId 相关，切换账户后应刷新为当前账户数据 |
| 重装口径 | 不再默认依赖本地缓存是否存在；当前述数据源均无结果时显示空态 |

### 5.1 交易列表

| 规则项 | 规则描述 |
|-------|---------|
| 排序 | 按时间倒序 |
| 分页 | 支持下拉加载更多 |

| 金额 | 显示转账金额和法币价值 |
| Gas 费 | 显示实际消耗的 Gas 费 |
| 状态 | 显示交易状态 |

### 5.3 状态更新

| 场景 | 规则 |
|-----|------|
| Pending 状态 | 定时轮询更新状态 |
| 确认数 | 显示当前确认数/所需确认数 |

## 7. 从交易所接收规则

### 7.1 入口与展示

| 规则项 | 规则描述 |
|-------|---------|
| 入口位置 | 接收页面「从交易所接收」区域 |
| 支持交易所 | Binance、OKX、Coinbase |
| 展示方式 | 交易所列表默认展开可见，无需点击展开 |
| 交易所图标 | 每个交易所显示对应 Logo 和名称 |

### 7.2 Binance 流程规则

| 规则项 | 规则描述 |
|-------|---------|
| 接口数据 | supportAsset 接口同时返回支持的网络列表和代币列表 |
| 所有网络模式 | 先选网络（Network） → 再选代币（Token） |
| 单网络模式 | 无网络选择步骤，直接选代币（Token） |
| 跳转行为 | 选择代币后跳转 Binance，自动代入所选代币 |
| 充值反馈（插件/桌面端） | 展示充值结果页 |
| 充值反馈（移动端） | 自动跳转回 App |
| 返回行为 | 提币完成后点击返回，原路返回 OneKey App |
| 影响范围 | Binance 流程不受 v2 优化影响，仍使用独立 BinanceConnect 流程 |

### 7.3 OKX 流程规则（v2）

| 规则项 | 规则描述 |
|-------|---------|
| 应用检测 | 检测设备是否安装 OKX App |
| 未安装 | 跳转浏览器帮助中心对应文章 |
| 非硬件钱包流程 | ReceiveSelector → 选择代币 → 获取地址 → ExchangeOpenRedirect → 3s 倒计时 → 打开 OKX → 关闭弹窗 |
| 硬件钱包流程 | ReceiveSelector → 选择代币 → ReceiveToken（QR + 设备验证） → 确认 → ExchangeOpenRedirect → 3s 倒计时 → 打开 OKX → 关闭弹窗 |
| 地址复制 | ExchangeOpenRedirect 页面加载时自动复制到剪贴板（不显示 Toast） |

### 7.4 Coinbase 流程规则（v2）

| 规则项 | 规则描述 |
|-------|---------|
| 应用检测 | 检测设备是否安装 Coinbase App |
| 已安装 | 走 ExchangeOpenRedirect 流程（与 OKX 一致） |
| 未安装 | 跳转浏览器帮助中心对应文章 |
| 非硬件钱包流程 | ReceiveSelector → 选择代币 → 获取地址 → ExchangeOpenRedirect → 3s 倒计时 → 打开 Coinbase → 关闭弹窗 |
| 硬件钱包流程 | ReceiveSelector → 选择代币 → ReceiveToken（QR + 设备验证） → 确认 → ExchangeOpenRedirect → 3s 倒计时 → 打开 Coinbase → 关闭弹窗 |

### 7.5 ExchangeOpenRedirect 页面规则（新增）

| 规则项 | 规则描述 |
|-------|---------|
| 适用端 | 移动端（iOS / Android） |
| 交易所 Logo | 页面顶部显示对应交易所的 Logo |
| 地址文案 | 显示 "已复制地址 '0x1234...abcd'"（缩略地址格式） |
| 倒计时 | "正在打开 OKX，还有 3 秒..."（带 Spinner 动画） |
| 手动按钮 | 底部 "Open OKX" 按钮（secondary 样式），倒计时期间可手动点击 |
| 自动行为 | 倒计时结束后自动打开交易所 App 并关闭弹窗 |
| 剪贴板 | 页面加载时自动复制地址到剪贴板，不显示 Toast（页面已有文字提示） |

### 7.6 通用规则

| 规则项 | 规则描述 |
|-------|---------|
| 未安装兜底 | 未安装对应交易所 App 统一跳转帮助中心 |
| 硬件钱包确认 | 涉及地址展示时需先在硬件设备上确认 |
| Binance 独立 | Binance 不走 ExchangeOpenRedirect，仍使用独立 BinanceConnect 流程 |

## 8. 通用规则

### 状态机
- 钱包首页加载 → 资产刷新 → Token 管理 → 历史记录

### 资金一致性
- 资产页/详情页/交易记录三处一致
- 隐藏资产不影响总余额

### 地址/规则
- 合约地址拦截/提示
- 自转自
- Memo/Tag 校验
- ENS/域名解析

### 链上不确定性
- Gas 波动
- RPC 不一致
- 链重组导致状态回滚
- 广播失败重试

### 风控
- 黑名单地址
- 风险合约提示
- 大额二次确认
- 频率限制

### 可观测
- 失败原因必须落到可验证字段
- error code + 用户文案 + hash/nonce

## 9. 规则维护指南

### 如何更新规则

1. **发现规则变更**：
   - 在测试过程中发现规则与文档不一致
   - 收到产品/开发通知规则变更

2. **更新文档**：
   - 通用规则更新本文档
   - 各链具体规则更新 `transfer-chain-rules.md`
   - 在变更记录中记录更新时间和原因
