# Wallet 模块测试规则文档

> 本文档记录 Wallet 模块的核心测试规则。
> 生成 Wallet 模块测试用例时，必须参考本文档。

---

## 1. 模块概述

Wallet 模块包含以下核心功能：
- 钱包首页（资产展示、余额刷新）
- 转账（主币、代币、NFT）
- Token 管理（添加、隐藏、搜索）
- 历史记录（交易列表、详情）
- NFT（展示、发送）
- 法币出入金
- 授权管理

---

## 2. 钱包首页规则

### 2.1 资产展示

| 规则项 | 规则描述 |
|-------|---------|
| 余额计算 | 显示所有 Token 余额的法币总价值 |

| 多链展示 | 按链分组或统一展示（可切换） |

### 2.2 Token 列表

| 规则项 | 规则描述 |
|-------|---------|
| 排序 | 按余额法币价值降序（或自定义排序） |
| 搜索 | 支持按名称、符号搜索 |
| 筛选 | 支持按链筛选 |
| 空状态 | 无 Token 时显示引导添加 |

## 3. 转账规则

### 3.1 核心功能覆盖

**软件钱包测试用例应包含**（按以下顺序输出）：
- (a) 代币转账（含普通代币、免 Gas 代币）
- (b) 主币转账
- (c) 私钥转账（如适用）
- (d) 私钥导入导出
- (e) DApp 交易（连接、签名、合约交互）

**硬件钱包测试用例应包含**（仅以下三项，按以下顺序输出）：
- (a) 代币转账（含普通代币、免 Gas 代币）
- (b) 主币转账
- (c) DApp 交易（连接、签名、合约交互）

> **排序说明**：代币转账优先于主币转账输出，因为代币转账场景更复杂、覆盖面更广。

### 3.2 钱包类型分组规则（强制）

**核心原则**：生成转账测试用例时，必须将软件钱包和硬件钱包分开，输出为两篇独立文档。

**分组方式**：
1. **软件钱包文档**：`YYYY-MM-DD_Wallet-<链名>-软件钱包测试.md`
   - 保存目录：`docs/qa/testcases/cases/wallet/`
2. **硬件钱包文档**：`YYYY-MM-DD_Hardware-<链名>-硬件钱包测试.md`
   - 保存目录：`docs/qa/testcases/cases/hardware/`

### 3.3 金额转账测试规则

所有涉及金额转账的用例必须覆盖：

| 金额类型 | 定义 | 注意事项 |
|---------|-----|---------|
| 最大值 | 钱包余额（扣除账户最低余额和 Gas） | 考虑锁定金额 |
| 最小值 | 链规则允许的最小转账金额 | 参考各链规则 |
| 中间值 | 余额的中间值 | 覆盖正常场景 |

### 3.4 转账状态机

```
构建 → 签名 → 广播 → Pending → Success/Fail → 加速/取消/替换
```

### 3.5 各链转账规则

**详细的各链转账规则请参考**：`docs/qa/rules/transfer-chain-rules.md`

包含内容：
- 各链最小转账金额
- 账户最低余额限制
- Gas 规则
- Memo/Tag 规则
- 硬件钱包支持情况

## 7. 从交易所接收规则

### 7.1 入口与展示

| 规则项 | 规则描述 |
|-------|---------|
| 入口位置 | 接收页面「从交易所接收」区域 |
| 支持交易所 | Binance、OKX、Coinbase |
| 展示方式 | 交易所列表默认展开可见，无需点击展开 |
| 交易所图标 | 每个交易所显示对应 Logo 和名称 |

### 7.2 Binance 流程规则

| 规则项 | 规则描述 |
|-------|---------|
| 接口数据 | supportAsset 接口同时返回支持的网络列表和代币列表 |
| 所有网络模式 | 先选网络（Network） → 再选代币（Token） |
| 单网络模式 | 无网络选择步骤，直接选代币（Token） |
| 跳转行为 | 选择代币后跳转 Binance，自动代入所选代币 |
| 充值反馈（插件/桌面端） | 展示充值结果页 |
| 充值反馈（移动端） | 自动跳转回 App |
| 返回行为 | 提币完成后点击返回，原路返回 OneKey App |
| 影响范围 | Binance 流程不受 v2 优化影响，仍使用独立 BinanceConnect 流程 |

### 7.3 OKX 流程规则（v2）

| 规则项 | 规则描述 |
|-------|---------|
| 应用检测 | 检测设备是否安装 OKX App |
| 未安装 | 跳转浏览器帮助中心对应文章 |
| 非硬件钱包流程 | ReceiveSelector → 选择代币 → 获取地址 → ExchangeOpenRedirect → 3s 倒计时 → 打开 OKX → 关闭弹窗 |
| 硬件钱包流程 | ReceiveSelector → 选择代币 → ReceiveToken（QR + 设备验证） → 确认 → ExchangeOpenRedirect → 3s 倒计时 → 打开 OKX → 关闭弹窗 |
| 地址复制 | ExchangeOpenRedirect 页面加载时自动复制到剪贴板（不显示 Toast） |

### 7.4 Coinbase 流程规则（v2）

| 规则项 | 规则描述 |
|-------|---------|
| 应用检测 | 检测设备是否安装 Coinbase App |
| 已安装 | 走 ExchangeOpenRedirect 流程（与 OKX 一致） |
| 未安装 | 跳转浏览器帮助中心对应文章 |
| 非硬件钱包流程 | ReceiveSelector → 选择代币 → 获取地址 → ExchangeOpenRedirect → 3s 倒计时 → 打开 Coinbase → 关闭弹窗 |
| 硬件钱包流程 | ReceiveSelector → 选择代币 → ReceiveToken（QR + 设备验证） → 确认 → ExchangeOpenRedirect → 3s 倒计时 → 打开 Coinbase → 关闭弹窗 |

### 7.5 ExchangeOpenRedirect 页面规则（新增）

| 规则项 | 规则描述 |
|-------|---------|
| 适用端 | 移动端（iOS / Android） |
| 交易所 Logo | 页面顶部显示对应交易所的 Logo |
| 地址文案 | 显示 "已复制地址 '0x1234...abcd'"（缩略地址格式） |
| 倒计时 | "正在打开 OKX，还有 3 秒..."（带 Spinner 动画） |
| 手动按钮 | 底部 "Open OKX" 按钮（secondary 样式），倒计时期间可手动点击 |
| 自动行为 | 倒计时结束后自动打开交易所 App 并关闭弹窗 |
| 剪贴板 | 页面加载时自动复制地址到剪贴板，不显示 Toast（页面已有文字提示） |

### 7.6 通用规则

| 规则项 | 规则描述 |
|-------|---------|
| 未安装兜底 | 未安装对应交易所 App 统一跳转帮助中心 |
| 硬件钱包确认 | 涉及地址展示时需先在硬件设备上确认 |
| Binance 独立 | Binance 不走 ExchangeOpenRedirect，仍使用独立 BinanceConnect 流程 |

## 8. Gas Account 规则

### 8.1 功能定位

Gas Account 是 OneKey 钱包内的"代付 Gas 账户"，用户充值 USDT 等值资产后，可用于支付多链交易的 Gas 费用。

### 8.2 账户体系

| 用户类型 | 能力 |
|---------|------|
| 未登录用户 | 可享受平台补贴额度（Guest 模式） |
| 已登录 OneKey ID 用户 | 充值、提现、查看记录，所有地址共享余额 |

### 8.3 赞助规则

| 规则 | 内容 | 不通过时行为 |
|------|------|-------------|
| 链白名单 | evm--1 (ETH), evm--137 (Polygon), evm--42161 (Arbitrum), evm--10 (Optimism), evm--56 (BSC) | 返回不支持 |
| 余额检查 | Gas Account 余额 ≥ 该笔 Gas 费用 | 返回余额不足 |
| 地址黑名单 | from_address 不在封禁列表 | 返回不支持 |
| Gas 价格熔断 | ETH maxGwei=200, Polygon maxGwei=500 | 暂停赞助 |

### 8.4 API 接口

**EstimateFee 返回字段**：`gasAccountInfo.isGasAccountAvailable` / `balanceIsEnough` / `chainNotSupport` / `errMsg` / `cost`（totalCost, txCost, gasCost, estimateTxCost）

**sendTransaction 请求字段**：`feePaymentMethod: "gas_account"`

### 8.5 费用结构

| 字段 | 含义 |
|------|------|
| totalCost | 用户实际扣费 = txCost + gasCost |
| txCost | 给用户转账的 gas 值 |
| gasCost | 链上 gas 成本 |
| estimateTxCost | 预估交易成本（前端展示参考） |

### 8.6 充值规则

| 项目 | 规则 |
|------|------|
| 充值方式 | Apple Pay（第一期） |
| 预设金额 | $10 / $20 / $50 |
| 自定义金额 | 支持用户自定义输入 |
| 到账时间 | 即时到账 |

### 8.7 交易确认页规则

| 项目 | 规则 |
|------|------|
| 开关位置 | 交易确认页 Network Fee 区域 |
| 开关条件 | EstimateFee 返回 isGasAccountAvailable=true 时显示 |
| 余额不足 | 引导登录/充值 |
| 不支持链 | 不显示开关或显示不支持提示 |

### 8.8 记录与流水

- 记录类型：Gas 代付、充值、提现
- 记录字段：network_id, from_address, user_tx_hash, gas_tx_hash, total_cost, tx_cost, gas_cost, status (pending/success/failed), created_at, confirmed_at
- 支持按类型筛选

## 9. 通用规则

### 状态机
- 钱包首页加载 → 资产刷新 → Token 管理 → 历史记录

### 资金一致性
- 资产页/详情页/交易记录三处一致
- 隐藏资产不影响总余额

### 地址/规则
- 合约地址拦截/提示
- 自转自
- Memo/Tag 校验
- ENS/域名解析

### 链上不确定性
- Gas 波动
- RPC 不一致
- 链重组导致状态回滚
- 广播失败重试

### 风控
- 黑名单地址
- 风险合约提示
- 大额二次确认
- 频率限制

### 可观测
- 失败原因必须落到可验证字段
- error code + 用户文案 + hash/nonce

## 10. 规则维护指南

### 如何更新规则

1. **发现规则变更**：
   - 在测试过程中发现规则与文档不一致
   - 收到产品/开发通知规则变更

2. **更新文档**：
   - 通用规则更新本文档
   - 各链具体规则更新 `transfer-chain-rules.md`
   - 在变更记录中记录更新时间和原因
