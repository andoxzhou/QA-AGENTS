# Wallet 模块测试规则文档

> 本文档记录 Wallet 模块的核心测试规则。
> 生成 Wallet 模块测试用例时，必须参考本文档。

---

## 1. 模块概述

Wallet 模块包含以下核心功能：
- 钱包首页（资产展示、余额刷新）
- 转账（主币、代币、NFT）
- Token 管理（添加、隐藏、搜索）
- 历史记录（交易列表、详情）
- NFT（展示、发送）
- 法币出入金
- 授权管理

---

## 1.1 钱包类型术语规范（全模块通用）

> 生成 Wallet / Account / Swap 等相关用例和文档时，**钱包类型描述必须使用以下简写**，不要再出现"软件钱包"/"硬件钱包"等旧称。

| 简写 | 含义 | 备注 |
|------|------|------|
| `HD（已备份）` | 已完成助记词备份的 HD 软件钱包 | 可参与转账、展示在「我的账户」「选择地址」等入口 |
| `HD（未备份）` | 未备份的 HD 软件钱包 | **不参与转账选择**，需先完成备份 |
| `HW` | 硬件钱包（Hardware Wallet） | Mini / Classic / Pro / Touch 等 |
| `已重置的 HW` | 在设备端执行过恢复出厂/重置的 HW | 从本地移除，其分组与账户**不再展示**在「账户」Tab，联想也一并清理 |
| `私钥账户` | 通过私钥导入的单账户 | 不参与助记词派生，不参与同助记词头像一致性 |
| `Keyless Wallet` | 无私钥钱包（社交登录等） | 作为独立分组展示 |
| `QR Wallet` | QR 离线签名钱包 | 作为独立分组展示；删除后联想同步清理 |
| `隐藏钱包` | 基于已备份主钱包创建的标准隐藏钱包 | 默认简称"隐藏钱包"，不再写"标准隐藏钱包" |
| `未创建标准钱包的隐藏钱包` | 仅有隐藏钱包、无主钱包账户的场景 | **仍必须独立显示并可选**，不能因无主钱包而隐藏整个分组 |
| `已删除的隐藏钱包` | 用户在钱包选择器主动删除的隐藏钱包 | 从本地移除；账户列表和联想一并清理 |
| `观察账户` / `外部账户` | 只读账户 | 无转账权限，**任何发送/转账入口的账户列表中都不展示** |

**写作约束**：
- 场景列与前置条件统一使用上表简写；不再使用"已解锁的标准隐藏钱包"这类冗余定语（解锁状态是隐藏钱包进入展示的前提，不是独立钱包类型）。
- "未解锁的隐藏钱包"**不是**持久状态，属于 UI 可操作场景，不作为隐藏范围的分类（真正需要从列表/联想中剔除的是**已删除**和**已重置**的钱包）。

## 2. 钱包首页规则

### 2.1 资产展示

| 规则项 | 规则描述 |
|-------|---------|
| 余额计算 | 显示所有 Token 余额的法币总价值 |
| 隐藏资产 | 支持隐藏小额/归零资产，不影响总余额计算 |
| 刷新策略 | 下拉刷新、自动刷新（间隔可配置） |
| 多链展示 | 按链分组或统一展示（可切换） |

### 2.2 Token 列表

| 规则项 | 规则描述 |
|-------|---------|
| 排序 | 按余额法币价值降序（或自定义排序） |
| 搜索 | 支持按名称、符号搜索 |
| 筛选 | 支持按链筛选 |
| 空状态 | 无 Token 时显示引导添加 |

## 3. 转账规则

### 3.1 核心功能覆盖

**软件钱包测试用例应包含**（按以下顺序输出）：
- (a) 代币转账（含普通代币、免 Gas 代币）
- (b) 主币转账
- (c) 私钥转账（如适用）
- (d) 私钥导入导出
- (e) DApp 交易（连接、签名、合约交互）

**硬件钱包测试用例应包含**（仅以下三项，按以下顺序输出）：
- (a) 代币转账（含普通代币、免 Gas 代币）
- (b) 主币转账
- (c) DApp 交易（连接、签名、合约交互）

> **排序说明**：代币转账优先于主币转账输出，因为代币转账场景更复杂、覆盖面更广。

### 3.2 钱包类型分组规则（强制）

**核心原则**：生成转账测试用例时，必须将软件钱包和硬件钱包分开，输出为两篇独立文档。

**分组方式**：
1. **软件钱包文档**：`YYYY-MM-DD_Transfer-<链名>-转账dApp软件钱包.md`
   - 保存目录：`docs/qa/testcases/cases/transfer/`
2. **硬件钱包文档**：`YYYY-MM-DD_Transfer-<链名>-转账dApp硬件钱包.md`
   - 保存目录：`docs/qa/testcases/cases/transfer/`

**参考现有用例**：
- `docs/qa/testcases/cases/transfer/2026-01-07_Transfer-Stellar-转账dApp软件钱包.md`
- `docs/qa/testcases/cases/transfer/2026-04-08_Transfer-Cosmos转账-软件钱包.md`
- `docs/qa/testcases/cases/transfer/2026-04-17_Transfer-闪电网络-转账dApp软件钱包.md`

**命名说明**：
- 文件名以 `Transfer-` 开头（不是 `Wallet-` 也不是 `Hardware-`），所有链的转账用例统一归到 `transfer/` 目录
- 软件 / 硬件通过文件名后缀区分（`软件钱包` vs `硬件钱包`），不再按目录拆分
- 原 `wallet/` 目录保留给**非转账类** Wallet 模块用例（首页、Token、NFT、历史、法币、授权等）

### 3.2.1 硬件钱包用例内容边界（强制）

**核心原则**：硬件钱包用例**只写需要硬件签名的操作**，不在硬件文档中重复软件侧的断言。

**必须覆盖的内容**：
- 当前链上**实际需要硬件签名**的操作（如发送 / dApp 支付 / 通道开关 / 授权签名）
- 硬件设备上显示的签名详情（发送方 / 金额 / 链名 / 代币 / 描述 / 网络费等）
- 签名确认后的交易上链结果、余额变化

**禁止覆盖的内容**（由软件钱包用例覆盖，硬件文档不重复）：
- 接收 / 生成地址 / 生成 Invoice 等**无需签名**的操作
- App 内的连接授权流程（弹窗显示、授权 / 拒绝按钮等 UI）
- App 内的 dApp 授权登录（Alby / WebLN / Nostr Wallet Connect）
- 账户选择 / 切换、地址簿、最近转账等 App 侧操作
- 历史记录字段展示（金额 / 时间格式 / 哈希跳转 / 网络标识等属 App 侧 UI）
- 交易确认页字段校验（App 侧展示）
- 金额输入边界 / 描述长度边界等 App 侧输入校验
- UI 无异常、loading 时长等通用性能检查

**用例写作要求**：
- 每条用例的"操作步骤"可简写为"App 粘贴地址并确认 → 硬件设备上查看签名详情 → 硬件按键确认"，不展开 App 内步骤
- 每条用例的"预期结果"聚焦 **硬件设备显示 + 交易上链 + 余额变化**，不混入 App 侧断言
- 如果整个功能在当前链上**不需要硬件签名**（如闪电网络接收、dApp 授权登录），硬件文档中**不创建该功能的用例分组**

**参考用例**：`docs/qa/testcases/cases/transfer/2026-04-17_Transfer-闪电网络-转账dApp硬件钱包.md`

### 3.3 金额转账测试规则

所有涉及金额转账的用例必须覆盖：

| 金额类型 | 定义 | 注意事项 |
|---------|-----|---------|
| 最大值 | 钱包余额（扣除账户最低余额和 Gas） | 考虑锁定金额 |
| 最小值 | 链规则允许的最小转账金额 | 参考各链规则 |
| 中间值 | 余额的中间值 | 覆盖正常场景 |

### 3.4 转账状态机

```
构建 → 签名 → 广播 → Pending → Success/Fail → 加速/取消/替换
```

### 3.5 各链转账规则

**详细的各链转账规则请参考**：`docs/qa/rules/transfer-chain-rules.md`

包含内容：
- 各链最小转账金额
- 账户最低余额限制
- Gas 规则
- Memo/Tag 规则
- 硬件钱包支持情况

### 3.6 软件钱包转账用例模板约束（强制）

生成软件钱包转账用例时，除遵守 `docs/qa/qa-rules.md` 的表格结构外，额外遵守以下约束以提升可读性与可维护性：

#### 3.6.1 前置条件与测试数据
- 用例开头用**纯文字**描述「前置条件与测试数据」，不使用表格，不使用“前置条件 0/第 0 章”等编号。
- 推荐结构（必须包含序号与换行）：
  - `1. 已导入测试助记词钱包`
  - `2. 账户 1 与账户 2 之间互转；余额不足则改用另一个账户`
  - `3. 覆盖网络：...`（仅当用例覆盖多个网络时需要；单一网络用例可省略该行）
- 前置条件中已声明的共性信息（如“已导入助记词、账户有余额”）**不得**在每条用例的“场景”里重复。
- 前置条件描述要**简洁**，避免冗余状态描述（如“钱包可解锁”）。
- 账户准备口径：默认使用**账户 1 与账户 2 互转**；余额不足时改用另一个账户作为付款账户。

#### 3.6.2 场景参数格式
- “场景”列仅写参数，统一为三行结构（按需省略第 3 行）：
  - `1. 网络：XXX、主币：YYY` 或 `1. 网络：XXX、代币：YYY`
  - `2. 金额=最小值 ...` / `2. 金额=中间值 ...` / `2. 金额=最大值 Max`
  - `3. 备注=...`（仅当填写备注时出现）
- 金额覆盖要求：必须显式标注 “最小值 / 中间值 / 最大值 Max”。

#### 3.6.3 操作步骤与预期结果的复用写法
- 第一条用例的“预期结果”写完整通用检查点；后续用例可使用：
  - 操作步骤：`同上（按场景参数）`
  - 预期结果：`同上` + 仅补充差异点
- 禁止在用例文件中新增“通用预期/附录”章节来承载通用断言；通用断言必须能与每条用例一一对应（通过“同上”复用实现）。
- 禁止在用例文件中新增“变更记录”章节（变更记录只保留在规则或需求文档中）。

#### 3.6.4 备注（Memo）用词与边界
- 用词统一：全篇统一使用“备注”，不混用 “Memo/memo”。
- 超过链规则上限时：预期结果统一写为 **“禁止提交”**。

## 5. 历史记录规则

### 5.0 最近转账

| 规则项 | 规则描述 |
|-------|---------|
| 数据源优先级 | 优先读取 `transfer-recipient` API；EVM 无结果时回退本地链历史；仍无结果时回退 stored recipients；其他链可再从当前链历史提取 |
| 条数上限 | 最近转账列表最多显示 20 条 |
| 排序 | 按最近转账时间倒序 |
| EVM 聚合 | EVM 链共享最近转账池，跨 EVM 链聚合 |
| EVM 去重 | EVM 地址按 lowercase 去重，避免 checksum 差异导致重复 |
| 链名展示 | EVM 聚合结果可展示最近一次转账所在链名称 |
| 过滤规则 | scam tx、Failed/Dropped、functionCall、合约地址不显示；**中心化 `transfer-recipient` 等接口返回已由服务端过滤，Swap 相关中心化转账记录不进入列表**；链上记录中「0 金额」与「转给自己」允许展示，客户端不做额外过滤 |
| Memo/Tag | 最近转账列表**可展示**与该笔记录关联的 memo/备注/tag；选择后**可**将地址与 memo/note/destinationTag 一并带入；**不要求**地址在地址簿；若同时命中地址簿联系人，合并/覆盖以产品实现为准 |
| 搜索与联想 | 仅地址、钱包账户名称；memo / note / destinationTag **无**搜索、无联想 |
| 账户维度 | 最近转账与当前 accountId 相关，切换账户后应刷新为当前账户数据 |
| 重装口径 | 不再默认依赖本地缓存是否存在；当前述数据源均无结果时显示空态 |

### 5.05 我的账户（选择地址 - 账户 Tab）

| 规则项 | 规则描述 |
|-------|---------|
| 展示范围 | HD（已备份）、HW、私钥账户、Keyless Wallet、QR Wallet、隐藏钱包、未创建标准钱包的隐藏钱包（该隐藏钱包所在的主钱包未创建标准钱包账户时仍必须可见可选） |
| 隐藏范围 | 观察账户（无转账权限）、外部账户、HD（未备份）、已删除的隐藏钱包、已重置的 HW（后两者已从本地移除，分组与账户应一并清除） |
| 分组与折叠 | 按钱包分组展示，默认全部展开；折叠状态在 Tab 切换间保持，退出弹窗后重置为展开 |
| BTC/LTC 路径切换 | 已创建多路径地址时，分组标题旁显示路径切换下拉按钮（Nested SegWit / Native SegWit / Legacy / Taproot）；仅创建单路径时不显示 |
| EVM/Solana 路径 | **不提供**快捷路径切换按钮；展示系统设置中当前选定派生路径对应的地址 |
| BTC 新鲜地址 | 开启新鲜地址时，列表展示当前新鲜地址，选中即代入新鲜地址；搜索已使用的旧地址时，展示并代入旧地址 |
| 搜索范围 | 钱包名称、账户名称、地址（含新鲜地址和历史旧地址）；大小写不敏感 |
| 搜索时路径过滤 | BTC/LTC 搜索时**跨所有派生路径**匹配，不受当前选中路径限制；清空搜索后恢复路径过滤；搜索结果**不显示派生路径标签**（如 "Legacy"、"Taproot" 等） |
| 默认 Tab | BTC/LTC（mergeDeriveAssetsEnabled）默认选中「账户」Tab；其他链默认选中「最近」Tab |
| Lightning Network | **不支持**账户与地址簿选择，仅显示「最近」Tab |
| Tab 搜索计数 | 搜索时 Tab 标签显示匹配数量（如 "账户 (3)"）；当前 Tab 无匹配但其他 Tab 有匹配时自动切换 |
| 钱包/账户头像一致性 | 头像由系统在创建时生成，**创建后不支持修改**；「账户」Tab 中钱包分组头像与**钱包选择器**中对应钱包头像完全一致；账户头像在钱包选择器、「账户」Tab、「最近」Tab、账户详情四处展示**完全一致**；同助记词下相同 index 账户头像一致（详见 `account-rules.md` §1） |
| 名称截断 | 钱包名称、账户名称超长时均需**省略号截断**（`MyVeryLongName...`），不得挤压路径切换按钮、头像或地址的展示 |
| Web 端 Swap 自定义接收地址 | 仅 Web 端 Swap 开启"自定义接收地址"场景**屏蔽「最近」「账户」「地址簿」全部三个 Tab**，仅保留手动输入入口；其他端不在约束范围内。详见 `swap-rules.md` §自定义接收地址 |

### 5.05.1 地址输入框联想（跨 Tab 全局行为）

> 地址输入框的联想提示是跨「最近」「账户」「地址簿」三个数据源的全局行为，与本章 §5.05 展示范围联动。

| 规则项 | 规则描述 |
|-------|---------|
| 我的账户联想 | 粘贴/输入属于「我的账户」分组（见 §5.05 展示范围）的地址时，下方联想提示展示"我的账户"标签和对应账户名 |
| BTC 新鲜地址联想 | 开启新鲜地址时，粘贴该账户**任一已使用过的旧地址**也应正确联想到该账户名（不得为空或指向错误账户） |
| 多来源标注 | 地址同时存在于「最近转账」和「我的账户」时，联想提示应同时标注两个来源 |
| 外部地址 | 地址不在三个来源中时，不展示联想提示，直接进入金额输入流程 |
| **钱包删除后联想清理** | 删除某钱包（含 HD、HW、隐藏钱包、QR Wallet 等）后，其下属账户地址**不再**出现在联想来源中（不展示"我的账户"标签及账户名，按外部地址处理） |
| **HW 重置后联想清理** | 对 HW 执行恢复出厂/重置后，该 HW 原有账户地址**不再**出现在联想来源中（等同钱包删除的联想清理行为） |
| **钱包重置后联想清理** | 执行钱包重置（清除所有钱包数据）后，重置前记录的所有账户地址**不再**出现在联想来源中；重新导入的同助记词钱包按新钱包生成的账户数据重建联想 |

## 8. Gas Account 规则

### 8.1 功能定位

Gas Account 是 OneKey 钱包内的"代付 Gas 账户"，用户充值 USDT 等值资产后，可用于支付多链交易的 Gas 费用。

### 8.2 账户体系

| 用户类型 | 能力 |
|---------|------|
| 未登录用户 | 可享受平台补贴额度（Guest 模式） |
| 已登录 OneKey ID 用户 | 充值、提现、查看记录，所有地址共享余额 |

### 8.3 赞助规则

| 规则 | 内容 | 不通过时行为 |
|------|------|-------------|
| 链白名单 | evm--1 (ETH), evm--137 (Polygon), evm--42161 (Arbitrum), evm--10 (Optimism), evm--56 (BSC) | 返回不支持 |
| 余额检查 | Gas Account 余额 ≥ 该笔 Gas 费用 | 返回余额不足 |
| 地址黑名单 | from_address 不在封禁列表 | 返回不支持 |
| Gas 价格熔断 | ETH maxGwei=200, Polygon maxGwei=500 | 暂停赞助 |

### 8.4 API 接口

**SignatureConfirm 赞助交易的主口径见 §8.9**（与 `docs/qa/testcases/cases/wallet/2026-03-31_Wallet-GasAccount功能测试.md` 一致）。本节下列 `gasAccountInfo` / `feePaymentMethod` 为 **Gas 面板与后端兼容字段**；若与 §8.9 字段并存，以实现版本为准。

**EstimateFee（兼容字段）**：`gasAccountInfo.isGasAccountAvailable` / `balanceIsEnough` / `chainNotSupport` / `errMsg` / `cost`（totalCost, txCost, gasCost, estimateTxCost）

**sendTransaction（兼容字段）**：`feePaymentMethod: "gas_account"`

### 8.5 费用结构

| 字段 | 含义 |
|------|------|
| totalCost | 用户实际扣费 = txCost + gasCost |
| txCost | 给用户转账的 gas 值 |
| gasCost | 链上 gas 成本 |
| estimateTxCost | 预估交易成本（前端展示参考） |

### 8.6 充值规则

| 项目 | 规则 |
|------|------|
| 充值方式 | Apple Pay（第一期） |
| 预设金额 | $10 / $20 / $50 |
| 自定义金额 | 支持用户自定义输入 |
| 到账时间 | 即时到账 |

### 8.7 交易确认页规则

| 项目 | 规则 |
|------|------|
| 开关位置 | 交易确认页 Network Fee 区域（若产品保留 Sponsor 开关） |
| 开关条件 | EstimateFee 返回 isGasAccountAvailable=true 时显示；若以 `/estimate-fee` 的 `payer` / `gasAccountEligible` 自动决定免费态，则以 §8.9 为准 |
| 余额不足 | 引导登录/充值 |
| 不支持链 | 不显示开关或显示不支持提示 |

### 8.8 记录与流水

- 记录类型：Gas 代付、充值、提现
- 记录字段：network_id, from_address, user_tx_hash, gas_tx_hash, total_cost, tx_cost, gas_cost, status (pending/success/failed), created_at, confirmed_at
- 支持按类型筛选

### 8.9 SignatureConfirm Gas Sponsor（estimate-fee / send-transaction）

> 覆盖发送、Swap、Perps、Earn、dApp 等走交易确认页的链路；**自定义 RPC 时禁用** Gas Account 估价。

| 项 | 规则 |
| --- | --- |
| 估价请求 | `/estimate-fee` 携带 `lockedUserNonce`；可用赞助时 `gasAccountEnabled=true`，B 类 fallback 后需 `gasAccountEnabled=false` 避免误切回 sponsor |
| 估价返回 | `payer` 为 `user` \| `megafuel` \| `gasAccount`；`gasAccountEligible`；赞助时返回 `gasAccountQuote`（`quoteId`、`maxFee`、`expiresAt`） |
| 提交 | `payer=gasAccount` 时 `/send-transaction` 携带 `quoteId`（来自 quote）与 `idempotencyKey`；`payer=user` 时不携带二者 |
| A 类错误 | 40201（quote 过期）、40202（nonce 变化）、40209、90201 等：不弹默认错误 toast，自动重新 `/estimate-fee` |
| B 类错误 | 40213、90200、40218、40219、90205 等：`gasAccountTemporarilyDisabled=true`，payer 切回 `user`，清空 quote，自动 re-estimate，用户自付完成交易 |
| C 类错误 | 40203、40214–40217、90207–90209：仅状态提示，不自动重估、不自动 fallback |
| 成功 Toast | Gas-sponsored 提交成功：文案 `Gas-sponsored transaction submitted`，图标 GiftSolid；用户自付沿用原默认完成提示 |
| 每日限额 | 用尽后 `payer=user` / `gasAccountEligible=false`，确认页显示常规 Network Fee，不包含 quote 字段 |

## 9. 通用规则

### 状态机
- 钱包首页加载 → 资产刷新 → Token 管理 → 历史记录

### 资金一致性
- 资产页/详情页/交易记录三处一致
- 隐藏资产不影响总余额

### 地址/规则
- 合约地址拦截/提示
- 自转自
- Memo/Tag 校验
- ENS/域名解析

### 链上不确定性
- Gas 波动
- RPC 不一致
- 链重组导致状态回滚
- 广播失败重试

### 风控
- 黑名单地址
- 风险合约提示
- 大额二次确认
- 频率限制

### 可观测
- 失败原因必须落到可验证字段
- error code + 用户文案 + hash/nonce
