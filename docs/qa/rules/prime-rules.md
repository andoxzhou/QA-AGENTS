# Prime 模块测试规则文档

> 本文档记录 Prime 模块的核心测试规则。
> 生成 Prime 相关测试用例时，必须参考本文档。

---

## 1. 模块概述

Prime 模块包含以下核心功能：
- 会员权益展示
- 订阅/续费流程
- 会员专属功能
- 订阅管理

| 方式 | 规则 |

### 3.3 订阅周期

| 周期 | 规则 |
|-----|------|
| 月付 | 按月订阅 |
| 年付 | 按年订阅（优惠价格） |

### 3.4 订阅状态机

```
未订阅 → 已订阅（自动扣费续订，状态持续保持已订阅）
              ↓ 用户手动取消订阅
         已取消（当前周期内保留权益）
              ↓ 当前周期到期
         未订阅（可重新订阅）
```

## 4.3 OneKey Cloud 规则

### 页面状态

| 状态 | 触发条件 | 页面表现 |
|-----|---------|---------|
| 未创建 Keyless Wallet | 无 Keyless Wallet，且同步关闭 | 显示插图、说明文案与 `Create and enable syncing` 按钮 |
| 已创建 Keyless Wallet，同步关闭 | 已有 Keyless Wallet，但同步关闭 | 显示 OneKey Cloud 开关，默认 OFF |
| Keyless Sync 已开启 | Keyless 同步已开启 | 显示开关 ON、Keyless Wallet 信息行（钱包 icon + 名称）与 `Sync now` |
| Syncing Paused | Keyless 同步开启但 Keyless Wallet 已删除 | 显示 `Syncing Paused` 告警与恢复按钮 |

### 入口迁移行为

| 用户历史状态 | 入口展示 | 规则描述 |
|------------|---------|---------|
| 曾使用过 OneKeyID 同步 | 提示切换为 Keyless 同步 | 引导用户从 OneKeyID 同步迁移到 Keyless 同步 |
| 未使用过 OneKeyID 同步 | 仅显示启用 Keyless 同步 | 新用户直接进入 Keyless 同步流程 |
| 已启用 OneKeyID 同步（当前生效中） | 不受影响 | 已开启的 OneKeyID 同步继续正常工作，不强制迁移 |

### 开通与开关

| 规则项 | 规则描述 |
|-------|---------|
| 自动开通 | 用户通过任一 Keyless Wallet 创建入口（如 Cloud Sync 页面、Onboarding）创建成功后，自动开启同步 |
| 取消创建 | 创建过程中取消时，页面保持原状态，不自动开启同步 |
| 用户意图优先 | 用户手动关闭同步后，重启 App 仍保持关闭，不自动重开 |
| 开关行为 | 在「已创建 Keyless Wallet，同步关闭」状态打开开关后进入「Keyless Sync 已开启」；关闭开关后返回关闭状态 |
| 自动锁定 | Keyless 同步模式不再限制自动锁定时长，`4 小时` 与 `永不` 选项可用 |



### 同步范围与触发

| 同步类型 | 规则 |
|---------|------|
| HD/HW/QR/隐藏钱包账户名 | 修改同步到其他端，旧名称加入设备历史 |
| 私钥/公钥/观察钱包账户名 | 仅同步对应账户 |
| 空passphrase隐藏钱包 与标准钱包 | 名称分别同步 |
| 自定义代币/网络/RPC、浏览器书签、市场观察列表、地址簿 | 新增/修改同步 |
| 浏览器书签排序 | 排序结果同步到其他端 |
| 自定义 RPC 开关 | 开关状态同步到其他端 |
| 修改即时同步 | 对同步范围内对象执行新增、修改、排序或开关变更后，立即上传云端；在线设备通过 WS 同步最新修改 |
| 异步触发 | 打开账户选择器、地址簿、管理 Token、网络选择器、浏览器首页、Market 首页等，5 分钟间隔，用于补拉最新云端数据 |
| 立即触发 | 云同步开关、断线重连、重新激活 Prime、手动同步无间隔 |
| 手动同步 | 在「Keyless Sync 已开启」状态点击 `Sync now` 时，若内存密码有效则直接进入同步 loading；若内存密码失效，则需先完成密码验证，再进入同步 loading，并在成功后展示 toast |
| 实时同步与内存密码 | 内存密码失效不阻断修改即时同步与在线端 WS 更新；仅「手动同步」在密码失效时须先验证 |

### 冲突与异常

| 场景 | 结果 |
|-----|------|
| 冲突原则 | 同一数据多端修改时，按最后一次修改结果生效 |
| B端关闭同步开关，账户名称：A端 先改 A2，B端 后改 B2 | B端开启同步开关，B2 生效 |
| B端关闭同步开关，通讯地址名称：B端 先改 B3，A端 后改 A3 | B端开启同步开关，A3 生效 |
| 钱包删除兜底 | Keyless Wallet 被删除后再次进入 OneKey Cloud，需进入 `Syncing Paused` 状态并展示恢复按钮 |
| 暂停态同步 | `Syncing Paused` 状态下点击 `Sync now`，显示错误 toast |
| 恢复流程 | 点击恢复按钮，进入 Keyless Wallet 创建/恢复流程 |
| 页面加载 | 进入 OneKey Cloud 页面时，不应先显示错误状态再切换 |

## 4.4 OneKey ID 个人资料规则

| 规则项 | 规则描述 |
|-------|---------|
| 未登录态 | 菜单顶部显示 OneKey ID 注册/登录入口和默认头像 |
| 菜单显示 | 头像、昵称、邮箱、当前登录客户端对应的 Prime 状态图标（正常绿标/超限黄标；无 Prime 不显示） |
| 点击行为 | 点击 Prime 状态图标打开 Prime 弹窗；点击头像打开个人资料编辑；点击其余区域打开 OneKey ID 弹窗 |

| 邮箱脱敏 | 首字母 3 位+***+后缀 |

| 网络异常 | 断开网络修改提示失败 |

## 4.5 Prime 登陆设备限制

| 规则项 | 规则描述 |
|-------|---------|
| 有效设备上限 | 同一 OneKey ID 登录多台设备并已开通 Prime 时，仅 5 台设备可同时生效 Prime 权益 |
| 设备判定 | 设备数未满 5 台时，新登录设备直接生效；满 5 台后新增设备为超限设备 |
| 状态展示 | 标志颜色仅在当前登录客户端对应的 Prime 状态区域展示，不直接展示其他客户端的有效/超限状态 |
| 设备列表 | 按最后活跃时间排序，供展示与补位使用 |
| 活跃刷新 | 访问 OneKey ID 相关功能会刷新最后活跃时间；仅影响排序，不直接改写当前生效设备集合 |
| 移除超限设备 | 不影响当前生效设备集合 |
| 移除未超限设备 | 生效设备减少后，由最近活跃的超限设备补位 |
| 未超限设备登出 | 生效设备减少后，由最近活跃的超限设备补位 |

### 弹窗规则

| 规则项 | 规则描述 |
|-------|---------|
| 网络列表 | 与网络选择器一致；不展示 All networks 和 Lightning network |
| 我的账户 Tab | 无账户显示空状态；有账户按派生路径展示账户数量 |

## 4.8 批量撤销授权规则

### 支持范围

| 支持 | 不支持 |
|-----|--------|
| 软件钱包、无私钥钱包、私钥账户、外部账户、硬件钱包、QRWallet | 观察账户 |

### 功能

| 模式 | 规则 |
|-----|------|
| 逐一取消 | 多笔时按顺序逐个签名提交，可取消当前笔并继续下一笔 |
| 批量撤销 | Prime 专属；软件钱包自动批量；硬件钱包需逐一签名；未开通或已过期时跳转 Prime 权益页 |
| 支持网络 | ethereum, bsc, polygon, arbitrum, avalanche, optimism, base |

### 进度状态

| 状态 | 说明 |
|-----|------|
| 待处理 | 尚未执行 |
| 进行中 | 当前正在撤销 |
| 成功 | 交易已提交并完成 |
| 跳过 | 因预估 Gas Fee 过高等原因跳过，可查看原因 |
| 失败 | 因 Gas 不足、断网、硬件拒绝、通信中断、4XX/5XX 等失败，可查看原因 |
| 已暂停 | 批量进程被用户或系统暂停 |

### 中断规则

| 操作 | 结果 |
|-----|------|
| 关闭弹窗/取消 | 停止未执行交易；已成功交易保留在交易历史 |
| 暂停 | 当前进程变为已暂停，可继续 |
| 继续 | 重新发起当前交易进程 |
| 锁定 App | 自动暂停批量撤销 |
| 解锁 App | 保留当前进程，可继续恢复 |
| 退出 App/杀进程 | 中断批量撤销；已成功交易保留在交易历史 |

### 结果校验

| 规则项 | 规则描述 |
|-------|---------|
| 前台约束 | 批量处理中需保持页面激活，退出页面会暂停进程 |
| 进度汇总 | 已执行/总数、成功数、跳过数需与实际一致 |
| 总 Gas Fee | 完成后显示总 Gas Fee 及法币换算，需与链上结果一致 |
| 详情信息 | 成功项可查看合约地址、交易哈希、复制及区块浏览器跳转 |
| 历史记录 | 成功撤销的交易需出现在交易历史 |

## 4.9 提升交易通知账户上限规则

| 规则项 | 规则描述 |
|-------|---------|
| 默认上限 | 20 个账户 |
| Prime 生效 | 开启 Prime 后上限变更为 100 个账户 |
| Prime 失效/登出 | Prime 失效或登出 OneKey ID 后，上限恢复为 20 个 |
| 超限关闭 | 权限生效期间开启超过 20 个账户通知时，Prime 失效后按开启时间自动关闭后开启的账户通知，保留前 20 个 |
| 再次开启 | 再次开启 Prime 后，自动恢复此前被关闭的账户通知 |

## 4.10 dApp 站点风险扫描缓存规则

| 规则项 | 规则描述 |
|-------|---------|
| Prime 主动请求 | Prime 用户每次打开 dApp URL 时，主动请求 Blockaid Site Scan；不再仅用于复核 |
| 服务端缓存 | Blockaid Site Scan 返回结果缓存于服务端 |
| 缓存复用 | 后续所有用户在缓存有效期内访问相同 dApp URL 时，可复用服务端缓存结果 |
| 原逻辑保留 | 原有 dApp URL 风险检测逻辑保持不变，本能力仅为增量补充 |
| 非 Prime 用户 | 不新增主动请求动作；若命中有效缓存结果可直接使用，否则返回 `unknown` |
| 缓存有效期 | Blockaid Site Scan 缓存有效期为 3 天 |
| 缓存过期结果 | 缓存过期后，与未使用 Blockaid 检查过一致，返回结果为 `unknown` |
| 缓存策略 | 缓存时效固定为 3 天；到期后的刷新和失效策略由服务端控制；测试时需验证命中、过期和未命中三种路径 |
| 异常兜底 | Blockaid 请求失败、超时或缓存不可用时，不应阻断原有 dApp URL 风险检测链路 |

## 4.11 批量转账规则

> 详见需求文档：`docs/qa/requirements/Prime-批量转账.md`
> 测试用例：`docs/qa/testcases/cases/prime/2026-04-14_Prime-批量转账.md`

### 4.11.1 网络与入口

| 规则项 | 规则描述 |
|-------|---------|
| 一对多支持网络 | EVM（ETH/BSC/Arbitrum/Polygon/Base/Optimism/Avalanche/Linea/zkSync Era）+ Tron + Solana + Bitcoin（开发环境含 sBTC） |
| 多对一/多对多支持网络 | 除 Lightning Network / Nostr 外的所有网络（跳过一对多的网络白名单限制） |
| 排除网络 | Lightning Network / Nostr |
| 流程模式 | 一对多 / 多对一 / 多对多 |
| 入口 | 菜单栏「批量转账」/ 钱包首页「更多」→「批量转账」 |

### 4.11.2 网络默认选择

| 规则项 | 规则描述 |
|-------|---------|
| All Networks（一对多） | 默认回退到 ETH 网络（支持列表第一个） |
| 不支持的 EVM 网络（一对多） | 回退到 ETH 网络（EVM 列表第一个） |
| 不支持的非 EVM 网络（一对多） | 回退到 ETH 网络（总列表第一个） |
| 多对一/多对多 | 保留当前网络不做修正（除 Lightning/Nostr 外） |

### 4.11.3 BTC 地址类型联动

| 规则项 | 规则描述 |
|-------|---------|
| 地址类型选择 | BTC 地址输入页提供地址类型选择器（Taproot / SegWit / Legacy 等） |
| 联动更新 | 切换地址类型后，发送账户地址与派生类型同步更新 |

### 4.11.4 输入格式校验

| 规则项 | 规则描述 |
|-------|---------|
| 仅地址模式 | 每行一个地址，进入金额设置页显示「指定金额」和「范围金额」标签页 |
| 地址+金额模式 | 每行地址+逗号+金额，进入金额设置页额外显示「自定义金额」标签页 |
| 格式混用 | 两种格式混用时报错并阻止提交 |
| 重复地址（一对多/多对多） | 警告模式（黄色），弹出确认弹窗可继续 |
| 重复地址（多对一） | 错误模式（红色），硬拦截不可继续 |

### 4.11.5 最小转账金额校验

| 规则项 | 规则描述 |
|-------|---------|
| 输入页校验 | 收款地址输入页粘贴”地址+金额”时执行最小转账金额校验 |
| 金额页校验 | 指定金额、范围金额、逐地址编辑三种模式统一执行最小转账金额校验 |
| 拦截行为 | 金额低于最小值时显示错误提示并阻止提交 |

### 4.11.6 间隔设置

| 规则项 | 规则描述 |
|-------|---------|
| 适用模式 | 仅多对一 / 多对多显示间隔设置页，一对多跳过 |
| 默认值 | 默认选中「指定范围」，最小 5 秒，最大 30 秒 |
| 最大间隔 | 600 秒 |
| 校验规则 | 最小值 ≥ 最大值时报错；值 > 600 秒时报错 |

### 4.11.7 构建与发送

| 规则项 | 规则描述 |
|-------|---------|
| EVM/Tron 代币 | 走合约批量调用，需先发送代币授权交易（授权额度不足时） |
| BTC/SOL 原生 | 使用原生批量构建多笔 unsignedTx，不走 approve 流程 |
| BTC 拆分上限 | 最多 200 个输出/笔，超出自动拆分 |
| SOL 拆分方式 | 按指令序列化大小自动拆分（trial serialization） |
| SOL ATA 租金 | 每个 ATA 约 0.00203928 SOL，批量查询每次 100 个地址 |
| 授权轮询 | 授权成功后轮询确认，5 秒间隔，30 秒超时 |
| Tron 逐笔发送 | 多对一/多对多模式下 Tron 逐笔发送，300 毫秒间隔 |

### 4.11.8 BTC 签名前刷新

| 规则项 | 规则描述 |
|-------|---------|
| 刷新时机 | 每笔 BTC 交易签名前刷新 unsignedTx |
| 刷新内容 | 重取 UTXO 并保留费率参数 |

### 4.11.9 SOL 复核页费用与拦截

### 4.11.10 拆分交易费用说明

### 4.11.11 执行状态

## 4.13 Dapp 页面 AI 翻译规则

> 详见需求文档：`docs/qa/requirements/Browser-DApp浏览器新增翻译设置入口.md`
> 测试用例：`docs/qa/testcases/cases/prime/2026-03-30_Prime-DApp浏览器翻译设置.md`

### 权益与访问控制

| 规则项 | 规则描述 |
|-------|---------|
| Prime 专属 | DApp 翻译为 Prime 专属功能，非 Prime 用户点击翻译入口弹出 Prime 订阅引导弹窗，无法执行翻译 |
| Prime 过期 | 过期用户行为与非 Prime 用户一致 |
| 功能开关 | 翻译能力通过 PrimeFeatures（DAppTranslate）功能开关控制 |

### 入口位置

| 平台 | 入口 | 位置 |
|-----|------|------|
| Desktop | 翻译图标 | 地址栏输入框右侧（与书签、Pin 图标并列） |
| iOS | 翻译按钮 | 底栏第 5 个按钮 |
| Android | 翻译按钮 | 底栏，支持 RNGH 手势点击（GestureDetector 包裹） |

### Popover 两层视图

| 视图 | 内容 | 切换方式 |
|-----|------|---------|
| 动作视图（默认） | 目标语言文本 + 「翻译」/「恢复原文」按钮 + 设置齿轮图标 | 点击齿轮图标 → 设置视图 |
| 设置视图 | 引擎、目标语言、显示模式选项 | 点击返回箭头 → 动作视图 |

### 翻译设置选项与默认值

| 设置项 | 选项 | 默认值 |
|-------|------|--------|
| 引擎 | AI / 传统（SegmentControl 切换） | AI |
| 目标语言 | 自动 / 自定义（自定义时出现语言选择下拉框，列表不包含 system 和 en-US） | 自动 |
| 显示模式 | 双语对照 / 替换（SegmentControl 切换） | 双语对照 |

### 翻译状态与图标

| 状态 | 图标 | 说明 |
|-----|------|------|
| 未翻译 | TranslateOutline（空心） | 点击执行翻译 |
| 已翻译 | TranslateSolid（实心） | 点击可恢复原文 |

### 配置持久化

| 规则项 | 规则描述 |
|-------|---------|
| 持久化方式 | 通过 jotai persist atom 持久化 |
| 持久化范围 | 引擎、目标语言、显示模式三项配置 |
| 生效行为 | 退出并重新进入 DApp 浏览器后，配置保持上次选择，不回退为默认值 |

### 页面导航与边界

| 规则项 | 规则描述 |
|-------|---------|
| 导航重置 | 翻译后跳转新页面（页内链接或地址栏输入新 URL），翻译状态自动清除，图标恢复空心，新页面显示原文 |
| 首页边界 | 浏览器首页（未加载 DApp）时翻译按钮置灰或不可点击 |
