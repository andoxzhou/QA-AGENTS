# Account 模块测试规则文档

> 本文档记录账户模型模块的核心测试规则。
> 生成账户模型相关测试用例时，必须参考本文档。

---

## 1. 账户类型规则

### 1.1 支持的账户类型

| 账户类型 | 英文名称 | 说明 | 能力限制 |
|---------|---------|------|---------|
| HD 钱包 | HD Wallet | 通过助记词派生的钱包 | 完整功能 |
| 硬件钱包 | Hardware Wallet | 通过硬件设备管理的钱包 | 需设备确认 |
| 观察账户 | Watched Only Account | 仅查看余额，无私钥 | 不支持签名/转账 |
| 外部账户 | External Account | 第三方钱包导入 | 受账户系列限制 |
| 私钥账户 | Private Key Account | 通过私钥导入 | 受账户系列限制 |
| 无私钥钱包 | Keyless Wallet | 通过社交账号 + PIN + App Passcode 创建/恢复 | 不支持导出助记词/KeyTag/Lite，不参与云备份/传输 |

> 二维码钱包（QR Wallet / QR Hidden Wallet）按硬件钱包连接形态归入硬件钱包规则。

### 1.2 账户能力矩阵

| 功能 | HD 钱包 | 硬件钱包 | 观察账户 | 外部账户 | 私钥账户 | 无私钥钱包 |
|-----|--------|---------|---------|---------|---------|-----------|
| 查看余额 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| 转账 | ✓ | ✓ | ✗ | ✓ | ✓ | ✓ |
| Swap | ✓ | ✓ | ✗ | ✓ | ✓ | ✓ |
| DApp 签名 | ✓ | ✓ | ✗ | ✓ | ✓ | ✓ |
| 导出私钥 | ✓ | ✗ | ✗ | ✗ | ✓ | ✓ |
| 导出助记词 | ✓ | ✗ | ✗ | ✗ | ✗ | ✗ |

## 2. 账户派生规则

### 2.1 BIP 标准支持

| BIP 标准 | 路径格式 | 适用链 | 地址类型 |
|---------|---------|-------|---------|
| BIP44 | m/44'/coin'/0'/0/index | 所有链 | Legacy |
| BIP49 | m/49'/coin'/0'/0/index | BTC | P2SH-SegWit |
| BIP84 | m/84'/coin'/0'/0/index | BTC | Native SegWit |
| BIP86 | m/86'/coin'/0'/0/index | BTC | Taproot |

### 2.2 派生一致性规则

- **同助记词派生**：同一助记词在不同设备/平台派生的地址必须一致
- **多账户派生**：支持派生多个账户（index 递增）
- **链间隔离**：不同链的派生路径相互独立
- **同助记词账户关联**：同助记词导入的软件/硬件/QR 钱包中，对应每个账户头像一致

### 2.3 批量创建规则

| 场景 | 规则 |
|-----|------|
| 管理账户（地址预览模式） | 展示地址预览页面，每页显示 10 个地址/余额，支持翻页 |
| 勾选创建 | 勾选默认未选中的地址后，确认时创建对应账户 |
| 取消勾选移除 | 取消勾选默认选中的地址后，确认时移除对应账户 |
| 切换网络保护 | 因切换网络导致的默认选中账户被取消时，不移除已有账户 |
| 确认按钮 | 显示待创建账户数量 |
| 高级模式 | 支持无地址预览直接批量创建，可设置生成数量、起始账户和派生路径 |
| 页码范围 | 输入 0 / 负数自动修正为 1；最大页码为 214748365 |
| 起始账户范围 | 起始序号最小为 1，最大为 2147483634 |
| 生成数量范围 | 单次最小 1、最大 100；最大地址序号2147483648，超出剩余可创建范围时自动修正为可用最大值 |
| 跳号创建 | 支持从指定起始账户连续创建，例如直接生成 18-20 号账户 |
| 进度反馈 | 创建过程中显示进度条和地址总数，完成后提示创建数量并返回钱包管理 |

### 2.4 自动创建与路径切换

| 场景 | 规则 |
|-----|------|
| All networks 添加账户 | 默认创建 BTC 4 种 / EVM-BIP44 / Solana-BIP44 / Tron 地址 |
| 指定网络添加账户 | 在默认地址基础上额外创建当前选中网络地址 |
| 不支持网络 | 当前钱包/设备不支持所选网络时，默认地址仍可创建，并提示所选网络地址创建失败 |
| 聚合网络 | BTC / LTC 自动创建所有派生路径地址 |
| 多派生网络 | EVM / Solana / Kaspa 自动创建当前派生路径地址，与默认创建地址同时生效 |
| 系列网络 | Cosmos / DOT 自动创建对应系列所有地址 |
| DOT 限制 | 15 / 18 / 21 位助记词软件钱包不支持创建 DOT 地址，仅 12 / 24 位支持 |

## 3. 助记词与软件钱包规则

### 3.1 创建流程

| 步骤 | 规则 | 验证点 |
|-----|------|-------|
| 生成 | 新建软件钱包生成 12 词助记词 | 词数正确 |
| 创建完成 | 生成新的软件钱包时不展示备份选择，正常创建完成后直接进入钱包首页 | 钱包标记为未备份钱包 |

### 3.2 导入流程

| 场景 | 规则 | 错误提示 / 结果 |
|-----|------|---------------|
| 手动输入 | 输入字符联想 BIP39 单词，点击联想后自动跳到下一格；非焦点输入框显示 4 位暗码 | - |
| 粘贴导入 | 助记词按空格自动分发到各输入框，粘贴后清空剪贴板 | - |
| clear / 切换位数 | 点击 `clear` 清空已输入助记词；切换助记词位数时清空当前输入 | - |
| 实际校验 | 导入以实际输入的合法助记词为准，不仅依赖位数选择器 | 合法助记词可继续导入 |
| 格式错误 | 非 BIP39 词库、非英文字符或无效单词 | "无效助记词" |
| 词数错误 | 实际输入非 12/15/18/21/24 词 | "无效助记词" |
| 校验和错误 | 助记词校验失败 | "无效助记词" |
| 重复导入 | 已存在相同钱包 | 定位到对应钱包并提示重复 |
| TON 助记词 | 24 位 TON 助记词导入为 TON 单链私钥钱包，不创建 HD 软件钱包 | 地址与预期一致 |

### 3.3 安全规则

- **截屏限制**：助记词页面禁止截屏
- **剪贴板清理**：助记词输入粘贴后自动清空剪贴板
- **掩码显示**：已录入助记词的非焦点输入框统一显示 4 位暗码
- **二次验证**：助记词相关的查看/备份/导出流程需密码或生物识别确认，具体步骤按对应备份章节执行
- **风险确认**：助记词复制或导出类流程需明确风险确认，具体交互按手动备份 / KeyTag / Lite 等章节执行

### 3.4 未备份钱包规则

| 场景 | 规则 |
|-----|------|
| 首页状态 | 不展示账户余额，显示备份提示、绑定邀请码、教程/客服入口 |
| 基础使用 | 切换 tab、创建账户/地址不受影响 |
| 受限能力 | 复制地址、理财认购、导出公钥/私钥、DApp 签名等操作弹出钱包备份提示，可取消 |
| 账户选择 | 发送代币 / NFT 的“我的账户”列表中隐藏未备份钱包 |
| 备份入口优先级 | 优先展示 iCloud / Google Drive，其次助记词；“更多”展示助记词 / KeyTag / Lite（移动端） |
| 状态变更 | 完成备份后移除未备份标记；数据传输不会移除未备份标记 |

### 3.5 手动备份规则

| 规则项 | 规则描述 |
|-------|---------|
| 入口 | 设置 / 账户选择器 / 首页备份提示均可进入；TON 助记词私钥钱包从“账户 -> 更多 -> 备份助记词”进入 |
| 验证 | 先验证 App 密码或生物识别，失败中断流程 |
| 展示 | 顶部显示钱包或账户名称；助记词明文展示且每词带序号 |
| 复制 | 点击复制需先提示风险，复制后剪贴板内容与助记词一致 |
| 校验 | 未备份新钱包在展示助记词后需进入随机词校验；导入钱包或已备份钱包再次查看不强制校验 |
| 重新进入 | 返回展示页后再次进入校验页，会刷新用于验证的指定单词 |

### 3.6 云备份（iCloud / Google Drive）

| 规则项 | 规则描述 |
|-------|---------|
| 备份范围 | 展示当前 App 内所有 HD 钱包和私钥 / 公钥 / 观察账户；不包含无私钥钱包 |
| 账号获取 | Google Drive 需登录/选择 Google 账号；iCloud 使用系统账号 |
| 首次备份 | 必须设置备份密码，两次一致且长度不少于 6 位 |
| 再次备份 | 必须验证已设置的备份密码 |
| 导入列表 | 新版格式数据列表展示备份时间、钱包账户数量；旧格式数据导入口位于新版格式数据列表下方 |
| 新版导入 | 校验备份密码后下载所选钱包，再设置当前 App 密码；当前 App 密码可与原备份客户端不同 |
| 旧版导入 | 兼容旧格式备份数据，允许从新版格式数据列表下方的旧数据入口发起导入 |
| 数据完整性 | 恢复后除 HD 钱包 ADA / LN 外，其余数据应完整一致 |
| 账号切换 | 切换 Google 账号后，备份列表随账号变化 |
| 删除备份 | 删除前需二次确认并验证 App 密码 |
| 重置备份密码 | 只影响新备份；旧密码创建的备份仍需旧密码导入 |

### 3.7 OneKey Lite 备份

| 规则项 | 规则描述 |
|-------|---------|
| NFC 权限 | 首次进入备份 / 导入 / 修改 PIN / 重置时申请 NFC 权限；关闭 NFC 时引导开启；设备不支持 NFC 时明确提示 |
| 备份能力 | 支持将 12 / 15 / 18 / 21 / 24 位助记词钱包备份到空卡，导入后首地址与原钱包一致 |
| 覆盖备份 | 新助记词可覆盖旧备份，覆盖后以最新备份为准 |
| PIN 规则 | 需设置并二次确认 6 位 PIN；两次不一致时确认按钮置灰 |
| PIN 验证 | 覆盖备份 / 导入 / 修改 PIN 流程都需先验证 Lite PIN 码，通过后才可继续 |
| PIN 错误 | 输入错误会递减剩余次数，累计 10 次错误后卡片重置 |
| 重置确认 | 重置流程需先输入 `reset` 后才能发起 |
| 空卡状态 | 导入或修改 PIN 时若卡中无备份，需明确提示 |
| 识别异常 | 贴卡取消、移开卡片、两次识别为不同卡、空卡等场景均提示连接或卡片错误 |
| 卡片识别 | 非 Lite NFC 卡不响应，不进入 Lite 备份流程 |

### 3.8 OneKey KeyTag

| 规则项 | 规则描述 |
|-------|---------|
| 入口 | 设置页显示“购买 / 备份 / 导入”入口，购买跳转官网页面 |
| 导入 | 助记词输入框规则与助记词导入一致，输入框下方展示 dotmap 助记词转换引导和 GitHub 链接 |
| 备份方式 | 支持选择现有软件钱包备份，也支持手动输入 12 / 15 / 18 / 21 / 24 位助记词生成 dotmap |
| 权限 | 选择现有钱包备份需生物识别或密码验证 |
| 教程 | 右上角帮助展示备份教程，勾选“我已备份”后才可关闭弹窗 |
| 核对 | dotmap 点位与行数需与助记词一一对应 |

### 3.9 数据传输

| 规则项 | 规则描述 |
|-------|---------|
| 建立连接 | 默认选中二维码 tab，二维码与配对码一致；默认连接 OneKey 官方服务器 |
| 配对码输入 | 支持粘贴/扫码录入，默认暗文显示，可切换明文 |
| 配对校验 | 不可连接自己的配对码；无效码、超时、一端退出、第三端抢占均需提示并中断 |
| 自定义服务器 | 支持域名、`http/https`、IP+端口；可自动补全 `http/https` 并记录历史 |
| 连接状态 | 状态灯按连接中 / 已连接 / 连接失败区分展示 |
| 迁移验证 | 导出方输入验证码并验证 App 密码后发送数据；导入方展示预览页 |
| 数据范围 | 支持按钱包分类导出/导入 软件钱包/私钥钱包/观察钱包；10 钱包 每个钱包200+ 地址场景数据需完整；不包含软件钱包/私钥钱包/观察钱包时提示无数据 |
| 传输限制 | 无私钥钱包不参与数据传输；HD 钱包 LN / ADA 数据不导入 |

### 3.10 无私钥钱包

| 规则项 | 规则描述 |
|-------|---------|
| 创建方式 | 通过 Google / Apple 登录 + 4 位数字 PIN + App Passcode 创建 |
| 单实例 | 当前设备已有无私钥钱包时不可再次启用；同一社交账号跨设备同时创建仅允许一个成功 |
| 恢复 | 恢复流程与创建一致，但 PIN 阶段变为验证；恢复后的地址需与创建地址一致 |
| PIN 规则 | 仅支持数字，长度固定 4 位；两次输入不一致不可继续 |
| 错误与冷却 | PIN 最多错误 7 次，冷却为 0 / 0.5 / 1 / 2 / 5 / 5 分钟后锁定 |
| 会话与重置 | PIN 输入页停留 8 分钟后会话过期；忘记或重置 PIN 需先验证 Passcode |
| 定期验证 | 创建后按 12 小时 -> 1 天 -> 3 天 -> 7 天 -> 14 天 -> 每月一次 触发 PIN 验证提醒 |
| 能力与限制 | 能力与软件钱包基本一致。但是不支持导出助记词 / KeyTag / Lite；不参与云备份、OneKey Cloud 同步、数据传输 |
| 同邮箱双渠道 | 相同邮箱分别通过 Apple / Google 注册后，两个无私钥账号都可正常使用；账户图标正确，PIN 验证与 PIN 重置流程均正常 |
| 账号校验 | 切换社交账号后发起 PIN 验证或重置时，需提示账号与钱包不匹配 |

### 3.10.1 Web / 插件 onboarding

| 场景 | 规则 |
|-----|------|
| Web 登录入口 | Web 端展示 `Google` 和 `Apple` 登录按钮，用户选择的登录方式需透传到插件侧 |
| 插件未安装 | 点击 `Google` / `Apple` 登录时若插件未安装，提示用户安装插件 |
| 首次安装判定 | 仅“卸载后重新安装”视为首次安装；禁用/启用或重载插件不应触发首次安装流 |
| 登录方式透传 | Web 端点击 `Google` / `Apple` 登录后，插件侧拉起的 Keyless 登录方式需与 Web 端选择一致 |
| 首次安装后拉起方式 | 插件首次安装成功后，因无用户手势触发限制，需自动进入大屏 onboarding，而不是 side panel |
| 已安装无钱包 | 插件已安装但无 Keyless 钱包时，需通过 Side Panel 完成 onboarding 和 Keyless 创建 |
| 已安装有钱包 | 插件已安装且已有 Keyless 钱包时，不重复进入创建流程，直接进入默认连接路径 |
| 创建后衔接 | 在 Web / 插件 onboarding 中完成 Keyless 创建后，需继续弹出连接窗以完成网页端连接闭环 |
| Web 自动回流 | 用户完成连接确认后，Web 端刷新并自动连接该钱包 |

## 4. 私钥管理规则

### 4.1 私钥格式支持

| 链类型 | 私钥格式 | 长度 |
|-------|---------|-----|
| EVM | 0x 开头十六进制 | 64 字符（不含 0x） |
| BTC | WIF 格式 | 51-52 字符 |
| Solana | Base58 | 87-88 字符 |

### 4.2 导入规则

- **格式校验**：自动识别私钥格式
- **链匹配**：私钥格式必须与目标链匹配
- **重复检测**：导入已存在的私钥定位到对应账户并提示"该钱包已存在"

### 4.3 导出规则

- **权限验证**：需密码/生物识别确认
- **安全提示**：显示风险警告
- **一次性显示**：退出后需重新验证才能再次显示

### 4.4 私钥导出/导入细则

| 规则项 | 规则描述 |
|-------|---------|
| 网络/派生路径 | 可选，默认与钱包首页选择一致 |
| 未创建地址 | 选择尚未创建地址的网络/路径时，私钥栏为空，提示该账户还没有创建地址 |
| 已创建地址 | 私钥栏掩码显示，可切换可见，可复制 |
| 复制确认 | 用户勾选确认信息前无法确认复制 |
| 导入预测 | 根据输入字符预测匹配网络 |
| 找不到网络 | 点击「找不到想要的网络」可展开所有网络列表选择 |
| BTC/fork 链 | 导入私钥的 BTC 及 fork 链账户可以导出对应公钥 |

### 4.5 公钥导出规则

| 规则项 | 规则描述 |
|-------|---------|
| 入口 | 账户选择器 → 第一个账户三个点 → 查看详情 → 导出公钥 |
| 权限 | 私钥/HD钱包导出需验证密码，公钥观察账户导出不需要验证密码 |
| 适用 | BTC 及 fork 链的HD/HW/私钥/公钥观察账户 |

## 5. 外部账户规则

### 5.1 WalletConnect 连接

| 规则项 | 规则描述 |
|-------|---------|
| 流程 | onboarding → 点击 WalletConnect → 弹出WalletConnect弹窗 → 其他设备外部钱包扫码/搜索本机外部钱包 → 在外部钱包客户端确认连接 → 创建外部钱包账户 |
| 账户命名 | 外部钱包名称 + 序号 |
| 显示校验 | 账户连接钱包图标、网络图标、地址正确 |

## 6. 硬件钱包规则

### 6.1 设备与连接方式

| 设备 | 支持连接方式 | 端限制 |
|-----|-------------|-------|
| Pro | USB / 蓝牙 / 二维码 | 全端 |
| Touch | USB / 蓝牙 | 全端 |
| Classic / 1S | USB / 蓝牙 | 全端 |
| Mini | USB | 移动端不显示 Mini，不显示蓝牙分类 |

| 端 | 展示分类 |
|---|---------|
| 桌面端（除Linux） | USB / 蓝牙 / 二维码 |
| 移动端 | 蓝牙 / 二维码 |
| 插件端/Linux桌面端 | USB / 二维码 |

### 6.2 连接流程

| 规则项 | 规则描述 |
|-------|---------|
| USB | 直接展示连接中的设备列表 |
| 蓝牙 | 直接开始搜索并展示可连接的设备 |
| 停留 10 秒 | 展示「设备连接有问题吗？」按钮，点击打开建议解决方案 |
| 拔掉 USB | 设备信息清除 |
| 关闭蓝牙 | 提示蓝牙关闭和跳转按钮 |
| 固件升级（强制条件） | 系统固件或 BLE 固件 major 版本落后，或同 major 版本下 minor 差值大于 2，或同 major + minor 下 patch 差值大于 2 时，不显示「跳过」按钮，仅保留更新入口 |
| 固件升级（可跳过条件） | 不满足强制条件时允许跳过；桌面端蓝牙连接升级时提示插入 USB 线 |
| 固件检查异常 | 固件检查异常（如设备断开连接）时显示错误状态，并保留「重试」和「跳过」 |

### 6.3 验真与 PIN

| 场景 | 规则 |
|-----|------|
| 验真取消 | 硬件上点击「取消」→ 回到 Onboarding 页面，提示操作取消 |
| 验真失败 | 设备验真不通过，App 弹窗提示设备异常，提供 Contact us 跳转 |
| 验真中断 | 断网/断开设备连接（拔出usb线，关闭蓝牙，关闭设备） → 提示硬件通信失败，恢复后可重试 |
| 跳过验真 | 验真中断后选择仍要继续 → 跳过验真，创建钱包后在设备信息中标识未验真 |
| PIN 错误（Classic/Mini/1S） | App 端提示 PIN 码错误 |
| PIN 错误（Pro/Touch） | 10 次错误后设备被重置，再次连接进入激活页面 |

### 6.4 Passphrase 与隐藏钱包

| 设备状态 | 创建选项 |
|---------|---------|
| 未开启 passphrase，未创建过标准钱包 | 直接创建标准钱包 |
| 开启 passphrase，未创建过标准钱包 | 显示创建标准钱包/隐藏钱包选项 |
| 开启 passphrase，已创建过标准钱包 | 直接提示输入 Passphrase 创建隐藏钱包 |
| 开启 passphrase，重置前创建过标准钱包 | 显示创建标准钱包/隐藏钱包选项 |

| 场景 | 规则 |
|-----|------|
| 未开启 passphrase | 无法创建隐藏钱包，需先启用；启用后钱包选择器显示快捷添加选项 |
| 未开启时操作 | 核对地址/创建地址/转账/swap 均提示打开 passphrase |
| 快捷添加按钮 | 标准钱包+已开启 passphrase 或 有隐藏钱包时显示；二维码钱包视为未开启 passphrase |

### 6.5 重复创建规则

| 场景 | 规则 |
|-----|------|
| 相同设备+相同助记词 | 创建并更新覆盖旧钱包，已验证则不需再次验证 |
| 不同设备+相同助记词 | 创建新钱包 |
| 相同设备+重置使用不同助记词 | 创建新钱包，旧钱包标记废弃 |
| 相同设备+重置使用相同助记词，新连接首先创建隐藏钱包 或 旧钱包没有标准钱包 | 创建新钱包，旧钱包标记废弃 |
| 相同设备+重置使用相同助记词，新连接首先创建标准钱包 且 旧钱包有标准钱包 | 绑定旧钱包，去掉废弃标记 |


### 6.6 Passphrase 保持连接

| 规则项 | 规则描述 |
|-------|---------|
| 默认 | 创建隐藏钱包时「保持连接」开关默认开启 |
| 关闭后重启 | 关闭「保持连接」的隐藏钱包在重启 APP 后被隐藏，从新创建后保留之前的账户创建和修改 |
| 相同 passphrase | 提示创建相同账户，定位到隐藏钱包并根据创建时选择更新开关状态 |

### 6.7 Passphrase 输入规则

| 场景 | 规则 |
|-----|------|
| 不输入 | APP禁止不输入，可切换到设备上操作，创建成功与标准钱包地址一致 |
| 中文/emoji | 提示不支持字符 |
| 英文特殊字符 @#$%^&*()_+ | 创建成功 |
| 空格 | 创建成功 |
| 超长（>50 字符） | 提示输入过长 |
| 过期 session | 输入错误 passphrase 提示不匹配，正确可完成操作 |

## 9. 钱包账户管理规则

### 9.1 创建钱包

| 规则项 | 规则描述 |
|-------|---------|
| 名称 | 支持自定义钱包名称，默认“钱包 N” |
| 头像 | 由系统在创建时生成；**创建后不支持修改** |
| 密码 | 创建或导入钱包时需设置 App 密码 |
| 备份状态 | 新建软件钱包创建后的备份状态与提醒规则，按 `3.1 创建流程` 和 `3.4 未备份钱包规则` 执行 |
| 重复创建 | 重复添加软件钱包时，按 `3.2 导入流程` 的重复导入规则处理 |

### 9.2 切换钱包/账户

| 场景 | 规则 |
|-----|------|
| 切换钱包 | 资产页面刷新，显示新钱包资产 |
| 切换账户 | 同钱包内切换派生账户 |
| 切换链 | 保持当前账户，切换显示链 |
| All networks 余额 | 显示 All networks 已加载资产；批量新建未回首页前余额显示 `--` |
| 单链余额 | 仅显示当前链资产；资产变化后同步刷新 |
| 地址展示 | 单链 / 观察 / 外部账户展示短地址；钱包账户在 All networks / BTC / LTC / 闪电网络下不展示地址 |

### 9.3 删除/重置

| 操作 | 规则 | 安全措施 |
|-----|------|---------|
| 删除软件/硬件/隐藏钱包账户 | 删除后账户从列表消失且不可恢复；同一次 App 启动仅首次展示移除确认 | 警告提示 |
| 删除单链账户 | 始终弹出移除确认；移除全部私钥账户后隐藏左侧私钥分组图标 | 二次确认 |
| 删除观察/外部账户 | 直接删除，不弹移除确认；移除全部后隐藏对应分组图标 | 直接删除 |
| QR / QR 隐藏钱包账户 | 不允许直接删除单个账户 | 禁用删除 |
| 删除钱包 | 删除整个钱包（含所有账户） | 需勾选确认并验证内存密码 |
| 删除硬件标准钱包 | 删除标准钱包时不连带删除隐藏钱包，标准钱包位显示为空并可重新创建 | 警告提示 |
| 删除硬件隐藏钱包 | 删除隐藏钱包不删除设备本身 | 警告提示 |
| 移除硬件设备 | 删除设备时连带删除其隐藏钱包 | 警告提示 |
| 重置 App | 清除所有数据 | 多重确认 |

### 9.4 账户展示与命名规则

| 规则项 | 规则描述 |
|-------|---------|
| 导入预设名 | 导入观察账户时可预设账户名；留空时使用默认名称 |
| 长度限制 | 名称最大 80 字符；超长输入不可继续，超长粘贴自动截断 |
| 唯一性 | 同一钱包内账户名不可重复 |
| 修改名称 | 重命名不能为空；成功后记录去重后的名称历史 |
| 同助记词名称同步 | xfp 数据补齐后，同助记词软/硬/QR 钱包的同账户名称保持同步 |
| 搜索 | 支持完全匹配和部分匹配；有结果时隐藏添加账户按钮；搜索结果中的复制地址、重命名、置顶、导出公/私钥、移除均可正常执行 |

### 9.5 地址复制 / 接收规则

| 场景 | 规则 |
|-----|------|
| All networks | 点击复制地址 / 接收需先选择具体网络/代币 |
| 聚合网络复制 | BTC / LTC 聚合网络复制当前选中派生路径，提示需包含网络与派生路径 |
| 软件钱包接收 | 直接显示二维码、地址字符串、网络名称；复制无需校验 |
| 硬件钱包接收 | 默认不显示二维码和地址字符串；需设备校验后显示，或选择“设备不在我身边”后以未验证状态展示 |
| 多代币单链 | 非 BTC fork 网络点击从其他钱包接收需先选择代币，再展示接收弹窗 |
| 闪电网络 | 点击从其他钱包接收进入“创建发票” |
| 观察账户 | 支持复制地址，不提供接收入口 |
| 校验差异 | 软件钱包 / 单链账户 / 观察账户 / 外部账户复制或接收不需要校验；硬件 / QR / 隐藏钱包需要校验 |

### 9.6 xfp 数据补充规则

| 规则项 | 规则描述 |
|-------|---------|
| 软件 / QR 钱包 | 解锁 App 并完成密码验证后自动静默补充 xfp，不弹更新提示 |
| 未解锁场景 | 未输入密码时不补充 xfp；在修改名称等需验证密码的操作后自动补充 |
| 新建钱包 | 新建软件 / QR / 硬件 / 隐藏钱包默认已带 xfp |
| 硬件钱包 | 钱包首页和修改名称时提示补充钱包数据；添加账户 / 批量添加账户时静默补充 |
| 废弃钱包 | 废弃钱包可直接修改名称，不提示补充数据 |
| 名称同步 | 同账户名称同步依赖 xfp 数据，补齐后按 `9.4 账户展示与命名规则` 执行 |

## 10. 通用规则

### 状态机
- 创建/导入 → 备份校验 → 解锁/锁定 → 切链/切账户 → 恢复

### 资金一致性
- 资产页/详情页/交易记录三处一致
- 隐藏资产功能不影响实际余额

### 权限/安全
- 助记词/私钥页截屏限制
- 剪贴板清除
- 导出权限
- biometric 失败分支处理

### 链上不确定性
- 多 RPC 切换
- 链切换中断恢复
- 地址派生一致性

### 风控
- 高风险钱包提示
- 观察钱包 vs 冷钱包能力差异限制

### 可观测
- 错误码、导入失败原因可解释（格式/校验和/路径）
