# Account - 硬件钱包连接与隐藏钱包

> 模块：Account  
> 来源：基于 `docs/qa/rules/account-rules.md` 补充整理  
> App 版本：待定  
> 日期：2026-04-14  
> 测试端：Desktop / Extension / iOS / Android

---

## 1. 需求背景

硬件钱包场景覆盖多设备、多连接方式、验真、PIN、Passphrase、隐藏钱包和重复创建逻辑，是 Account 模块中交互链路最复杂的区域之一，需要单独沉淀规则。

---

## 2. 功能描述

### 2.1 设备与连接方式

| 项目 | 说明 |
|-----|------|
| 支持设备 | Pro、Touch、Classic / 1S、Mini |
| 连接方式 | USB、蓝牙、二维码按设备和端能力区分 |
| 端限制 | 桌面端、移动端、插件端、Linux 桌面端显示的连接分类不同 |

### 2.2 连接流程与验真

| 项目 | 说明 |
|-----|------|
| USB | 直接展示连接中的设备列表 |
| 蓝牙 | 直接开始搜索并展示可连接设备 |
| 连接停留 | 停留 10 秒后显示 `设备连接有问题吗？` 入口 |
| 断连处理 | 拔掉 USB、关闭蓝牙、关闭设备后清除或中断当前连接状态 |
| 固件升级（强制条件） | 系统固件或 BLE 固件 major 版本落后，或同 major 版本下 minor 差值大于 2，或同 major + minor 下 patch 差值大于 2 时，不显示「跳过」按钮，仅保留更新入口 |
| 固件升级（可跳过条件） | 不满足强制条件时允许跳过；桌面端蓝牙连接升级时提示插入 USB 线 |
| 固件检查异常 | 固件检查异常（如设备断开连接）时显示错误状态，并保留「重试」和「跳过」 |
| 验真取消 / 失败 / 中断 | 分别回到 Onboarding、显示设备异常弹窗，或显示通信失败提示 |
| 跳过验真 | 验真中断后仍可继续创建，并在设备信息中标记未验真 |

### 2.3 PIN 与隐藏钱包

| 项目 | 说明 |
|-----|------|
| PIN 错误 | Classic / Mini / 1S 在 App 侧提示 PIN 错误；Pro / Touch 错误 10 次后设备重置 |
| Passphrase 开关 | 未开启时不能创建隐藏钱包 |
| 创建分流 | 根据是否开启 passphrase、是否已创建标准钱包决定创建标准钱包或隐藏钱包路径 |
| 快捷添加 | 标准钱包且已开启 passphrase，或已有隐藏钱包时显示快捷添加入口 |

### 2.4 重复创建与保持连接

| 项目 | 说明 |
|-----|------|
| 相同设备 + 相同助记词 | 绑定旧钱包并覆盖更新 |
| 不同设备 + 相同助记词 | 创建新钱包 |
| 重置后不同助记词 | 创建新钱包，旧钱包标记废弃 |
| 保持连接 | 创建隐藏钱包时默认开启；关闭后重启 App 需隐藏该钱包 |
| 相同 passphrase | 提示创建相同账户，定位到原隐藏钱包并更新保持连接状态 |
| Passphrase 输入 | 不输入、中文、emoji、超长都按规则提示；特殊字符和空格按规则支持 |

## 3. 业务规则

| 规则项 | 规则描述 |
|-------|---------|
| 端能力隔离 | 各端连接方式和设备分类按端限制显示 |
| 验真闭环 | 验真取消、失败、中断和跳过后的页面表现必须可区分 |
| PIN 机型差异 | 不同硬件设备的 PIN 错误处理口径不同 |
| 隐藏钱包前置 | 未开启 passphrase 时不能直接创建隐藏钱包 |
| 重复创建策略 | 相同设备、不同设备、设备重置和相同 passphrase 的创建结果必须稳定 |
| 保持连接状态 | 隐藏钱包的保持连接状态会影响重启后的可见性 |

---

## 4. 已知风险

- 多设备与多端连接方式组合较多，容易出现列表展示、搜索状态和连接提示不一致。
- 验真、固件升级、PIN、Passphrase 和隐藏钱包创建串联后，状态恢复复杂。
- 同设备重置、相同助记词和相同 passphrase 三类重复创建场景容易相互干扰。
