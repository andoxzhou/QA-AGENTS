---

## 1. 需求背景

Keyless Wallet 是 Account 模块中的无私钥账户体系，通过社交登录、PIN 和 App Passcode 完成创建与恢复。该能力与软件钱包相近，但在导出、备份、同步和数据传输上存在独立限制，需要与 Web / 插件 onboarding 口径一起完整整理。

---

## 2. 功能描述

### 2.1 创建与恢复

| 项目 | 说明 |
|-----|------|
| 创建方式 | 通过 Google / Apple 登录 + 4 位数字 PIN + App Passcode 创建 |
| 单实例 | 当前设备已有 Keyless Wallet 时不可再次启用；同一社交账号跨设备同时创建仅允许一个成功 |
| 恢复 | 恢复流程与创建一致，但 PIN 阶段变为验证 |
| 地址一致性 | 恢复后的地址需与创建地址一致 |

### 2.2 PIN 规则

| 项目 | 说明 |
|-----|------|
| 输入规则 | 仅支持数字，长度固定 4 位 |
| 二次确认 | 两次 PIN 不一致不可继续 |
| 错误与冷却 | PIN 最多错误 7 次，冷却时间按 0 / 0.5 / 1 / 2 / 5 / 5 分钟递增 |
| 会话与重置 | PIN 输入页停留 8 分钟后会话过期；忘记或重置 PIN 需先验证 Passcode |
| 定期验证 | 创建后按 12 小时、1 天、3 天、7 天、14 天、每月一次触发 PIN 验证提醒 |

### 2.3 能力与限制

| 项目 | 说明 |
|-----|------|
| 账户能力 | 与软件钱包能力基本一致 |
| 不支持项 | 不支持导出助记词、KeyTag、Lite |
| 参与限制 | 不参与云备份、OneKey Cloud 同步和数据传输 |

### 2.4 账号边界

| 项目 | 说明 |
|-----|------|
| 同邮箱双渠道 | 相同邮箱通过 Apple 与 Google 注册后，两个账号都可使用 |
| 账号校验 | 切换社交账号后再发起 PIN 验证或重置时，需提示账号与钱包不匹配 |
| Web / 插件 onboarding | Web 与插件连接场景下的首次安装、Side Panel 与自动回流，详见独立需求文档 |

---

## 3. 业务规则

| 规则项 | 规则描述 |
|-------|---------|
| 创建入口一致性 | Google / Apple 登录、PIN 和 App Passcode 三段流程必须完整串联 |
| 单实例约束 | 当前设备不能重复创建 Keyless Wallet |
| 恢复一致性 | 恢复地址必须与创建地址一致 |
| PIN 安全策略 | PIN 长度、错误次数、冷却和会话过期策略必须严格生效 |
| 能力边界 | Keyless Wallet 不参与助记词导出、KeyTag、Lite、云备份、OneKey Cloud 同步和数据传输 |
| 账号隔离 | 社交账号切换后必须校验是否与当前钱包匹配 |

---

## 4. 已知风险

- 社交登录、PIN、Passcode 三段流程跨端实现差异较大，容易出现半完成状态。
- PIN 冷却和 8 分钟会话过期属于时间敏感逻辑，需要重点验证前后台切换和重进页面。
- 同邮箱 Apple / Google 双渠道并存时，图标、账号校验和重置 PIN 容易出现串号。
