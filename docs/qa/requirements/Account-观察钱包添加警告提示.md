# Account - 观察钱包添加警告提示

> 模块：Account & Wallet  
> 来源：OK-44830（观察钱包添加警告提示）  
> App 版本：6.2.0（影响版本：6.1.0）  
> 日期：2026-04-07  
> 测试端：Desktop / Extension / iOS / Android / Web

---

## 1. 需求背景

近期收到多起观察钱包相关客诉。典型风险场景是用户被误导导入他人公钥，误以为该地址由自己控制，并持续向该地址转入资产，直到尝试发送时才发现观察钱包无法签名和转账。为降低误充风险，需要在观察钱包关键页面增加持续可见的风险提示。

---

## 2. 功能描述

### 2.1 提示条生效范围

| 项目 | 说明 |
|-----|------|
| 生效账户类型 | 观察钱包 / 观察账户（Watched Only Account） |
| 首页位置 | 钱包首页顶部 |
| Token 详情页位置 | Token 详情页顶部，位于余额区域上方 |
| 提示样式 | 持久显示的黄色 warning 提示条 |
| 提示文案 | `这是观察账户，请勿转入资产。` |

### 2.2 提示条展示行为

| 项目 | 说明 |
|-----|------|
| 持续显示 | 进入观察钱包后提示条始终显示，不可手动关闭 |
| 页面切换 | 在观察钱包下进入 Token 详情页，详情页继续显示同一风险提示 |
| 账户切换 | 切换到非观察钱包后，提示条自动消失 |
| 返回行为 | 从 Token 详情返回首页后，若当前仍为观察钱包，首页提示条继续显示 |

### 2.3 多端一致性

| 项目 | 说明 |
|-----|------|
| 覆盖平台 | Desktop、Extension、Mobile（iOS / Android）、Web |
| 一致性要求 | 各端均按相同生效条件、位置语义与文案展示风险提示 |

---

## 3. 场景与预期

| 场景 | 操作 | 预期结果 |
|-----|------|---------|
| 首页显示 | 切换到观察钱包 | 首页顶部显示黄色 warning 提示条，文案为 `这是观察账户，请勿转入资产。` |
| 首页消失 | 从观察钱包切换到 HD 钱包或其他非观察钱包 | 首页提示条消失 |
| Token 详情显示 | 在观察钱包下点击任意 Token 进入详情页 | 详情页顶部（余额上方）显示黄色 warning 提示条 |
| Token 详情不显示 | 切换到非观察钱包后进入 Token 详情页 | 不显示该 warning 提示条 |
| 提示条持久性 | 在观察钱包页面停留、返回、再次进入详情页 | 提示条不可关闭，符合页面条件时持续显示 |

---

## 4. 边界与异常口径

| 类型 | 规则 |
|------|------|
| 账户限定 | 仅观察钱包显示该提示；HD 钱包、硬件钱包、私钥钱包、外部账户、无私钥钱包等非观察账户不显示 |
| 页面限定 | 当前需求仅覆盖首页顶部与 Token 详情页顶部；未明确要求的其他页面不应擅自扩散提示 |
| 文案固定 | 提示文案需与需求一致，不可缺字、改字或展示为可关闭提示 |
| 多 Token 一致性 | 观察钱包下进入任意 Token 详情页，均应按相同规则展示 |

---

## 5. 已知风险

- 首页顶部提示条可能与已有顶部内容竞争空间，需要验证小屏设备和窄窗口下的换行与截断表现。
- 账户切换、页面返回和 Token 详情跳转过程中，提示条显隐可能出现残留或闪烁，需要重点验证状态刷新时机。
- 多端实现可能存在容器层级差异，需要确认提示条在 Desktop、Extension、Mobile、Web 上的相对位置一致。
