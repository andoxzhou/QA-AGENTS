---

## 1. 需求背景

Web URL 钱包用于通过地址或 URL 直接访问链上账户视图，覆盖搜索访问、直接链接访问、浏览器兼容和跨页面切换。该场景既涉及 URL 路由，又涉及钱包二级页和网络选择器的差异行为，需要单独定义。

---

## 2. 功能描述

### 2.1 搜索访问

| 项目 | 说明 |
|-----|------|
| 搜索框 | 输入地址后自动发起检索，检索中显示灰色占位符 |
| 完全匹配 | 显示网络图标、网络名称和钱包地址 |
| 点击结果 | URL 转为 `1key.so/url-account/{网络}/{地址}`；Web 端改变网址，桌面 / 插件端进入二级页面 |
| EVM 网络 | 网络选择器可切换其他 EVM 网络 |
| 非 EVM 网络 | 网络选择器不可用 |
| 页面内容 | 现货、DeFi、NFT、历史 tab 切换展示 |

### 2.2 链接地址访问

| URL 类型 | 说明 |
|---------|------|
| 正确 URL | 打开对应 Web URL 钱包 |
| 地址与网络不匹配 | 显示错误提示页面 |
| 地址错误 | 显示错误提示页面 |
| 网络错误 | 显示错误提示页面 |
| `evm/0x...` | 打开 EVM 地址对应的 All Network 钱包页 |

### 2.3 浏览器兼容与页面切换

| 项目 | 说明 |
|-----|------|
| 浏览器兼容 | 支持 Chrome、Safari、Edge、Firefox、OneKey 内置浏览器 |
| 连续跳转 | 多次搜索不同地址后，显示和返回路径保持可用 |
| 跨页面切换 | 从 Swap / Market 等上级页面切换后，可再返回 URL 钱包二级页 |
| 二级页中断 | 杀进程或重启后回到主钱包账户页 |

---

## 3. 业务规则

| 规则项 | 规则描述 |
|-------|---------|
| URL 映射规则 | 搜索结果点击后必须映射到固定 URL 结构 |
| 网络选择器限制 | 仅 EVM 网络支持切换其他 EVM 网络 |
| 错误 URL 处理 | 地址错误、网络错误和地址网络不匹配都显示错误页面 |
| 浏览器兼容性 | 主流浏览器和内置浏览器都需支持 |
| 中断恢复 | 杀进程或重启后的返回路径固定为主钱包账户页 |

---

## 4. 已知风险

- URL 解析、地址格式和网络映射同时参与判断，错误页面容易出现错分支。
- Web 端直接改网址与桌面 / 插件端进入二级页的行为不同，需要分别验证。
- 连续跳转和跨页面返回路径容易出现历史栈异常。
