# Wallet - 闪电网络转账与 dApp

## 需求背景

闪电网络（Lightning Network）是基于 BTC 的链下支付协议，与链上转账在地址类型、金额单位、费用、历史、浏览器等维度不同。需覆盖：
- 三种收款：**发票**（BOLT11）/ LN Address / LNURL
- sats 与法币双向切换
- 单笔上限（1 BTC = 100,000,000 sats）与 1% 预留规则
- **发票描述** ≤ 40 字符（默认 `OneKey Invoice`）
- 无区块浏览器，交易哈希仅可复制
- 历史记录仅展示当前账户
- dApp（LApp）连接、支付、授权登录
- HD 钱包与 HW 钱包均需覆盖

## 功能描述

### 发送
- 粘贴或扫码：发票 / LN Address / LNURL
- 指定金额发票：金额自动填入不可改
- 未指定金额发票 / LN Address / LNURL：手动输入，法币 ↔ sats、Max
- 确认页：发送方、金额、手续费、**发票描述**
- 交易完成后展示历史记录

### 接收
- 生成发票（固定金额或开放金额）
- **发票描述** 最大 40 字符，默认 `OneKey Invoice`
- 展示发票码 + 二维码

### dApp（LApp）交互
- **连接授权**：站点发起连接请求，APP 弹出授权弹窗（显示 Origin、图标、名称）
- **支付**：站点生成发票 → 调起 APP → APP 弹出交易确认 → 用户确认 → 返回站点
- **赎回**：站点请求提现 → APP 生成发票发送给站点 → 站点支付 → APP 收款
- **授权登录**：站点通过 Alby / WebLN / Nostr Wallet Connect 协议请求登录，APP 授权后返回站点

## 业务规则

| 项 | 规则 |
|----|------|
| 最小单位 | 1 sat（整数）；1 BTC = 100,000,000 sats |
| 单笔上限 | 100,000,000 sats（1 BTC） |
| 最小发送 | 1 sat（未指定金额发票 / LN Address / LNURL） |
| **法币输入 → sats 四舍五入** | 输入法币金额时，换算出的 sats **必须四舍五入为整数**，不得显示小数；否则交易会失败（协议不支持非整数 sats），确认页与历史记录金额同为整数 |
| 余额预留 | 发票金额 > 余额 × 99% 时禁止提交，提示「余额不足！请至少预留 invoice 金额的 1%」 |
| **发票描述** 长度 | 最大 40 字符（含空格、换行、Emoji） |
| **发票描述** 默认 | 未填写时 `OneKey Invoice` |
| 交易哈希 | 显示，可复制，**无跳转**（闪电网络无区块浏览器） |
| 交易对象 | 历史记录中**仅展示当前账户**，不展示交易对象用户 |
| 网络费 | 显示，**不可修改、不可切档**（由路由节点决定，金额不固定） |
| 网络标识 | `Lightning` / `闪电网络` |
| 账户隔离 | 多个闪电账户余额、历史记录独立 |
| 选择地址 | 发送流程中「我的账户」Tab **不显示**闪电账户（仅「最近」Tab） |
| 主币 = 0 拦截 | 闪电网络**不受**单链视图「主币余额为 0」发送前置拦截影响 |
| 代币 | 闪电网络无代币概念，仅主币 sats |

### 异常发票处理
| 场景 | 预期 |
|------|------|
| 已过期发票 | 「地址无效」 |
| 已支付发票 | 「发票已支付」 |
| 格式错误字符串 | 提示「地址无效」 |

### Loading / 性能
- 每次 loading 时长 ≤ 2 秒
- 交易通常秒级完成，状态不长时间卡在处理中

### HW 钱包支持
- ✅ 发送（发票 / LN Address / LNURL）：**需要 HW 签名**
- ✅ dApp 支付（LApp Deposit）：**需要 HW 签名**
- ❌ 接收（生成发票）：App 侧，**无 HW 签名**
- ❌ dApp 连接授权、赎回（Withdraw）：**无 HW 签名**
- ⚠️ dApp 授权登录（LNURL-auth）：HD 在 App 内完成；**HW 需设备签名**（见链规则）
- ❌ 代币转账：闪电网络无代币概念
- HW 签名屏：发送方、金额（sats）、链 `Lightning`、网络费、**发票描述**（如有）

## 已知风险

1. **区块浏览器缺失的用户困惑**：对齐其他链心智的用户习惯点击哈希跳转，需要文案 / icon 做预期管理
2. **历史记录仅展示当前账户**：无 From/To 对方地址，用户可能对"交易是否完成"产生疑虑
3. **发票金额锁定**：指定金额发票输入框不可改，若无提示易误判为 bug
4. **1% 预留规则**：文案信息密度高，建议拆两行
5. **通道容量限制**：超过通道入/出站容量的大额转账会失败，需要适当提示
6. **路由失败**：偶发的路由不通导致交易失败，不应阻塞用户重试
