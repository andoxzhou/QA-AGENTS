# Hardware - Onboarding 强制固件更新

> 模块：HW & App  
> 来源：OK-51595（Implement forced firmware update during onboarding for devices with outdated versions）  
> App 版本：6.1.0  
> 日期：2026-03-24  
> 测试端：Desktop / iOS / Android（硬件 Onboarding 场景）

---

## 1. 需求背景

在硬件钱包 Onboarding 流程中，当前固件更新步骤允许用户跳过。为降低旧固件带来的稳定性和安全风险，需要在设备固件版本显著落后时，强制用户先完成固件更新，禁止跳过后继续流程。

---

## 2. 功能描述

### 2.1 强制更新触发范围

| 项目 | 说明 |
|-----|------|
| 触发阶段 | 硬件钱包 Onboarding 流程中的固件检查/更新步骤 |
| 生效对象 | 系统固件（Firmware）与蓝牙固件（BLE Firmware） |
| 生效动作 | 满足强制更新条件时，隐藏「跳过」按钮，仅保留「立即更新」 |
| 适用设备 | Pro / Touch / Classic / Classic 1S / Classic Pure / Mini |

### 2.2 版本判断规则

| 规则项 | 判定逻辑 | 结果 |
|-------|---------|------|
| major 落后 | 设备版本 major < 最新版本 major | 强制更新（不可跳过） |
| minor 落后 | major 相同，且 `latest.minor - device.minor > 2` | 强制更新（不可跳过） |
| patch 落后 | major + minor 相同，且 `latest.patch - device.patch > 2` | 强制更新（不可跳过） |
| 未超阈值 | 不满足上述任一条件 | 允许跳过 |

### 2.3 示例口径

- 示例：最新固件 `4.19.0`，设备固件 `4.16.1`，minor 差值为 3（>2），命中强制更新，固件步骤不显示「跳过」。

---

## 3. 场景与预期

| 场景 | 输入示例 | 预期结果 |
|-----|---------|---------|
| 系统固件落后 >2 minor | 设备 `4.16.1`，最新 `4.19.0` | 无「跳过」按钮，仅「立即更新」 |
| 系统固件落后 ≤2 minor | 设备 `4.17.0`，最新 `4.19.0` | 有「跳过」按钮 |
| 系统固件已最新 | 设备与最新版本一致 | 固件检查步骤直接通过（✓），无更新提示 |
| 固件检查过程出错 | 例如设备断开连接 | 显示错误状态，「重试」与「跳过」均可用 |
| BLE 固件落后 >2 patch | 设备 `2.3.3`，最新 `2.3.6` | 无「跳过」按钮 |
| BLE 固件落后 ≤2 patch | 设备 `2.3.4`，最新 `2.3.6` | 有「跳过」按钮 |

---

## 4. 边界与异常口径

| 类型 | 规则 |
|------|------|
| 阈值边界 | 差值等于 2（minor/patch）时不强制；差值大于 2 时强制 |
| 错误兜底 | 固件检查异常（如连接中断）时，不进入强制更新态，保留「重试」和「跳过」以保障流程可恢复 |
| 多固件并存 | 系统固件与 BLE 固件任一命中强制条件，即整体进入不可跳过状态 |

---

## 5. 已知风险

- 不同设备型号的固件字段上报口径可能存在差异，需要确认版本比较逻辑在各型号一致生效。
- 断连/重连过程中固件检查状态切换可能出现短暂 UI 闪烁，需要验证按钮显隐稳定性。
- 若最新版本元数据拉取失败，需明确是否回退到可跳过策略。
