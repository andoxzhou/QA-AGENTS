## 1. 需求背景

观察账户用于在无私钥前提下查看资产和交易信息，支持地址、域名和公钥导入。该能力涉及网络匹配、地址合法性、重复导入、公钥导出与风险提示衔接，需要单独整理。

---

## 2. 功能描述

### 2.1 导入方式

| 方式 | 说明 |
|-----|------|
| 地址 | 支持不同网络地址，选择对应网络后导入 |
| 域名 | 支持不同后缀或网络域名，解析后导入 |
| 公钥 | 支持不同网络公钥，选择对应网络后导入 |

### 2.2 校验与异常

| 项目 | 说明 |
|-----|------|
| 输入为空 | 按钮置灰不可点击 |
| 非法字符 / 无效地址 / 无效域名 | 显示错误提示 |
| 网络与地址不匹配 | 显示错误提示 |
| 未激活 Cardano 地址 | 地址校验通过，但导入时提示账户未激活 |
| BTC xpub 不同派生路径 | 可导入不同地址，导出公钥与导入值一致 |
| 重复导入地址 | 直接定位到已导入的观察账户 |
| 导入相同地址对应公钥 | 允许导入，且可导出公钥 |
| 空格处理 | 地址前后空格过滤后导入；地址中间空格显示错误 |

### 2.3 关联显示

| 项目 | 说明 |
|-----|------|
| 默认名称 | 导入时可预设账户名；留空时使用默认名称 |
| 风险提示 | 观察账户首页与 Token 详情页风险提示按独立需求文档执行 |

---

## 3. 业务规则

| 规则项 | 规则描述 |
|-------|---------|
| 导入入口支持 | 地址、域名和公钥三种导入方式都必须可用 |
| 网络匹配优先 | 地址、公钥和域名都必须与所选网络匹配 |
| 重复导入处理 | 重复地址不能新建第二个相同观察账户 |
| 公钥例外 | 相同地址对应公钥可以导入并支持公钥导出 |
| 空格处理 | 仅允许自动过滤首尾空格，不允许容忍中间空格 |

---

## 4. 已知风险

- 地址、域名和公钥三种导入入口的网络校验逻辑不同，容易出现规则不一致。
- BTC xpub、多派生路径和 Cardano 未激活地址属于高风险边界场景。
- 观察账户导入后的风险提示与复制 / 接收能力需要和钱包账户管理口径联动验证。
