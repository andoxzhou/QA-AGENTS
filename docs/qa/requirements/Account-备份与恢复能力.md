# Account - 备份与恢复能力

> 模块：Account  
> 来源：基于 `docs/qa/rules/account-rules.md` 补充整理  
> App 版本：待定  
> 日期：2026-04-14  
> 测试端：Desktop / Extension / iOS / Android / Web

---

## 1. 需求背景

Account 模块除手动助记词备份外，还支持云备份、OneKey Lite、OneKey KeyTag 和数据传输等多种备份与恢复方式。不同能力在数据范围、权限验证、媒介限制和恢复结果上存在明显差异，需要统一整理。

---

## 2. 功能描述

### 2.1 云备份与导入

| 项目 | 说明 |
|-----|------|
| 备份范围 | 展示当前 App 内所有 HD 钱包和私钥 / 公钥 / 观察账户；不包含无私钥钱包 |
| 账号来源 | Google Drive 需登录或选择 Google 账号；iCloud 使用系统账号 |
| 首次备份 | 必须设置备份密码，两次输入一致且长度不少于 6 位 |
| 再次备份 | 必须验证已设置的备份密码 |
| 导入列表 | 新版格式数据列表展示备份时间和钱包账户数量；旧格式数据导入口位于新版列表下方 |
| 新版导入 | 校验备份密码后下载所选钱包，再设置当前 App 密码 |
| 旧版导入 | 支持从旧数据入口导入旧格式备份 |
| 删除与重置 | 删除备份前需要二次确认并验证 App 密码；重置备份密码仅影响新备份 |

### 2.2 OneKey Lite

| 项目 | 说明 |
|-----|------|
| NFC 权限 | 首次进入备份、导入、修改 PIN、重置时申请 NFC 权限 |
| 备份能力 | 支持将 12 / 15 / 18 / 21 / 24 位助记词钱包备份到空卡 |
| PIN 规则 | 需设置并二次确认 6 位 PIN |
| PIN 验证 | 覆盖备份、导入、修改 PIN 前都需先验证 Lite PIN |
| PIN 错误 | 错误次数递减，累计 10 次错误后卡片重置 |
| 重置确认 | 需先输入 `reset` 才可发起重置 |
| 识别异常 | 贴卡取消、移开卡片、空卡、不同卡片等都显示连接或卡片错误提示 |

### 2.3 OneKey KeyTag

| 项目 | 说明 |
|-----|------|
| 入口 | 设置页显示购买、备份、导入入口 |
| 导入 | 助记词输入框规则与助记词导入一致，并显示 dotmap 转换引导 |
| 备份方式 | 支持选择现有软件钱包备份，也支持手动输入助记词生成 dotmap |
| 权限 | 选择现有钱包备份前需完成密码或生物识别验证 |
| 教程 | 右上角帮助展示备份教程，勾选“我已备份”后才可关闭 |
| 核对 | dotmap 点位和行数需与助记词一一对应 |

### 2.4 数据传输

| 项目 | 说明 |
|-----|------|
| 建立连接 | 默认选中二维码 tab，二维码与配对码一致 |
| 配对码 | 支持粘贴、扫码录入，默认暗文显示，可切换明文 |
| 配对校验 | 不可连接自己的配对码；无效码、超时、一端退出、第三端抢占都需要中断并提示 |
| 自定义服务器 | 支持域名、`http/https`、IP+端口，自动补全协议并记录历史 |
| 数据范围 | 支持导出和导入软件钱包、私钥钱包、观察钱包；无私钥钱包不参与 |
| 数据限制 | HD 钱包 LN / ADA 数据不导入 |

## 3. 业务规则

| 规则项 | 规则描述 |
|-------|---------|
| 备份范围边界 | 云备份不包含无私钥钱包 |
| 密码隔离 | 云备份密码与当前 App 密码可不同；重置备份密码仅影响新备份 |
| 版本兼容 | 云备份导入同时兼容新版与旧版数据 |
| Lite 安全门槛 | Lite 覆盖、导入、修改 PIN 都必须先验证 Lite PIN |
| KeyTag 校验 | KeyTag 的导入规则、风险确认和 dotmap 核对需保持一致 |
| 数据传输限制 | 无私钥钱包不参与数据传输，HD 钱包 LN / ADA 不导入 |
| 异常可解释 | NFC、卡片识别、配对码、超时与连接中断都需显示可解释提示 |

---

## 4. 已知风险

- 云备份新版与旧版数据入口并存，容易出现列表展示、导入密码和结果归属不一致。
- Lite 的 NFC、PIN、空卡、识别异常组合较多，端间实现差异明显。
- KeyTag dotmap 与原助记词的映射校验容易因行列展示差异出现错误。
- 数据传输涉及双端状态与服务器配置，超时、中断和第三端抢占需要重点验证。
