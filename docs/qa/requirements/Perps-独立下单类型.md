# Perps - 独立下单类型（市价/限价止盈止损合并）

## 1. 需求背景

- 模块：Perps（合约交易）
- 功能：在现有市价单 / 限价单基础上，新增触发下单能力，下单语义与 Hyperliquid 对齐，并将创建入口由 4 种合并为 2 个触发下单模式（市价止盈止损、限价止盈止损）。
- 目的：
  - 将 TP/SL 从仅支持附加在开仓单上的「附单能力」，扩展为可单独创建的触发型订单。
  - 覆盖 Hyperliquid 的 4 种触发语义：市价止盈、市价止损、限价止盈、限价止损。
  - 修复当前「trigger 下单在服务层写死 isMarket=true，只支持 market-trigger」的能力缺口，补足 limit-trigger（isMarket=false + execution price）。

## 2. 功能概述

### 2.1 新增的独立订单类型

在 Perps 下单面板中，除现有「市价单」「限价单」类型外，新增 2 个触发下单模式入口，并在模式内切换止盈/止损语义：

1. 市价止盈止损（Market Trigger）：覆盖市价止盈（Take Profit Market）与市价止损（Stop Loss Market）
2. 限价止盈止损（Limit Trigger）：覆盖限价止盈（Take Profit Limit）与限价止损（Stop Loss Limit）

要求：
- 覆盖的 4 种触发语义仍保持「独立订单记录与撤单流程」，但 UI 创建入口合并为 2 个触发下单模式。
- 不影响现有「市价单」「限价单」下单逻辑与接口。
- 原有 TP/SL 附单能力保留、语义暂不调整（仅在下单侧新增触发下单模式）。

### 2.2 与 TP/SL 附单的关系

- 附单 TP/SL：
  - 仍通过开仓单上的「附带 TP/SL」链路下发，由服务封装成对应 trigger 订单。
  - 现有语义、默认行为、接口字段保持不变。
- 独立 TP/SL 订单（本需求）：
  - 通过单独的下单模式创建，无需依赖当前仓位的开仓操作。
  - 可对已有仓位补充 / 调整止盈止损，或对未来仓位设置预埋单（以 Hyperliquid 行为为参考）。

## 3. 服务层与接口规则

### 3.1 isMarket 与 execution price

当前实现：
- trigger 类订单在服务层写死 `isMarket = true`，仅覆盖「市价触发」场景（Take Market / Stop Market）。

新增要求：
- 支持 4 种触发订单，服务侧需要区分：
  - 市价止盈 / 市价止损：
    - `isMarket = true`
    - 不需要显式填写 `executionPrice`（按 Hyperliquid 市价触发语义执行）。
  - 限价止盈 / 限价止损：
    - `isMarket = false`
    - 必须提供 `executionPrice` 字段，代表触发后挂出的限价执行价。
- 服务端需对 `isMarket` 与 `executionPrice` 做参数校验：
  - `isMarket = true` 时禁止传入 `executionPrice` 或忽略该字段。
  - `isMarket = false` 时 `executionPrice` 必填且需通过价格范围/精度校验。

### 3.2 触发价与执行价

抽象口径（对标 Hyperliquid）：
- **触发价格（trigger price）**：达到该价格条件时，触发生成实际执行订单。
- **执行价格（execution price）**：
  - 市价类：由撮合/撮合代理按市价成交，不固定为某个价格。
  - 限价类：触发后挂出的实际限价价格，由 `executionPrice` 指定。

预期：
- 市价止盈 / 市价止损：用户只需填触发价；触发时生成「市价单」执行。
- 限价止盈 / 限价止损：用户需同时填「触发价」与「限价执行价」。

## 4. 交互与文案

### 4.1 下单模式入口

- 在当前下单模式切换区域（如：市价单 / 限价单 Tab 或下拉）中，新增 2 个「止盈止损」触发下单模式：
  - 文案示例：市价止盈止损、限价止盈止损。
- 选择不同触发下单模式时：
  - 表单字段联动更新（如是否展示执行价、触发价/执行价校验状态等）。
  - 保证已有市价单 / 限价单表单不受影响。
- 在触发下单模式内再切换「止盈/止损」，以决定触发语义与触发方向校验逻辑。

### 4.2 表单字段

- 市价止盈止损：
  - 必填：触发价格、数量、方向（买入/卖出）、杠杆 / 保证金模式（沿用现有设置）。
  - 不展示「执行价格」输入框。
- 限价止盈止损：
  - 必填：触发价格、执行价格、数量、方向。
  - 执行价格需有精度/上下限校验。

### 4.3 订单列表与详情

- 新增的 4 种订单类型需在：
  - 未触发订单列表（如「触发订单」「条件单」列表）
  - 历史订单 / 历史触发记录
  中以清晰文案标识订单类型与方向。
- 列表与详情需展示至少：
- 类型（市价止盈 / 市价止损 / 限价止盈 / 限价止损）
  - 状态（待触发 / 已触发 / 已撤单 / 失败等）
  - 触发价格
  - 执行价格（仅限价类）
  - 最终成交价格区间或成交均价

## 5. 风险与边界

- 需覆盖不同方向（多仓/空仓）下，4 种订单类型的触发条件是否正确（如多仓止盈/止损触发方向 vs 空仓相反）。
- 价格极端波动时：
  - 触发价被瞬间跨越时，订单仍需按 Hyperliquid 相同策略处理（例如市价触发后在可成交区间撮合，限价触发后可能直接成交或挂单）。
  - 若 `executionPrice` 显著偏离市价导致无法成交，需要有明确状态与提示。
- 需验证与原有 TP/SL 附单的共存情况：
  - 对同一仓位同时存在附单 TP/SL 与独立 TP/SL 订单的处理优先级与行为（由产品/服务端进一步补充）。
