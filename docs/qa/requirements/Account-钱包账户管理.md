# Account - 钱包账户管理

> 模块：Account  
> 来源：基于 `docs/qa/rules/account-rules.md` 补充整理  
> App 版本：待定  
> 日期：2026-04-14  
> 测试端：Desktop / Extension / iOS / Android / Web

---

## 1. 需求背景

钱包账户管理覆盖创建钱包、切换钱包与账户、删除与重置、名称与搜索、复制与接收、xfp 数据补充等高频基础能力，是所有账户模型的统一承载层，需要独立沉淀。

---

## 2. 功能描述

### 2.1 创建与切换

| 项目 | 说明 |
|-----|------|
| 创建钱包 | 支持自定义钱包名称，默认 `钱包 N`，支持选择或自定义头像 |
| 密码设置 | 创建或导入钱包时需设置 App 密码 |
| 切换钱包 | 切换后刷新资产并显示新钱包资产 |
| 切换账户 | 支持同钱包内切换派生账户 |
| 切换链 | 保持当前账户，切换显示链 |
| 余额与地址展示 | All networks、单链、观察账户、外部账户与聚合网络按规则展示余额和地址 |

### 2.2 删除与重置

| 项目 | 说明 |
|-----|------|
| 删除软件 / 硬件 / 隐藏钱包账户 | 删除后账户消失且不可恢复；同一次 App 启动仅首次显示移除确认 |
| 删除单链账户 | 始终弹出移除确认 |
| 删除观察 / 外部账户 | 直接删除，不弹移除确认 |
| 删除 QR / QR 隐藏钱包账户 | 不允许直接删除单个账户 |
| 删除钱包 | 删除整个钱包前需勾选确认并验证内存密码 |
| 删除硬件钱包细分 | 删除标准钱包不连带删除隐藏钱包；移除设备时连带删除隐藏钱包 |
| 重置 App | 清除所有数据，需要多重确认 |

### 2.3 名称、搜索与复制接收

| 项目 | 说明 |
|-----|------|
| 预设名称 | 导入观察账户时可预设账户名；留空时用默认名称 |
| 名称限制 | 最大 80 字符；同一钱包内名称不可重复；重命名不能为空 |
| 名称历史 | 修改成功后记录去重后的名称历史 |
| 搜索 | 支持完全匹配和部分匹配；有结果时隐藏添加账户按钮 |
| 复制 / 接收 | 按网络、账户类型、硬件校验与代币选择规则展示复制和接收能力 |

### 2.4 xfp 数据补充

| 项目 | 说明 |
|-----|------|
| 软件 / QR 钱包 | 解锁 App 并完成密码验证后自动静默补充 xfp |
| 未解锁场景 | 未输入密码时不补充 xfp；在需要密码的操作后自动补充 |
| 新建钱包 | 新建软件 / QR / 硬件 / 隐藏钱包默认已带 xfp |
| 硬件钱包 | 钱包首页和修改名称时提示补充钱包数据；添加账户与批量添加账户时静默补充 |
| 名称同步 | 同账户名称同步依赖 xfp 数据 |

## 3. 业务规则

| 规则项 | 规则描述 |
|-------|---------|
| 创建入口统一 | 名称、头像和 App 密码是创建或导入钱包的基础字段 |
| 删除安全门槛 | 删除钱包和重置 App 需要更高等级确认 |
| 名称治理 | 名称长度、唯一性、为空限制和历史记录都必须生效 |
| 搜索联动 | 搜索结果页中的复制、重命名、置顶、导出和移除都需可用 |
| 复制接收边界 | All networks、硬件钱包、观察账户和单链多代币账户的复制 / 接收口径不同 |
| xfp 依赖 | 同账户名称同步依赖 xfp 数据补充 |

---

## 4. 已知风险

- 钱包切换、账户切换和切链共存时，资产、地址与余额刷新容易不同步。
- 删除标准钱包、隐藏钱包和移除设备三类路径容易影响硬件钱包残留状态。
- 名称同步依赖 xfp 补充，若补充时机不一致，会引发跨钱包同账户名称不一致。
