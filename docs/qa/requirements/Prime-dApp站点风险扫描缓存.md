# Prime - dApp站点风险扫描缓存

> 模块：Prime  
> 来源：OK-52221（Browser - Prime 用户打开 dApp 时主动请求 Blockaid Site Scan 并缓存结果）  
> App 版本：6.2.0  
> 日期：2026-04-07  
> 测试端：iOS / Android / Desktop / Extension / Web（Browser / DApp 场景）

---

## 1. 需求背景

Prime 会员在 Browser（DApp）场景下新增一项风险检测增强权益：打开 dApp URL 时，主动触发 Blockaid Site Scan，并将结果缓存到服务端，供后续所有用户访问相同 dApp URL 时复用。在此基础上，原有 dApp URL 风险检测逻辑保持不变。

---

## 2. 功能描述

### 2.1 Prime 用户主动请求

| 项目 | 说明 |
|-----|------|
| 触发用户 | Prime 用户 |
| 触发时机 | 每次打开 dApp URL 时 |
| 请求动作 | 主动请求 Blockaid Site Scan |
| 能力定位 | 不再仅用于复核，而是作为主动风险检测输入来源 |

### 2.2 服务端缓存与结果复用

| 项目 | 说明 |
|-----|------|
| 缓存对象 | Blockaid Site Scan 返回结果 |
| 缓存位置 | 服务端 |
| 复用范围 | 后续所有用户访问相同 dApp URL 时，在缓存有效期内可复用缓存结果 |
| 缓存有效期 | 3 天 |
| 缓存过期结果 | 缓存过期后，与未使用 Blockaid 检查过一致，返回结果为 `unknown` |
| 缓存策略 | 缓存有效期为 3 天；到期后的更新机制由服务端策略定义 |

### 2.3 与现有方案关系

| 项目 | 说明 |
|-----|------|
| 现有风险检测 | 原有 dApp URL 风险检测逻辑保持不变 |
| 能力关系 | 本需求为增量能力，不替换既有检测流程 |
| 非 Prime 用户 | 不新增主动请求动作；若命中已有缓存结果，可直接复用 |

---

## 3. 场景与预期

| 场景 | 操作 | 预期结果 |
|-----|------|---------|
| Prime 主动请求 | Prime 用户打开任意 dApp URL | 触发 Blockaid Site Scan 请求流程 |
| 缓存写入 | Prime 用户首次打开某 dApp URL 并获得返回结果 | 返回结果被缓存到服务端 |
| 全量用户复用缓存 | 其他用户在缓存有效期内访问相同 dApp URL | 可复用已有缓存结果 |
| 缓存过期 | 相同 dApp URL 的缓存超过 3 天后再次访问 | 返回结果为 `unknown`，表现与未使用 Blockaid 检查过一致 |
| 原逻辑保持 | 任意用户访问 dApp URL | 现有 dApp URL 风险检测链路保持不变，不因本需求被替换 |
| 非 Prime 行为 | 非 Prime 用户打开 dApp URL | 不新增 Prime 专属的主动请求动作；若缓存有效则可直接使用，否则返回 `unknown` |

---

## 4. 业务规则

| 规则项 | 规则描述 |
|-------|---------|
| Prime 主动触发 | Prime 用户每次打开 dApp URL 时，都需要进入主动请求 Blockaid Site Scan 的流程 |
| 缓存共享 | 同一 dApp URL 的扫描结果在 3 天有效期内可被后续所有用户共享使用 |
| 增量接入 | Blockaid Site Scan 主动请求与缓存复用为现有 dApp URL 风险检测的增量补充 |
| 服务端主导缓存 | 缓存有效期为 3 天；缓存是否命中及到期后的刷新由服务端策略控制 |
| 过期结果回退 | 缓存过期后，返回结果回退为 `unknown`，与未使用 Blockaid 检查过一致 |

---

## 5. 边界与异常口径

| 类型 | 规则 |
|------|------|
| 适用范围 | 当前需求仅说明 Prime 用户的主动请求行为，以及同 URL 缓存结果的全量用户复用 |
| URL 粒度 | 复用范围以“相同 dApp URL”为准，具体归一化或去参策略由服务端实现决定 |
| 缓存策略 | 当前缓存有效期固定为 3 天；若后续时效或更新机制变更，应同步补充需求文档 |
| 过期处理 | 缓存过期后不继续复用旧结果，返回 `unknown` |
| 异常兜底 | Blockaid 请求失败、超时或缓存不可用时，不应影响原有 dApp URL 风险检测逻辑继续执行 |

---

## 6. 已知风险

- 3 天缓存有效期若与实际服务端 URL 归一化策略、刷新策略组合不一致，可能导致命中率或更新时机与预期存在偏差。
- 相同 dApp URL 的判定若涉及域名归一化、查询参数或跳转链路，可能影响缓存命中率。
- 主动请求 Blockaid Site Scan 可能增加 dApp 首次打开时延，需要关注性能和超时兜底表现。
