> 测试端：Desktop / Extension / iOS / Android / Web

---

## 1. 需求背景

外部账户能力用于连接第三方钱包并在 OneKey 内展示对应账户。其行为同时覆盖 WalletConnect、浏览器注入、连接异常、重导入更新以及 OneKey 作为外部钱包的反向连接场景，需要统一梳理。

---

## 2. 功能描述

### 2.1 WalletConnect 连接

| 项目 | 说明 |
|-----|------|
| 连接流程 | onboarding → 点击 WalletConnect → 弹出 WalletConnect 弹窗 → 外部钱包扫码或搜索 → 在外部钱包确认连接 → 创建外部钱包账户 |
| 命名规则 | 外部钱包名称 + 序号 |
| 显示校验 | 显示连接钱包图标、网络图标与地址 |
| 单连接 | 每次重新点击 WalletConnect 时，之前连接的 WalletConnect 钱包被断开，仅保留最新连接账户 |

### 2.2 注入连接

| 项目 | 说明 |
|-----|------|
| 常驻钱包 | OneKey、Phantom、Coinbase、OKX、Zerion 常驻显示 |
| 其他插件 | 安装后显示，未安装不显示 |
| 未安装处理 | 点击未安装的常驻钱包时跳转对应 Chrome 商店页；非常驻插件未安装时不显示 |
| 已安装处理 | 已安装的钱包连接对应注入账户 |

### 2.3 异常与变更

| 项目 | 说明 |
|-----|------|
| 拒绝连接 | 外部钱包拒绝后显示拒绝提示，可再次发起连接 |
| 不删除重导入 | 不删除已注入账户时，修改外部钱包网络或地址后重新导入，更新网络图标和地址，名称与序号保持不变 |
| 删除后重导入 | 删除已注入账户后重新导入，名称保持不变，序号递增 |
| 插件禁用 | 禁用非常驻第三方插件后，对应钱包不再出现在注入钱包选择列表中；OneKey、Phantom、Coinbase、OKX、Zerion 等常驻钱包入口继续显示 |
| 相同地址 | 支持重复连接 |

### 2.4 OneKey 作为外部钱包与 Web 端限制

| 项目 | 说明 |
|-----|------|
| OneKey 反向连接 | OneKey 可通过 WalletConnect 或注入方式连接其他客户端 |
| 支持账户 | 软件钱包、观察账户、私钥账户、硬件钱包、QR 钱包都可作为外部钱包发起交易 |
| 交易回传 | 连接后交易结果和状态提示按连接客户端回传 |
| Web 端限制 | 钱包页与网络选择器隐藏，网络锁定为 All networks，连接入口只显示 WalletConnect 与注入两种方式 |

## 3. 业务规则

| 规则项 | 规则描述 |
|-------|---------|
| 单连接规则 | WalletConnect 每次仅保留最新连接账户 |
| 注入列表规则 | 常驻钱包始终显示，其他插件按安装状态显示 |
| 重导入更新规则 | 删除与不删除账户两种路径的名称、序号、网络图标和地址更新口径不同 |
| 插件禁用处理 | 被禁用的第三方插件钱包不应继续出现在可连接列表中 |
| Web 限制口径 | Web 外部账户场景隐藏本地钱包相关页面与设置 |

---

## 4. 已知风险

- WalletConnect、注入连接和反向连接共存时，连接源和账户来源容易混淆。
- 重导入路径会同时影响名称、序号、网络图标和地址展示，需要重点验证删除与不删除两条分支。
- Web 端外部账户模式下的页面裁剪和返回路径容易出现遗漏。
