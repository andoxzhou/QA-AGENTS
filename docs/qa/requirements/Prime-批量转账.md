# Prime - 批量转账（BTC 与 SOL 一对多）

> 模块：Prime  
> App 版本：5.20.0（PR #10473）  
> 日期：2026-03-30  
> 测试端：Desktop / iOS / Android

---

## 1. 需求背景

Bulk Send 一对多能力从原有 EVM/TRON 扩展到 BTC 与 SOL。该变更覆盖网络选择、地址输入、金额校验、交易构建、签名发送与复核展示，目标是支持 BTC/SOL 原生批量发送并降低批量场景失败率。

---

## 2. 功能描述

### 2.1 网络支持扩展

| 项目 | 说明 |
|-----|------|
| 功能入口 | 批量转账（Bulk Send） |
| 新增网络 | BTC、SOL（开发环境含 sBTC） |
| 转账模式 | 一对多批量发送 |

### 2.2 BTC 地址类型选择

| 项目 | 说明 |
|-----|------|
| 页面 | 发送地址输入页（BTC） |
| 新增能力 | 地址类型选择器 |
| 预期行为 | 切换地址类型后，发送账户地址与派生类型联动更新 |

### 2.3 最小转账金额校验

| 项目 | 说明 |
|-----|------|
| 生效页面 | 收款地址输入页、金额设置页（指定金额 / 范围金额 / 逐地址编辑） |
| 校验规则 | 金额低于链最小转账金额时阻止提交 |
| 提示行为 | 显示最小金额错误文案 |

### 2.4 BTC/SOL 原生批量构建与发送

| 项目 | 说明 |
|-----|------|
| 构建方式 | 一对多拆分为多笔 unsignedTx |
| 授权流程 | BTC/SOL 不走 approve 流程 |
| 发送方式 | 多笔交易可连续发送 |

### 2.5 BTC 签名前刷新

| 项目 | 说明 |
|-----|------|
| 触发时机 | 每笔 BTC 交易签名前 |
| 刷新内容 | 刷新 unsignedTx（重取 UTXO，保留费率） |
| 目标 | 降低双花/UTXO 失效导致失败 |

### 2.6 SOL 复核页费用与余额拦截

| 项目 | 说明 |
|-----|------|
| 新增展示 | ATA 租金费用、所需 SOL 总额 |
| 余额校验 | SOL 余额不足时拦截提交 |
| 按钮状态 | 提交按钮禁用 |

### 2.7 多链拆分交易费说明

| 项目 | 说明 |
|-----|------|
| 生效场景 | 转账拆分为多笔时 |
| 展示内容 | 拆分交易笔数与网络费说明弹层 |
| 用户感知 | 总网络费由多笔叠加构成 |

---

## 3. 业务规则

| 规则项 | 规则描述 |
|-------|---------|
| 网络扩展 | Bulk Send 支持网络包含 BTC、SOL（开发环境含 sBTC） |
| 地址联动 | BTC 地址类型切换后，地址与派生类型同步更新 |
| 最小金额 | 所有金额输入模式统一执行 minTransferAmount 校验 |
| 提交流程 | BTC/SOL 批量发送使用原生多笔构建，不走 approve |
| BTC 稳定性 | 每笔签名前刷新 unsignedTx 以降低 UTXO 冲突风险 |
| SOL 费用可见 | 复核页显示 ATA 租金与所需 SOL 总额 |
| SOL 余额拦截 | 余额不足时提交按钮禁用并显示告警 |
| 费用说明 | 拆分交易场景显示笔数与网络费叠加说明 |

---

## 4. 已知风险

- BTC 多笔发送过程中 UTXO 状态变化快，存在签名前后可用输入变化风险
- SOL ATA 租金与网络费波动可能导致复核页展示与提交时刻存在差异
- 多笔拆分交易在弱网下可能出现部分成功、部分失败的用户理解成本
- 多端金额输入模式校验口径不一致会导致边界场景行为偏差
