# Account - 网页端Keyless连接插件Onboarding体验

> 模块：Account（Keyless Wallet）  
> 来源：OK-49859（打通网页端 Keyless 连接插件的 onboarding 体验）  
> App 版本：6.2.0  
> 日期：2026-04-07  
> 测试端：Web / Extension（网页端连接插件场景）

---

## 1. 需求背景

为让用户可以直接在网页端通过邮箱登录并交易，而无需先下载独立 App，需要打通网页端与 OneKey 插件之间的 Keyless onboarding 与连接流程。插件作为中间层负责承接 Keyless 登录、创建钱包与 DApp 连接，从而兼顾安全性和服务端压力控制。

---

## 2. 功能描述

### 2.1 Web 端登录入口

| 项目 | 说明 |
|-----|------|
| 登录入口 | Web 端展示 `Google` 和 `Apple` 登录按钮 |
| 入口作用 | 用户选择某个社交登录方式后，驱动后续插件侧的 Keyless onboarding 流程 |
| 登录方式透传 | 插件自动拉起的 Keyless 登录方式需与 Web 端用户点击的登录项保持一致 |

### 2.2 插件未安装场景

| 项目 | 说明 |
|-----|------|
| 未安装提示 | 当用户点击 Web 端 `Google` / `Apple` 登录入口且插件未安装时，提示用户先安装插件 |
| 首次安装模拟 | 本地测试需先卸载插件后重新拖拽安装；禁用/启用插件或重载插件均不视为首次安装场景 |
| 安装后行为 | 插件安装成功后，等待几秒，自动进入大屏 onboarding 流程 |
| 触发原因 | 因安装场景没有用户动作触发，受插件权限限制，无法直接弹出 side panel |

### 2.3 插件已安装但无 Keyless 钱包

| 项目 | 说明 |
|-----|------|
| 触发条件 | 插件已安装，但插件端不存在 Keyless 钱包 |
| 入口行为 | 用户在 Web 端点击 `Google` / `Apple` 登录后，插件弹出 Side Panel 模式 |
| onboarding 行为 | 用户在 Side Panel 中完成 onboarding 并创建 Keyless 钱包 |
| 收尾行为 | onboarding 完成后弹出 DApp 连接窗，完成 Web 端连接 |

### 2.4 插件已安装且已有 Keyless 钱包

| 项目 | 说明 |
|-----|------|
| 触发条件 | 插件已安装，且插件端已有 Keyless 钱包 |
| 流程行为 | 系统直接走默认连接流程 |
| 对齐口径 | 体验与点击 `OneKey` 连接插件一致 |
| 钱包选择 | 最终连接哪个钱包由用户自主决定 |

### 2.5 连接完成后的回流

| 项目 | 说明 |
|-----|------|
| 连接窗 | onboarding 流程完成后，插件端弹出 DApp 连接窗 |
| 连接完成 | 用户确认连接后，Web 端刷新并自动连接该钱包 |
| 成功结果 | 用户可在 Web 端以该 Keyless 钱包身份继续后续交易流程 |

## 3. 场景与预期

| 场景 | 操作 | 预期结果 |
|-----|------|---------|
| 首次安装引导 | Web 端点击 `Google` / `Apple` 登录，插件未安装 | 提示安装插件；安装后自动进入大屏 onboarding |
| 登录方式继承 | Web 端选择 `Google` 或 `Apple` | 插件自动弹出的 Keyless 登录方式与 Web 端选择一致 |
| 首装后自动 onboarding | 首次安装成功后等待几秒 | 自动进入大屏 onboarding，而不是 side panel |
| 已安装无 Keyless | 插件已安装但无 Keyless 钱包时点击登录 | 弹出 Side Panel，完成 onboarding 后再弹连接窗 |
| 已安装有 Keyless | 插件已安装且已有 Keyless 钱包时点击登录 | 直接进入默认连接流程，用户自主选择钱包 |
| 连接完成回流 | 完成插件连接窗确认 | Web 端刷新并自动连接该钱包 |

---

## 4. 业务规则

| 规则项 | 规则描述 |
|-------|---------|
| 登录入口一致性 | Web 端提供的 `Google` / `Apple` 入口，必须与插件侧实际拉起的 Keyless 登录方式一致 |
| 首次安装特殊流 | 插件首次安装完成后必须走大屏 onboarding；不得因无手势触发而错误尝试弹 side panel |
| 钱包存在性分流 | 插件是否已有 Keyless 钱包决定后续进入 side panel onboarding 还是默认连接流程 |
| 连接闭环 | onboarding 完成后必须继续进入 DApp 连接窗，完成连接后 Web 端自动连接钱包 |
| 默认连接兼容 | 已有 Keyless 钱包场景下，体验需与既有 `OneKey` 插件连接流程保持一致 |

---

## 5. 边界与异常口径

| 类型 | 规则 |
|------|------|
| 首次安装判定 | 仅“卸载后重新安装”视为首次安装；禁用/启用、重载插件不应触发首次安装流 |
| 权限限制 | 首次安装场景因缺少用户动作触发，不能依赖 side panel 拉起流程 |
| 钱包缺失 | 插件已安装但无 Keyless 钱包时，必须进入 Side Panel onboarding，不得直接进入默认连接 |
| 钱包已存在 | 插件已安装且已有 Keyless 钱包时，不重复走 onboarding 创建流程 |
| 连接前置 | Web 端自动连接必须发生在用户完成插件连接确认之后 |

---

## 6. 已知风险

- 首次安装后的自动跳转依赖插件安装完成后的状态感知与延时处理，容易出现未拉起或重复拉起 onboarding。
- Web 端所选登录方式与插件侧实际拉起方式之间存在跨端状态同步风险，需要重点验证 Google / Apple 两条路径。
- onboarding 完成后到连接窗弹出、再到 Web 自动刷新连接之间存在串联时序，若任一环节延迟或中断，可能造成“钱包已创建但网页未连接”的半成功状态。
