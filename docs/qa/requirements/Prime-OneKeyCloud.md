# Prime - OneKey Cloud（Keyless Sync）

> 模块：Prime  
> 来源：OK-51225（Keyless wallet Sync）  
> App 版本：6.2.0（影响版本：6.1.0，修复版本：6.2.0）  
> 日期：2026-03-24  
> 测试端：iOS / Android / Desktop / Extension / Web

---

## 1. 需求背景

OneKey Cloud 以后以 Keyless Wallet 作为唯一同步载体，产品规则、页面状态与测试口径均按 Keyless Sync 设计，不再从 OneKey ID 同步模式出发整理长期规则。用户创建或恢复 Keyless Wallet 后，可通过 OneKey Cloud 开启、关闭、手动同步并恢复同步能力。

---

## 2. 功能描述

### 2.1 OneKey Cloud 页面状态

| 状态 | 触发条件 | 页面表现 |
|-----|---------|---------|
| 未创建 Keyless Wallet | 无 Keyless Wallet，且同步关闭 | 显示插图、说明文案与 `Create and enable syncing` 按钮 |
| 已创建 Keyless Wallet，同步关闭 | 已有 Keyless Wallet，但同步关闭 | 显示 OneKey Cloud 开关，默认 OFF |
| Keyless Sync 已开启 | Keyless 同步已开启 | 显示开关 ON、Keyless Wallet 信息行（钱包 icon + 名称）、`Sync now` |
| Syncing Paused | Keyless 同步已开启，但对应 Keyless Wallet 已删除 | 显示 `Syncing Paused` 告警与恢复按钮 |

### 2.2 功能入口迁移行为

| 用户历史状态 | 入口展示 | 说明 |
|------------|---------|------|
| 曾使用过 OneKeyID 同步 | 提示切换为 Keyless 同步 | 引导用户从 OneKeyID 同步迁移到 Keyless 同步 |
| 未使用过 OneKeyID 同步 | 仅显示启用 Keyless 同步 | 新用户直接进入 Keyless 同步流程 |
| 已启用 OneKeyID 同步（当前生效中） | 不受影响，保持原有功能 | 已开启的 OneKeyID 同步继续正常工作 |

### 2.3 创建 Keyless Wallet 后自动开启同步

| 项目 | 说明 |
|-----|------|
| 生效入口 | 任一 Keyless Wallet 创建入口（如 Cloud Sync 页面、Onboarding） |
| 自动行为 | 创建 Keyless Wallet 成功后自动开启 Keyless 同步 |
| 取消创建 | 创建中途取消，页面状态保持原场景，不触发自动开通 |
| 用户主动关闭 | 用户手动关闭同步后，重启 App 仍保持关闭，不自动重开 |

### 2.4 同步开启、关闭与手动同步

| 项目 | 说明 |
|-----|------|
| 开启同步 | 在「已创建 Keyless Wallet，同步关闭」状态打开开关后进入「Keyless Sync 已开启」状态 |
| 关闭同步 | 在「Keyless Sync 已开启」状态关闭开关后回到关闭状态 |
| 钱包信息行 | 开启同步后展示 Keyless Wallet 信息行（钱包 icon + 名称） |
| 手动同步 | 点击 `Sync now` 时，若内存密码仍有效则直接进入同步 loading；若内存密码已失效，则先进行密码验证，再进入同步 loading；成功后展示 toast |
| 实时同步与内存密码 | 内存密码失效仅约束「手动同步」须先完成密码验证；修改即时同步、在线端通过 WS 接收更新不因内存密码失效而中断 |
| 暂停态同步 | `Syncing Paused` 状态下点击 `Sync now`，显示错误 toast |

### 2.5 自动锁定策略

| 项目 | 说明 |
|-----|------|
| Keyless Sync | 不限制自动锁定时长，`4 小时` 与 `永不` 选项可正常选择 |

### 2.6 同步范围与触发

| 同步对象 | 说明 |
|---------|------|
| HD / HW / QR / 隐藏钱包账户名 | 修改后同步到其他端，旧名称进入设备历史 |
| 私钥 / 公钥 / 观察钱包账户名 | 仅同步对应账户 |
| 空 passphrase 隐藏钱包与标准钱包 | 名称分别同步 |
| 自定义代币 / 网络 / RPC | 支持新增与修改同步 |
| 浏览器书签 | 支持新增、修改、排序同步 |
| 市场观察列表 | 支持新增与修改同步 |
| 地址簿 | 支持新增与修改同步 |
| 自定义 RPC 开关 | 开关状态同步到其他端 |

| 触发类型 | 说明 |
|---------|------|
| 修改即时同步 | 对支持同步的对象执行新增、修改、排序或开关变更后，立即将最新内容上传云端；在线设备通过 WS 同步最新修改（与内存密码是否有效无关，见 2.4「实时同步与内存密码」） |
| 异步触发 | 打开账户选择器、地址簿、管理 Token、网络选择器、浏览器首页、Market 首页等入口时触发，默认 5 分钟间隔，用于补拉最新云端数据 |
| 立即触发 | 云同步开关、断线重连、重新激活 Prime、手动同步可立即触发 |

### 2.7 冲突处理与异常兜底

| 项目 | 说明 |
|-----|------|
| 冲突原则 | 同一数据在多端被修改时，按最后一次修改结果生效 |
| 钱包删除兜底 | 开启同步后若对应 Keyless Wallet 被删除，进入 `Syncing Paused` 状态 |
| 恢复流程 | 点击恢复按钮后，跳转到 Keyless Wallet 创建/恢复流程 |

### 2.8 页面加载与发布行为

| 项目 | 说明 |
|-----|------|
| 页面初始态 | 进入 OneKey Cloud 页面时，不应先出现错误场景再闪切 |
| 生产环境 | 页面 Header 不展示 Debug 按钮 |

## 3. 业务规则

| 规则项 | 规则描述 |
|-------|---------|
| Keyless 唯一性 | OneKey Cloud 仅以 Keyless Wallet 作为同步载体整理规则，不再包含 OneKey ID Sync 长期产品口径 |
| 入口迁移 | 曾使用过 OneKeyID 同步的用户在入口处提示切换为 Keyless 同步；未使用过的用户仅显示启用 Keyless 同步 |
| OneKeyID 同步兼容 | 已启用的 OneKeyID 同步功能不受影响，继续正常工作 |
| 状态唯一性 | OneKey Cloud 页面任一时刻只应命中一个状态 |
| 自动开通 | Keyless Wallet 创建成功后自动开启同步，无需二次手动开关 |
| 用户意图优先 | 用户手动关闭同步后，系统不可在重启后自动恢复开启 |
| 自动锁定 | Keyless Sync 不得因为自动锁定时长而限制开启或使用 |
| 实时同步与内存密码 | 内存密码失效不阻断修改即时同步与 WS 推送；仅「手动同步」在密码失效时须先验证 |
| 暂停态可恢复 | Keyless Wallet 删除后进入 `Syncing Paused` 状态，并可通过恢复流程回到可同步状态 |
| 同步结果一致性 | 同步对象、排序与开关状态需在支持范围内跨端保持一致 |
| 修改即时上传 | 对同步范围内内容的新增、修改、排序和开关变更，需立即上传云端 |
| 在线设备即时更新 | 已在线设备在收到 WS 消息后，应同步应用最新修改 |
| 冲突处理 | 多端同字段冲突统一按最后一次修改结果生效 |

## 4. 验收覆盖建议

| 覆盖域 | 核心验证点 |
|------|-----------|
| 页面状态 | 4 个页面状态入口条件与页面展示一一对应，不串态不闪态 |
| 自动开通 | Cloud / Onboarding 等 Keyless Wallet 创建入口均可自动开启同步 |
| 开关与手动同步 | 开关切换、钱包信息行展示、`Sync now` 在内存密码有效/失效两种状态下的行为正确，且成功反馈正确；内存密码失效时修改即时同步仍正常 |
| 修改同步 | 对账户名、地址簿、浏览器书签、市场观察列表、自定义代币/网络/RPC、RPC 开关等内容的修改后，会立即上传云端，且在线设备通过 WS 收到同步更新 |
| 异常路径 | Keyless Wallet 删除后进入暂停态，`Sync now` 错误反馈与恢复流程正确 |
| 策略约束 | Auto Lock 不限制 Keyless Sync；同步范围、触发时机与冲突规则正确生效 |

## 5. 已知风险

- Keyless Wallet 创建、删除、恢复后的本地状态持久化可能导致页面初始化状态判断异常，需要验证刷新与重启场景。
- 手动同步与异步同步共存时，去重和展示反馈的时序需要进一步验证。
- 多端同时修改同一同步对象时，最后写入时间基准需与客户端实现保持一致。
